## When to Write an ADR

Write an ADR if two or more of these are true:

1. **Could we have gone a different way?** There were viable alternatives we evaluated.
2. **Would someone new ask "why"?** The choice is non-obvious and needs explanation.
3. **Is it hard to reverse?** Changing this decision later would require significant rework.

## Status Lifecycle

Proposed -> Accepted -> Deprecated -> Superseded

Superseded ADRs stay in place with updated Status and a link to the replacement ADR. They are not archived or deleted.
