    2. **Config validation** (validation.rs): Add a validation check: if `config.mcp.enabled && config.mcp.auth_token.is_none()`, return a validation error "MCP enabled but mcp.auth_token is not set -- all /mcp endpoints require authentication". Follow existing validation patterns in the file.

    3. **Cargo.toml** (blufio-mcp-server): Add `transport-streamable-http-server` to rmcp features:

       rmcp = { workspace = true, features = ["server", "macros", "transport-io", "transport-streamable-http-server"] }

Return 401 Unauthorized if missing or wrong. Same pattern as gateway auth_middleware but simpler (bearer only, no keypair).

    6. **Update lib.rs**: Add `pub mod auth;` and `pub mod transport;`. Keep existing `serve_stdio` function.

  <action>
    1. **Gateway server.rs**: Modify `start_server` to accept an optional MCP Router. Add parameter `mcp_router: Option<Router>`. If Some, merge it into the app with `.nest("/mcp", mcp_router)` BEFORE the permissive CorsLayer (so MCP routes get their own CORS, not the permissive one).

       Restructure the app builder:
       ```rust
       let mut app = Router::new()
           .merge(public_routes)
           .merge(api_routes)
           .merge(ws_routes);

       if let Some(mcp) = mcp_router {
           app = app.nest("/mcp", mcp);
       }

       // Permissive CORS only for non-MCP routes
       // NOTE: The MCP router already has its own restricted CORS layer applied internally
       let app = app.layer(CorsLayer::permissive());
       ```

    2. **blufio/Cargo.toml**: Ensure `blufio-mcp-server` is a dependency with appropriate feature flag. Add it if not present:
       ```toml
       blufio-mcp-server = { path = "../blufio-mcp-server", optional = true }
       ```
       Ensure the `mcp-server` feature includes this dependency.

    3. **serve.rs**: After the gateway channel setup section, add MCP HTTP wiring:
       ```rust
       #[cfg(feature = "mcp-server")]
       let mcp_router = if config.mcp.enabled {
           // SEC: Validate auth_token is set (defense in depth -- validation.rs also checks)
           let auth_token = config.mcp.auth_token.clone().ok_or_else(|| {
               BlufioError::Security("MCP enabled but mcp.auth_token is not set".to_string())
           })?;

           // Redact MCP auth token in logs
           blufio_security::RedactingWriter::<std::io::Stderr>::add_vault_value(
               &vault_values, auth_token.clone(),
           );

           let mcp_cancel = cancel.child_token();
           let mcp_config = blufio_mcp_server::transport::mcp_service_config(mcp_cancel);
           let handler = blufio_mcp_server::BlufioMcpHandler::new(
               tool_registry.clone(), &config.mcp,
           );
           let router = blufio_mcp_server::transport::build_mcp_router(
               handler, mcp_config, &config.mcp.cors_origins, auth_token,
           );
           info!("MCP HTTP transport enabled at /mcp");
           Some(router)
       } else {
           None
       };

       #[cfg(not(feature = "mcp-server"))]
       let mcp_router: Option<axum::Router> = None;
       ```

       Pass `mcp_router` to `start_server`. Update the call site to include the new parameter.

    Ensure `cargo build` succeeds with and without the `mcp-server` feature. The gateway should start normally when MCP is disabled.
