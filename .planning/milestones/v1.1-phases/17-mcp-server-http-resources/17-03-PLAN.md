  <action>
    1. **Cargo.toml**: Add dependencies on `blufio-memory` and `blufio-core`:
       ```toml
       blufio-memory = { path = "../blufio-memory" }
       ```
       blufio-core is already a dependency.

    2. **resources.rs**: Create `crates/blufio-mcp-server/src/resources.rs`:

       Define a URI parsing enum and helper functions:
       ```rust
       pub enum ResourceRequest {
           MemoryById(String),
           MemorySearch { query: String, limit: usize },
           SessionList,
           SessionHistory(String),
       }

       /// Parse a blufio:// resource URI into a ResourceRequest.
       pub fn parse_resource_uri(uri: &str) -> Result<ResourceRequest, String> { ... }
       ```

       Parsing rules:
       - `blufio://memory/{id}` -> MemoryById (id is the path segment after "memory/", but NOT if it starts with "search")
       - `blufio://memory/search?q={query}&limit={limit}` -> MemorySearch (limit defaults to 10 if absent)
       - `blufio://sessions` -> SessionList
       - `blufio://sessions/{id}` -> SessionHistory

       Helper functions (each returns a `serde_json::Value`):
       ```rust
       pub async fn read_memory_by_id(store: &MemoryStore, id: &str) -> Result<serde_json::Value, String>
       pub async fn read_memory_search(store: &MemoryStore, query: &str, limit: usize) -> Result<serde_json::Value, String>
       pub async fn read_session_list(storage: &dyn StorageAdapter) -> Result<serde_json::Value, String>
       pub async fn read_session_history(storage: &dyn StorageAdapter, session_id: &str) -> Result<serde_json::Value, String>
       ```

       For `read_memory_by_id`: Call `store.get_by_id(id)`. If found, serialize to JSON object with fields: id, content, source (as string), confidence, status (as string), created_at, updated_at. Explicitly EXCLUDE the embedding field. If not found, return error "Memory not found: {id}".

       For `read_memory_search`: Call `store.search_bm25(query, limit)` to get (id, score) pairs. For each result, call `store.get_by_id(id)` to get the full memory. Return JSON array of objects (same fields as above, plus `relevance_score`).

       For `read_session_list`: Call `storage.list_sessions(None)`. Return JSON array of session summary objects: id, channel, created_at.

       For `read_session_history`: Call `storage.get_messages(session_id, None)`. Return JSON array of message objects: id, role, content, created_at.

    3. **handler.rs**: Extend `BlufioMcpHandler`:
       - Add fields: `memory_store: Option<Arc<MemoryStore>>`, `storage: Option<Arc<dyn StorageAdapter + Send + Sync>>`
       - Update constructor to accept these (use a builder pattern or additional parameters)
       - Add `with_resources(memory_store, storage)` method that sets these fields (allows stdio mode to skip them)
       - Update `get_info()` to advertise resources capability when memory_store or storage is Some
       - Implement `list_resources`: Return static resources (blufio://sessions if storage available)
       - Implement `list_resource_templates`: Return URI templates for memory and sessions
       - Implement `read_resource`: Parse URI via `resources::parse_resource_uri`, dispatch to appropriate helper function, return `ReadResourceResult` with `ResourceContents::text` containing the JSON string with `application/json` MIME type

    4. **lib.rs**: Add `pub mod resources;`

    Write extensive tests in resources.rs for URI parsing and in handler.rs for the ServerHandler methods. For handler tests, create mock MemoryStore (use a real in-memory SQLite connection via tokio_rusqlite) and mock StorageAdapter.

  <action>
    Update the MCP HTTP wiring section in serve.rs (added in Plan 01) to pass MemoryStore and StorageAdapter to BlufioMcpHandler.

    In the `#[cfg(feature = "mcp-server")]` block where the handler is created:
    ```rust
    let handler = blufio_mcp_server::BlufioMcpHandler::new(
        tool_registry.clone(), &config.mcp,
    );

    // Provide resource access for HTTP mode
    #[cfg(feature = "onnx")]
    let handler = if let Some(ref _mp) = memory_provider {
        // memory_store was created during initialize_memory -- need to expose it
        // Pass the memory_store Arc to the handler
        handler.with_resources(
            Some(memory_store.clone()),
            Some(storage.clone() as Arc<dyn StorageAdapter + Send + Sync>),
        )
    } else {
        handler.with_resources(None, Some(storage.clone() as Arc<dyn StorageAdapter + Send + Sync>))
    };

    #[cfg(not(feature = "onnx"))]
    let handler = handler.with_resources(
        None,
        Some(storage.clone() as Arc<dyn StorageAdapter + Send + Sync>),
    );
    ```

    NOTE: The `memory_store` Arc is currently created inside `initialize_memory` and not returned. You need to either:
    - Change `initialize_memory` to also return `Arc<MemoryStore>`, OR
    - Store `memory_store` in a variable accessible in the MCP section

    The cleanest approach: modify `initialize_memory` to return `(MemoryProvider, Arc<MemoryExtractor>, Arc<MemoryStore>)` and destructure at the call site. Then `memory_store` is available for MCP wiring.

    Also ensure the `mcp_server.rs` (stdio subcommand) continues to work by creating the handler with `with_resources(None, None)` or by keeping the existing constructor as-is (which doesn't include resource fields).
