3. Create `crates/blufio-agent/src/sdnotify.rs` with the following structure:

```rust
// SPDX-FileCopyrightText: 2026 Blufio Contributors
// SPDX-License-Identifier: MIT OR Apache-2.0

//! systemd sd_notify integration for process lifecycle signaling.
//!
//! Wraps the `sd-notify` crate to provide typed helpers for READY=1,
//! STOPPING=1, STATUS=, and WATCHDOG=1 notifications. All calls are
//! best-effort: failures are logged at debug level and never propagated.
//! On non-systemd platforms (macOS, Docker), all calls are silent no-ops
//! because the sd-notify crate returns Ok(()) when NOTIFY_SOCKET is absent.
```

Implement these four public functions plus one private helper:

- `pub fn notify_ready(status: &str)` -- Sends `[NotifyState::Ready, NotifyState::Status(status)]`. Logs at debug level. Errors are best-effort (debug log, no propagation). ALWAYS pass `false` to `sd_notify::notify()` (never `true` which unsets NOTIFY_SOCKET).

- `pub fn notify_stopping(status: &str)` -- Sends `[NotifyState::Stopping, NotifyState::Status(status)]`. Same error handling pattern.

- `pub fn notify_status(status: &str)` -- Sends `[NotifyState::Status(status)]`. Same error handling pattern.

- `fn notify_watchdog()` (private) -- Sends `[NotifyState::Watchdog]`. Same error handling pattern but no status message parameter.

- `pub fn spawn_watchdog(cancel: CancellationToken) -> Option<tokio::task::JoinHandle<()>>` -- Calls `sd_notify::watchdog_enabled(false, &mut usec)`. If not enabled or usec==0, returns None with debug log. Otherwise: computes interval as `Duration::from_micros(usec / 2)`, logs the watchdog config at info level (this is the one info-level log -- it is significant operational info), spawns a tokio task with `tokio::time::interval(interval)` + `select!` + `cancel.cancelled()` pattern (same as memory_monitor in serve.rs). First tick is skipped. Each tick calls `notify_watchdog()`. On cancel, debug logs "watchdog task shutting down".

Follow the exact patterns from 24-RESEARCH.md Code Examples section. All error handling is best-effort: log at debug, never panic, never propagate errors.

Unit tests to add in the same file:
- `test_notify_functions_no_panic`: Call `notify_ready("test")`, `notify_stopping("test")`, `notify_status("test")` -- verify they do not panic when NOTIFY_SOCKET is absent.
- `test_spawn_watchdog_returns_none_without_env`: Call `spawn_watchdog(CancellationToken::new())` and assert it returns `None` (since WATCHDOG_USEC is not set in test environment).
