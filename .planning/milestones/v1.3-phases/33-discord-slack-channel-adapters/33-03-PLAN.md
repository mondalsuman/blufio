    **2. Create `crates/blufio-slack/src/lib.rs`** — SlackChannel implementing ChannelAdapter:
    Follow the TelegramChannel pattern:
    - Struct with `config: SlackConfig`, `inbound_rx: Mutex<mpsc::Receiver<InboundMessage>>`, `inbound_tx: mpsc::Sender<InboundMessage>`, `bot_user_id: Option<String>`, `client: Option<Arc<SlackClient>>`, `bot_token: Option<SlackApiToken>`, `socket_handle: Option<JoinHandle<()>>`
    - `new(config: SlackConfig) -> Result<Self, BlufioError>` validates both bot_token and app_token are set and non-empty
    - `PluginAdapter` impl: name="slack", version=0.1.0, adapter_type=Channel
    - `health_check`: Call `auth.test` via the Slack Web API if connected
    - `ChannelAdapter` impl:
      - `capabilities`: supports_edit=true, supports_typing=true, supports_images=true, supports_documents=true, supports_voice=false, max_message_length=Some(40000), supports_embeds=true (Block Kit), supports_reactions=true, supports_threads=true
      - `connect`:
        1. Create `SlackClient` with `SlackClientHyperConnector`
        2. Call `auth.test` to discover bot user ID (needed for @mention filtering)
        3. Create Socket Mode config and app token
        4. Create event handler (`SlackEventHandler`) that receives messages and forwards to inbound_tx
        5. Register Socket Mode client and start in background task
        6. Store handles
      - `send`: Use `chat.postMessage` API via the Slack client. Extract channel_id from metadata. Convert text with `markdown_to_mrkdwn()`. Return MessageId.
      - `receive`: Receive from inbound_rx (identical to Telegram)
      - `edit_message`: Use `chat.update` API
      - `send_typing`: Slack does not have a direct typing indicator API like Telegram — this is a no-op (Slack shows typing automatically for Socket Mode responses within a short window)
    - Add unit tests: new requires both tokens, capabilities are correct, plugin adapter metadata

    **3. Create `crates/blufio-slack/src/handler.rs`** — Socket Mode event handler:
    - `SlackEventHandler` struct with `inbound_tx`, `bot_user_id`, `allowed_users`
    - Implement `SlackSocketModeClientListener` (or equivalent trait from slack-morphism 2.18):
      - Parse incoming Socket Mode envelope JSON
      - Handle `event_callback` type for `message` events:
        - Skip bot messages (check `subtype` or `bot_id`)
        - For DMs: always respond
        - For channels: only respond when `<@bot_user_id>` appears in text
        - Check authorization against allowed_users
        - Strip `<@bot_user_id>` from text
        - Convert to InboundMessage with metadata containing channel_id and team_id
        - Send to inbound_tx
      - Handle `slash_commands` type: delegate to `commands::handle_slash_command()`
      - **CRITICAL**: Acknowledge every envelope IMMEDIATELY (within 3 seconds) before processing. Process asynchronously after acknowledgment. Slack disconnects on unacknowledged envelopes.
    - `should_respond(text, channel_type, bot_user_id) -> bool`: DMs always, channels only when @mentioned
    - `is_authorized(user_id, allowed_users) -> bool`: Same pattern as Discord/Telegram
    - `strip_mention(text, bot_user_id) -> String`: Remove `<@BOT_ID>` from text
    - Add unit tests for should_respond, is_authorized, strip_mention

    **4. Create `crates/blufio-slack/src/markdown.rs`** — Markdown to mrkdwn conversion:
    Per the RESEARCH.md guidance, Slack's mrkdwn differs significantly from standard markdown:
    - Bold: `**text**` -> `*text*`
    - Italic: `*text*` -> `_text_` (careful ordering — convert bold first)
    - Strikethrough: `~~text~~` -> `~text~`
    - Links: `[text](url)` -> `<url|text>`
    - Headers: `# text` -> `*text*` (bold, since mrkdwn has no headers)
    - Code blocks: triple backtick blocks pass through unchanged
    - Inline code: single backtick passes through unchanged

    Use regex for conversions. Be careful with ordering:
    1. Protect code blocks/inline code (replace with placeholders)
    2. Convert bold (** -> *)
    3. Convert italic (* -> _) — only remaining single asterisks
    4. Convert strikethrough
    5. Convert links
    6. Convert headers
    7. Restore code blocks/inline code

    Add comprehensive unit tests for each conversion and edge cases (nested formatting, multiple links, code blocks with special characters).

    **5. Create `crates/blufio-slack/src/blocks.rs`** — Block Kit message builders:
    - `build_status_blocks(status: &str, model: &str) -> serde_json::Value`:
      Build a Block Kit message for bot status with Header, Section (fields for Status, Model, Uptime), Divider
    - `build_help_blocks() -> serde_json::Value`:
      Build a Block Kit message for help text with Header, multiple Sections explaining available commands
    - `build_error_blocks(error: &str) -> serde_json::Value`:
      Build a Block Kit message for errors with a red-colored context section
    - Use serde_json::json! macros for Block Kit structure (simpler than the typed slack-morphism Block Kit types for our use case — fewer compile-time constraints but cleaner code)
    - Add unit tests verifying block structure contains expected elements

    **6. Create `crates/blufio-slack/src/commands.rs`** — Slash command handling:
    - `handle_slash_command(text: &str, user_id: &str, channel_id: &str, inbound_tx: &mpsc::Sender<InboundMessage>) -> serde_json::Value`:
      Parse the text after `/blufio`:
      - `status` -> Return status blocks (from blocks.rs)
      - `help` -> Return help blocks
      - `<message>` or `chat <message>` -> Convert to InboundMessage, send to inbound_tx, return acknowledgment text
      - Empty or unrecognized -> Return help blocks
    - Returns JSON for the response that should be sent back via Socket Mode acknowledgment or `chat.postMessage`
    - Add unit tests for command routing

    **7. Create `crates/blufio-slack/src/streaming.rs`** — Slack streaming:
    - Implement `StreamingEditorOps` for `SlackStreamOps`:
      - `send_initial`: Post message via `chat.postMessage`, return ts (timestamp-based message ID)
      - `edit_message`: Update via `chat.update` using channel + ts
      - `max_message_length`: 40000 (Slack's limit)
      - `throttle_interval`: 3000ms (conservative — Slack `chat.update` is Tier 3 at 50+/min)
    - Split threshold: ~4000 chars (Slack's limit is 40000, but individual blocks have 3000-char limits for text sections — use a practical threshold)
    - Add a `SlackStreamingEditor` wrapper that creates `StreamingBuffer` with threshold 4000 and `SlackStreamOps`
    - No typing indicator function needed (Slack handles this automatically for Socket Mode)

    #[cfg(feature = "slack")]
    {
        if config.slack.bot_token.is_some() && config.slack.app_token.is_some() {
            let slack = SlackChannel::new(config.slack.clone()).map_err(|e| {
                error!(error = %e, "failed to initialize Slack channel");
                eprintln!(
                    "error: Slack bot_token and app_token required for Socket Mode. \

            // Redact Slack tokens in logs.

                    &vault_values,
