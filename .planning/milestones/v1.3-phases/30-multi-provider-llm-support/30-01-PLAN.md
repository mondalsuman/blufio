pub trait ProviderAdapter: PluginAdapter {

**types.rs** -- OpenAI wire format types:
- `ChatRequest`: model, messages (Vec<ChatMessage>), max_completion_tokens, stream, tools (Option<Vec<OpenAITool>>), response_format
- `ChatMessage`: role, content (ChatContent enum: String or Vec<ContentPart>), tool_calls, tool_call_id
- `ContentPart`: text or image_url variant (for vision)
- `OpenAITool`: type="function", function: FunctionDef (name, description, parameters)
- `ChatResponse`: id, choices (Vec<Choice>), model, usage
- `Choice`: message (ChatMessage), finish_reason, index
- `SseChunk`: id, choices (Vec<SseDelta>), model, usage
- `SseDelta`: delta (DeltaMessage), finish_reason
- `DeltaMessage`: role, content, tool_calls (Vec<DeltaToolCall>)
- `DeltaToolCall`: index, id, function (DeltaFunction: name, arguments)

All types derive Serialize/Deserialize as needed. Use `#[serde(skip_serializing_if = "Option::is_none")]` for optional fields.

**client.rs** -- HTTP client:
- `OpenAIClient` struct with reqwest::Client, default_model, base_url, max_retries
- Constructor: takes api_key, base_url, model, optional SecurityConfig
- `complete_chat()`: POST to `{base_url}/chat/completions`, parse `ChatResponse`
- `stream_chat()`: POST with `stream: true`, return SSE event stream
- Retry logic: same pattern as AnthropicClient (429, 500, 503 retried once after 1s)
- Headers: `Authorization: Bearer {api_key}`, `Content-Type: application/json`
- `#[cfg(test)] with_base_url()` for wiremock testing

**sse.rs** -- SSE stream parser:
- `parse_openai_sse_stream()`: uses eventsource-stream, maps SSE events to typed chunks
- Handle `data: [DONE]` as stream terminator
- Parse each `data:` line as `SseChunk`
- Map to OpenAI-specific stream event types

Write comprehensive wiremock tests for the client (success, retry, error cases) and serde tests for all types.

   - Construct OpenAIClient with base_url from config (default: `https://api.openai.com/v1`)

4. Implement `ProviderAdapter`:
   - `complete()`: Convert ProviderRequest -> ChatRequest, call client, map ChatResponse -> ProviderResponse
   - `stream()`: Convert ProviderRequest -> ChatRequest, call client stream, map SSE chunks -> ProviderStreamChunk stream

5. `to_chat_request()` conversion:
   - `system_prompt` -> prepend `ChatMessage { role: "system", content: text }` to messages
   - `messages` -> map ProviderMessage to ChatMessage, handling ContentBlock variants:
     - Text -> plain text content or text content part
     - Image -> `{"type": "image_url", "image_url": {"url": "data:{mime};base64,{data}"}}` content part

     - ToolResult -> `{"role": "tool", "tool_call_id": "...", "content": "..."}`

   - `max_tokens` -> `max_completion_tokens`

   - Track tool_calls by index: HashMap<usize, (id, name, accumulated_args)>
   - Text delta -> ProviderStreamChunk with text
   - Tool call delta -> accumulate id/name/args
   - finish_reason -> emit accumulated tool_use chunks, then stop
   - "tool_calls" finish_reason -> stop_reason "tool_use"
   - "stop" finish_reason -> stop_reason "end_turn"
   - Usage -> map prompt_tokens to input_tokens, completion_tokens to output_tokens
   - `[DONE]` -> MessageStop event

Write tests using `#[cfg(test)] with_client()` pattern for unit tests and `#[cfg(test)] with_base_url()` for integration tests.
