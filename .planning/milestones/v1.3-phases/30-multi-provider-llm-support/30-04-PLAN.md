**types.rs** -- Gemini wire format:
- `GenerateContentRequest`: contents (Vec<GeminiContent>), system_instruction (Option<GeminiSystemInstruction>), tools (Option<Vec<GeminiTool>>), generation_config (Option<GenerationConfig>)
- `GeminiContent`: role (String), parts (Vec<GeminiPart>)
- `GeminiPart` (enum): text (TextPart), function_call (FunctionCallPart), function_response (FunctionResponsePart), inline_data (InlineDataPart)
- `TextPart`: { text: String }
- `FunctionCallPart`: { function_call: FunctionCall } where FunctionCall { name, args: serde_json::Value }
- `FunctionResponsePart`: { function_response: FunctionResponse } where FunctionResponse { name, response: serde_json::Value }
- `InlineDataPart`: { inline_data: InlineData } where InlineData { mime_type, data (base64) }
- `GeminiSystemInstruction`: parts (Vec<GeminiPart>)
- `GeminiTool`: function_declarations (Vec<FunctionDeclaration>)
- `FunctionDeclaration`: name, description, parameters (serde_json::Value -- JSON Schema)
- `GenerationConfig`: max_output_tokens (Option<u32>), temperature (Option<f32>)
- `GenerateContentResponse`: candidates (Vec<Candidate>), usage_metadata (Option<UsageMetadata>)
- `Candidate`: content (GeminiContent), finish_reason (Option<String>)
- `UsageMetadata`: prompt_token_count (u32), candidates_token_count (u32), total_token_count (u32)

Use `#[serde(rename_all = "camelCase")]` for Gemini's camelCase JSON convention.

**stream.rs** -- Streaming response parser:
- Gemini streamGenerateContent returns chunked JSON: a stream of GenerateContentResponse objects (sometimes as JSON array elements, sometimes as newline-delimited).
- `parse_gemini_stream(response: reqwest::Response) -> Pin<Box<dyn Stream<Item = Result<GenerateContentResponse, BlufioError>> + Send>>`
- Read bytes, accumulate into buffer, try to parse complete JSON objects
- Handle: single objects per chunk, partial objects across chunks
- Use a simple JSON depth counter: track `{` depth, emit when depth returns to 0 after an opening `{`

**client.rs** -- HTTP client:
- `GeminiClient`: reqwest::Client, api_key, base_url (default: "https://generativelanguage.googleapis.com/v1beta"), default_model
- Constructor: takes api_key, model, optional SecurityConfig. API key is NOT set as a header -- appended as `?key={api_key}` query parameter.
- `generate_content()`: POST to `{base_url}/models/{model}:generateContent?key={api_key}`
- `stream_generate_content()`: POST to `{base_url}/models/{model}:streamGenerateContent?key={api_key}`, return stream
- Retry logic: same pattern (429, 500, 503)
- SSRF-safe resolver + TLS 1.2+ from SecurityConfig
- Content-Type: application/json header
- `#[cfg(test)] with_base_url()` for wiremock testing

Write comprehensive wiremock tests for the client and serde tests for all types.

2. `new(config: &BlufioConfig)`:
   - Resolve API key: `config.providers.gemini.api_key` first, then `GEMINI_API_KEY` env var
   - Load system prompt: file > inline > default pattern
   - Construct GeminiClient

3. Implement `PluginAdapter`:
   - `name()` -> "gemini"
   - `version()` -> 0.1.0
   - `adapter_type()` -> AdapterType::Provider
   - `health_check()` -> Healthy
   - `shutdown()` -> Ok(())

4. Implement `ProviderAdapter`:
   - `complete()`: Convert ProviderRequest -> GenerateContentRequest, call client, map response
   - `stream()`: Convert ProviderRequest -> GenerateContentRequest, call client stream, map chunks

5. `to_gemini_request()` conversion -- KEY DIFFERENCES from OpenAI format:
   - `system_prompt` -> `system_instruction: GeminiSystemInstruction { parts: [TextPart { text }] }` (NOT in contents array)
   - Role mapping: "user" stays "user", "assistant" -> "model"
   - ContentBlock::Text -> `GeminiPart::Text(TextPart { text })`
   - ContentBlock::Image -> `GeminiPart::InlineData(InlineDataPart { inline_data: InlineData { mime_type, data } })`
   - ContentBlock::ToolUse -> `GeminiPart::FunctionCall(FunctionCallPart { function_call: FunctionCall { name, args: input } })` in "model" role content
   - ContentBlock::ToolResult -> `GeminiPart::FunctionResponse(FunctionResponsePart { function_response: FunctionResponse { name: tool_use_id, response: json!({"result": content}) } })` in "user" role content. Note: Gemini uses `name` to identify which function the response is for (matches the function call name), and wraps the result in a response object.
   - `tools` -> `GeminiTool { function_declarations: defs.map(|td| FunctionDeclaration { name: td.name, description: td.description, parameters: td.input_schema }) }`
   - `max_tokens` -> `generation_config.max_output_tokens`

6. Response mapping:
   - Extract text from candidates[0].content.parts where part is TextPart, concatenate
   - Extract function calls from candidates[0].content.parts where part is FunctionCallPart -> map to ToolUseData
   - finishReason: "STOP" -> "end_turn", "MAX_TOKENS" -> "max_tokens", "SAFETY" -> "content_filter"
   - If any functionCall parts present: stop_reason = "tool_use"
   - Generate UUID for response ID (Gemini doesn't provide one in the same format)
   - usageMetadata: promptTokenCount -> input_tokens, candidatesTokenCount -> output_tokens, cache fields = 0

7. Stream mapping:
   - Each streamed GenerateContentResponse chunk maps to ProviderStreamChunk:
     - Text parts -> ContentBlockDelta with text
     - FunctionCall parts -> ContentBlockStop with tool_use data (Gemini sends complete calls, not partials)
     - finishReason present -> MessageDelta with stop_reason + usage
     - Last chunk -> MessageStop
   - Synthesize MessageStart from first chunk

Write comprehensive unit tests.
