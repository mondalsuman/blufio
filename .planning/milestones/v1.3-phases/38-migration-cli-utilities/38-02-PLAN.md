2. Create crates/blufio/src/bench.rs:

**System Info Collection:**
- Function `collect_system_info() -> serde_json::Value` using `sysinfo` crate:
  - CPU model, core count
  - Total RAM
  - OS name and version
  - Blufio version (from `env!("CARGO_PKG_VERSION")`)

**Peak RSS Measurement:**
- Function `get_peak_rss() -> Option<u64>` using platform-specific APIs:
  - macOS: `getrusage(RUSAGE_SELF)` via libc, read `ru_maxrss` (bytes on macOS)
  - Linux: read `/proc/self/status` and parse `VmHWM` line
  - Fallback: return None

**Benchmark Registry:**
- Struct `BenchmarkResult { name: String, median: Duration, min: Duration, max: Duration, peak_rss: Option<u64>, iterations: u32 }`
- Enum `BenchmarkKind { Startup, ContextAssembly, Wasm, Sqlite }` with `Display` and `FromStr`
- Function `run_benchmark(kind: BenchmarkKind, iterations: u32) -> Result<BenchmarkResult, BlufioError>`:
  - 1 warm-up run (discarded)
  - N measured runs (default 3)
  - Collect timings, compute median/min/max
  - Measure peak RSS after benchmark completes

**Individual Benchmarks:**
- `bench_startup()` -- measure time to load config and initialize components (not full server start)
- `bench_context_assembly()` -- create synthetic deterministic messages, measure context window assembly time using blufio-context patterns
- `bench_wasm()` -- measure WASM module load time using blufio-skill patterns (load a minimal test .wasm if available, or skip with warning)
- `bench_sqlite()` -- measure batch insert + query operations on temporary SQLite database

**Main Entry Point:**
- Function `pub async fn run_bench(only: Option<Vec<String>>, json: bool, compare: bool, baseline: bool, iterations: Option<u32>, ci: bool, threshold: Option<f64>) -> Result<(), BlufioError>`:
  - Parse `--only` filter: if provided, run only matching benchmarks
  - Print system info header (CPU, RAM, OS, Blufio version)
  - Run each selected benchmark
  - Save results to bench_results table via SQLite
  - If `--compare`: load previous results, show delta (faster/slower %)
  - If `--baseline`: mark current results as baseline (set is_baseline=1, clear previous baselines)
  - If `--ci`: compare against baseline, exit non-zero if any benchmark exceeds baseline by >threshold% (default 20%)
  - Table output by default (aligned columns: Benchmark, Median, Min, Max, Peak RSS, Delta)
  - `--json` flag: serialize results as JSON array

3. Create crates/blufio/src/uninstall.rs:

**Uninstall Logic:**
- Function `pub async fn run_uninstall(purge: bool) -> Result<(), BlufioError>`:
  - Step 1: Detect and stop running Blufio processes:
    - Check systemd: `systemctl is-active blufio` / `systemctl stop blufio`
    - Check launchd: `launchctl list | grep blufio` / `launchctl unload`
    - Check PID file or process list as fallback
    - If cannot stop, refuse with error message
  - Step 2: Always remove:
    - Binary (`which blufio` or `std::env::current_exe()`)
    - Systemd service file (`/etc/systemd/system/blufio.service`)
    - Launchd plist (`~/Library/LaunchAgents/io.bluf.blufio.plist`)
    - Shell completions (bash: `/etc/bash_completion.d/blufio`, zsh: `~/.zfunc/_blufio`, fish: `~/.config/fish/completions/blufio.fish`)
  - Step 3: Handle data removal:
    - If `--purge`: auto-backup to timestamped archive (`~/.local/share/blufio/backup-{timestamp}.tar.gz`) using backup.rs pattern, then remove data directory
    - If interactive (no --purge): prompt user "Remove data directory? (y/N)" -- use stdin reading
    - Print backup path as last-chance recovery info
  - Step 4: Print summary of what was removed
