2. Create crates/blufio/src/migrate.rs implementing the full migration module:

**OpenClaw Detection:**
- Function `detect_openclaw_dir(override_path: Option<&str>) -> Result<PathBuf, BlufioError>` that checks:
  - `--data-dir` override first
  - `$OPENCLAW_HOME` env var
  - `~/.openclaw` default path
  - Returns `BlufioError::Migration` with helpful message if not found

**OpenClaw Data Parsing:**
- Struct `OpenClawData` representing parsed OpenClaw directory contents:
  - `config: serde_json::Value` (parsed JSON config)
  - `sessions: Vec<OpenClawSession>` (parsed session history)
  - `cost_records: Vec<OpenClawCostRecord>` (parsed cost data)
  - `personality_files: Vec<PathBuf>` (markdown files found)
  - `secrets: Vec<(String, String)>` (detected API keys/tokens from config)
- Function `parse_openclaw_dir(path: &Path) -> Result<OpenClawData, BlufioError>` that reads and parses the OpenClaw data directory. Parse JSON config, scan for SQLite database or JSON session files, collect markdown files from workspace/personality directories, detect secret patterns (keys matching `*api_key*`, `*token*`, `*secret*`).

**Preview (Dry Run):**
- Function `run_migrate_preview(data_dir: Option<&str>, json: bool) -> Result<(), BlufioError>`:
  - Detects and parses OpenClaw directory
  - Categorizes items into: Will Import (sessions, cost records, known personality files), Needs Manual Attention (sessions with unsupported features, unknown config fields), Cannot Import (binary data, incompatible formats)
  - Shows estimated cost comparison: maps OpenClaw cost records to Blufio provider pricing using `blufio_cost::pricing::get_pricing()`
  - Table output by default; JSON with `--json` flag
  - Never modifies any data

**Import (Full Migration):**
- Function `run_migrate(data_dir: Option<&str>, json: bool) -> Result<(), BlufioError>`:
  - Calls detect + parse
  - Opens Blufio database and vault
  - Creates indicatif progress bar (MultiProgress with bars for sessions, costs, files, secrets)
  - For each item, check migration_log for existing entry (idempotent):
    - `SELECT 1 FROM migration_log WHERE source = 'openclaw' AND item_type = ? AND source_id = ?`
    - If exists, skip with progress increment
  - Import sessions: map OpenClaw session format to Blufio Session struct, call `create_session()` and `insert_message()` for each message. Tag sessions with metadata: `source: openclaw`, `unmapped_features: [...]` for unsupported features.
  - Import cost records: map to CostRecord, call `ledger.record()`
  - Import personality files: copy known files (SOUL.md -> personality dir), unknown files to `~/.local/share/blufio/import/openclaw/`
  - Auto-vault secrets: call `vault.store_secret()` for each detected secret
  - Record each import in migration_log
  - Print categorized summary at end (imported X sessions, Y cost records, Z files, W secrets)

**Config Translate:**
- Function `run_config_translate(input: &str, output: Option<&str>) -> Result<(), BlufioError>`:
  - Read OpenClaw JSON config file
  - Map known fields to BlufioConfig struct fields
  - For unmappable fields, collect as comments: `# UNMAPPED: field_name = value`
  - Serialize BlufioConfig to TOML using `toml::to_string_pretty()`
  - Append unmapped fields as comments at end
  - Print to stdout or write to `--output` path
  - Print summary: N fields mapped, M fields unmapped
