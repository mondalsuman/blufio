  <name>Task 1: Create webhook types, store, and delivery engine</name>
  <files>

  <behavior>
    - WebhookStore::create() stores a webhook with auto-generated HMAC secret and returns it (secret only visible at creation)
    - WebhookStore::list() returns all active webhooks without exposing secrets
    - WebhookStore::get() retrieves a webhook by ID with its secret (for delivery engine internal use)
    - WebhookStore::delete() removes a webhook by ID
    - WebhookStore::list_for_event() returns webhooks whose events filter includes the given event type
    - WebhookStore::insert_dead_letter() stores a failed delivery in the dead letter queue
    - sign_payload(secret, payload) produces correct HMAC-SHA256 hex signature
    - deliver_with_retry attempts up to 5 times with backoff: 1s, 5s, 25s, 120s, 600s
    - deliver_with_retry inserts into dead letter on final failure
    - Webhook event types: chat.completed, response.completed, tool.invoked, batch.completed
  </behavior>
  <action>
    1. Create `crates/blufio-gateway/src/webhooks/mod.rs`:
       - `Webhook` struct: id (String), url (String), secret (String -- only populated when needed), events (Vec<String>), active (bool), created_at (String), updated_at (String)
       - `WebhookListItem` struct: id, url, events, active, created_at (NO secret)
       - `CreateWebhookRequest` struct: url (String), events (Vec<String>)
       - `CreateWebhookResponse` struct: id, url, secret (shown once), events, created_at
       - `DeadLetterEntry` struct: id, webhook_id, event_type, payload, last_attempt_at, attempt_count, last_error, created_at
       - `WebhookPayload` struct: event_type (String), timestamp (String), data (serde_json::Value)

    2. Create `crates/blufio-gateway/src/webhooks/store.rs`:
       - `WebhookStore` wrapping `tokio_rusqlite::Connection`
       - `create(req: &CreateWebhookRequest) -> Result<CreateWebhookResponse>` -- generates 32-byte random hex secret, stores webhook
       - `list() -> Result<Vec<WebhookListItem>>` -- SELECT without secret column
       - `get(id: &str) -> Result<Option<Webhook>>` -- SELECT with secret (internal use)
       - `delete(id: &str) -> Result<()>` -- DELETE
       - `list_for_event(event_type: &str) -> Result<Vec<Webhook>>` -- SELECT WHERE events JSON contains event_type AND active = 1
       - `insert_dead_letter(webhook_id: &str, event_type: &str, payload: &str, error: &str) -> Result<()>`

    3. Create `crates/blufio-gateway/src/webhooks/delivery.rs`:
       - `sign_payload(secret: &[u8], payload: &[u8]) -> String` using hmac::Hmac<sha2::Sha256>
       - `deliver_single(client: &reqwest::Client, webhook: &Webhook, payload: &WebhookPayload) -> Result<u16, String>`:
         - Serialize payload to JSON
         - Sign with webhook secret
         - POST to webhook.url with headers: Content-Type: application/json, X-Webhook-Signature: {signature}
         - Timeout: 10 seconds per attempt
         - Returns Ok(status_code) or Err(error_message)
       - `deliver_with_retry(client: &reqwest::Client, webhook: &Webhook, payload: &WebhookPayload, store: &WebhookStore, bus: &EventBus) -> bool`:
         - Retry delays: [1, 5, 25, 120, 600] seconds
         - On each attempt: call deliver_single, publish WebhookEvent::DeliveryAttempted to bus
         - If 2xx: return true
         - If all 5 attempts fail: call store.insert_dead_letter(), return false
       - `run_webhook_delivery(bus: Arc<EventBus>, store: Arc<WebhookStore>, client: reqwest::Client)`:
         - Call bus.subscribe_reliable(256)
         - Loop: receive BusEvent, map to event_type:
           - BusEvent::Channel(ChannelEvent::MessageSent { .. }) => "chat.completed"
           - BusEvent::Skill(SkillEvent::Completed { .. }) => "tool.invoked"
           - BusEvent::Batch(BatchEvent::Completed { .. }) => "batch.completed"
           - Skip all others
         - For each matched event: list_for_event(event_type), spawn tokio::spawn for each delivery
         - NOTE: The "response.completed" event will be added when Responses API events are published to the bus (already handled by existing Responses endpoint code)

    4. Add `pub mod webhooks;` to `crates/blufio-gateway/src/lib.rs`.
