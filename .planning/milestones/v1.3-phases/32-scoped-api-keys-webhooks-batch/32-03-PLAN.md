  <behavior>
    - BatchStore::create_batch() creates a batch record and N item records, returns batch_id
    - BatchStore::get_batch() returns batch status with all items
    - BatchStore::update_item() marks an item as completed or failed with response/error
    - BatchStore::update_batch_status() updates batch status when all items complete
    - BatchProcessor::process() executes items in parallel using Semaphore(concurrency_limit)
    - BatchProcessor::process() publishes BatchEvent::Submitted at start and BatchEvent::Completed at end
    - Batch items inherit caller's AuthContext scopes (checked before execution)
    - Max batch size enforced (default 100, configurable)
  </behavior>
  <action>
    1. Create `crates/blufio-gateway/src/batch/mod.rs`:
       - `BatchRequest` struct: items (Vec<ChatCompletionRequest>)
       - `BatchResponse` struct: id (String), status (String: "processing"|"completed"|"failed"), total_items (usize), completed_items (usize), failed_items (usize), items (Option<Vec<BatchItemResult>>), created_at (String), completed_at (Option<String>)
       - `BatchItemResult` struct: index (usize), status (String: "completed"|"failed"), response (Option<serde_json::Value>), error (Option<String>)
       - `BatchSubmitResponse` struct: id (String), status ("processing"), total_items (usize), created_at (String) -- returned immediately on POST
       - Derive Serialize, Deserialize for all types

    2. Create `crates/blufio-gateway/src/batch/store.rs`:
       - `BatchStore` wrapping `tokio_rusqlite::Connection`
       - `create_batch(items: &[ChatCompletionRequest], api_key_id: Option<&str>) -> Result<String>`:
         - Generate UUID batch_id
         - INSERT INTO batches with status="processing", total_items=items.len()
         - INSERT each item into batch_items with item_index, status="pending", request=serde_json::to_string(item)
         - Return batch_id
       - `get_batch(batch_id: &str) -> Result<Option<BatchResponse>>`:
         - SELECT from batches + batch_items
         - Map each batch_item to BatchItemResult
         - Return combined BatchResponse
       - `update_item(batch_id: &str, item_index: usize, status: &str, response: Option<&str>, error: Option<&str>) -> Result<()>`:
         - UPDATE batch_items SET status, response/error, completed_at
       - `finalize_batch(batch_id: &str) -> Result<(usize, usize)>`:
         - Count completed and failed items
         - UPDATE batches SET status based on results, completed_at
         - Return (success_count, error_count)

    3. Create `crates/blufio-gateway/src/batch/processor.rs`:
       - `process_batch(batch_id: String, items: Vec<ChatCompletionRequest>, providers: Arc<dyn ProviderRegistry>, store: Arc<BatchStore>, bus: Option<Arc<EventBus>>, auth_ctx: AuthContext, concurrency: usize)`:
         - Publish BatchEvent::Submitted to bus
         - Create Semaphore with concurrency limit
         - For each (index, request) in items.iter().enumerate():
           - Acquire semaphore permit
           - tokio::spawn a task that:
             a. Checks auth_ctx.has_scope("chat.completions") -- if not, mark item as failed with "scope denied"
             b. Resolves provider from model name
             c. Calls provider.chat_completion(request) (non-streaming)
             d. On success: store.update_item(batch_id, index, "completed", Some(response_json), None)
             e. On error: store.update_item(batch_id, index, "failed", None, Some(error_msg))
         - Wait for all tasks with JoinSet or join_all
         - Call store.finalize_batch(batch_id) to get (success_count, error_count)
         - Publish BatchEvent::Completed to bus
       - This function runs as a background task (tokio::spawn) -- the handler returns immediately with batch_id

    4. Add `pub mod batch;` to `crates/blufio-gateway/src/lib.rs`.

    Concurrency control:
    ```rust
    use tokio::sync::Semaphore;
    let semaphore = Arc::new(Semaphore::new(concurrency));
    let mut handles = Vec::new();
    for (index, request) in items.into_iter().enumerate() {
        let permit = semaphore.clone().acquire_owned().await.unwrap();
        let handle = tokio::spawn(async move {
            let _permit = permit; // held until task completes
            // ... execute request ...
        });
        handles.push(handle);
    }
    for handle in handles {
        let _ = handle.await;
    }
    ```
