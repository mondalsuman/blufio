Create the `blufio-irc` crate.

**Cargo.toml:** Dependencies: blufio-core, blufio-config (path), async-trait, tokio (workspace), serde, serde_json, tracing, semver, uuid (workspace), `irc = { version = "1", default-features = false, features = ["tls-rust"] }`, base64 = "0.22".

**splitter.rs:** Implement message splitting for IRC PRIVMSG:
- `pub fn split_message(target: &str, nick: &str, message: &str, max_line_bytes: usize) -> Vec<String>`: Calculate IRC protocol overhead: `:nick!~nick@host PRIVMSG target :` is approximately `nick.len()*2 + 15 + target.len() + 4` bytes plus `\r\n` (2 bytes). The total line limit is 512 bytes (RFC 2812). Available payload = 512 - overhead - 2.
- Split message at word boundaries (spaces) to fit within available payload bytes. If a single word exceeds the limit, split at byte boundary.
- Return Vec of message chunks.
- Add unit tests: short message returns single chunk, long message splits at word boundary, handles no-space strings, handles empty input.

**flood.rs:** Implement rate-limited message sending:
- `pub struct FloodProtectedSender` with `queue_tx: mpsc::Sender<(String, String)>` (target, message).
- `pub fn new(client: Arc<irc::client::Client>, rate_limit_ms: u64, nick: String) -> Self`: Spawn a tokio task that uses `tokio::time::interval(Duration::from_millis(rate_limit_ms))` to dequeue and send messages. For each (target, message), call `splitter::split_message()` to split into chunks, then send each chunk via `client.send_privmsg(&target, &chunk)` with a rate limit tick between each chunk.
- `pub async fn send(&self, target: &str, message: &str) -> Result<(), BlufioError>`: Queue the message.
- Add unit tests: verify queue accepts messages (mock or channel-based test).

**sasl.rs:** Implement SASL PLAIN authentication:
- `pub async fn authenticate_sasl(client: &irc::client::Client, nickname: &str, password: &str) -> Result<(), BlufioError>`:
  1. Send `CAP REQ :sasl` via `client.send(Command::CAP(None, "REQ".into(), None, Some("sasl".into())))`.
  2. Note: The actual CAP ACK handling happens in the message stream. This function sends the initial request. The stream processing in lib.rs handles the CAP ACK -> AUTHENTICATE PLAIN -> base64 credentials -> RPL_SASLSUCCESS/ERR_SASLFAIL flow.
- `pub fn encode_sasl_plain(nickname: &str, password: &str) -> String`: Encode `\0nickname\0password` as base64. Use `base64::engine::general_purpose::STANDARD`.
- `pub async fn send_authenticate(client: &irc::client::Client, encoded: &str) -> Result<(), BlufioError>`: Send `AUTHENTICATE <encoded>` as raw command.
- `pub async fn finish_cap(client: &irc::client::Client) -> Result<(), BlufioError>`: Send `CAP END`.

**lib.rs:** Implement `IrcChannel` struct:
- Fields: config (IrcConfig), inbound_rx (Mutex<mpsc::Receiver<InboundMessage>>), inbound_tx (mpsc::Sender<InboundMessage>), client_handle (Option<Arc<irc::client::Client>>), flood_sender (Option<FloodProtectedSender>), receive_handle (Option<JoinHandle<()>>).
- `pub fn new(config: IrcConfig) -> Result<Self, BlufioError>`: Validate server is set, nickname is set. Create mpsc channel(100).
- PluginAdapter: name "irc", version 0.1.0, adapter_type Channel, health_check checks if client is connected (return Healthy if client exists, Unhealthy otherwise), shutdown sends QUIT message.
- ChannelAdapter capabilities: supports_edit: false, supports_typing: false, supports_images: false, supports_documents: false, supports_voice: false, max_message_length: Some(450) (conservative for PRIVMSG payloads), supports_embeds: false, supports_reactions: false, supports_threads: false.
- `connect()`:
  1. Build `irc::client::prelude::Config` from IrcConfig: server, port (default 6697 for TLS, 6667 without), nickname, channels, use_tls (from config.tls, default true). If auth_method == "nickserv", set nick_password and should_ghost.
  2. Create `irc::client::Client::from_config(irc_config).await?`.
  3. If auth_method == "sasl": call `sasl::authenticate_sasl()` before `client.identify()`.
  4. Call `client.identify()?` to complete connection registration.
  5. Create FloodProtectedSender with client clone and rate_limit_ms from config.
  6. Spawn background task: get client.stream()?, iterate messages. For each message:
     - Handle SASL flow messages (CAP ACK -> AUTHENTICATE -> RPL_SASLSUCCESS 903 / ERR_SASLFAIL 904).
     - Handle PRIVMSG: extract target (channel or nick), sender nick, text. If target is bot's nick (DM), always process. If target is a channel, check for @mention of bot nick in text. Skip if sender is in ignored list or not in allowed_users (if non-empty). Create InboundMessage with id=uuid, channel="irc", sender_id=sender_nick, content=text (stripped of @mention), metadata=json({"chat_id": target, "is_dm": bool}). Send via inbound_tx.
     - Handle PING (auto-handled by irc crate, but log it).
     - Handle numeric errors (log warnings).
- `send()`: Extract target from metadata chat_id. Use flood_sender.send(target, content).
- `receive()`: Read from inbound_rx.
- `edit_message()` / `send_typing()`: Return Ok(()) (IRC does not support these).

                password.clone(),
            );
        }
    } else {
        info!("irc channel skipped (no server configured)");
    }
}
