Create the `blufio-signal` crate.

**Cargo.toml:** Dependencies: blufio-core, blufio-config (path), async-trait, tokio (workspace, features: net, io-util, sync, time, macros), serde, serde_json, tracing, semver, uuid (workspace).

**types.rs:** Define signal-cli JSON-RPC message types:
- `SignalNotification { jsonrpc: String, method: String, params: SignalParams }`
- `SignalParams { envelope: SignalEnvelope }`
- `SignalEnvelope { source: Option<String>, source_number: Option<String>, source_name: Option<String>, timestamp: Option<u64>, data_message: Option<SignalDataMessage> }` (use `#[serde(rename_all = "camelCase")]`)
- `SignalDataMessage { message: Option<String>, timestamp: Option<u64>, group_info: Option<SignalGroupInfo> }` (use `#[serde(rename_all = "camelCase")]`)
- `SignalGroupInfo { group_id: String }` (use `#[serde(rename_all = "camelCase")]`)
- `SignalJsonRpcRequest { jsonrpc: String, method: String, params: serde_json::Value, id: String }`
- `SignalJsonRpcResponse { jsonrpc: String, id: Option<String>, result: Option<serde_json::Value>, error: Option<serde_json::Value> }`

**jsonrpc.rs:** Implement the JSON-RPC transport layer:
- `enum Transport` wrapping tokio `OwnedReadHalf`/`OwnedWriteHalf` for TCP and `OwnedReadHalf`/`OwnedWriteHalf` for Unix (use `tokio::io::split()`).
- `pub struct JsonRpcClient` with fields: reader (BufReader wrapping read half), writer (write half), request_counter (AtomicU64).
- `pub async fn connect(config: &SignalConfig) -> Result<Self, BlufioError>`: Auto-detect transport. If `config.socket_path` is Some, connect via `tokio::net::UnixStream::connect(path)`. Else connect via `TcpStream::connect(format!("{}:{}", host.unwrap_or("127.0.0.1"), port.unwrap_or(7583)))`. Split stream into read/write halves. Wrap read half in BufReader.
- `pub async fn send_request(&mut self, method: &str, params: serde_json::Value) -> Result<SignalJsonRpcResponse, BlufioError>`: Build SignalJsonRpcRequest with incrementing ID, serialize to JSON, write as single line + newline, flush. Read response line, parse as SignalJsonRpcResponse.
- `pub async fn read_notification(&mut self) -> Result<Option<SignalNotification>, BlufioError>`: Read one line from BufReader. If EOF, return None. Parse as SignalNotification. If method != "receive", log and skip. Return parsed notification.

Use `tokio::io::AsyncBufReadExt` for line reading and `AsyncWriteExt` for writing.

**lib.rs:** Implement `SignalChannel` struct:
- Fields: config (SignalConfig), inbound_rx (Mutex<mpsc::Receiver<InboundMessage>>), inbound_tx (mpsc::Sender<InboundMessage>), health (Arc<Mutex<HealthStatus>>), receive_handle (Option<JoinHandle<()>>), client (Arc<Mutex<Option<JsonRpcClient>>>).
- `pub fn new(config: SignalConfig) -> Result<Self, BlufioError>`: Validate that either socket_path or host is configured. Create mpsc channel(100). Set initial health to Unhealthy("not connected").
- PluginAdapter: name "signal", version 0.1.0, adapter_type Channel, health_check reads from health Arc, shutdown aborts receive_handle.
- ChannelAdapter capabilities: supports_edit: false, supports_typing: false, supports_images: false, supports_documents: false, supports_voice: false, max_message_length: Some(4096), supports_embeds: false, supports_reactions: false, supports_threads: false.
- `connect()`: Spawn a background task that:
  1. Connects to signal-cli via JsonRpcClient::connect().
  2. Sets health to Healthy.
  3. Enters read loop: read_notification(). On Some(notification), parse envelope, extract source/source_name/data_message. Skip if no data_message or no message text. Check for group_info to determine if group message. For group messages, check for @mention of bot (configurable bot_number from config). For DMs, always process. Create InboundMessage with id=uuid, channel="signal", sender_id=source_number, content=message, metadata=json({"chat_id": source_number or group_id, "source_name": source_name, "is_group": bool}). Send via inbound_tx.
  4. On None (EOF) or error: set health to Degraded, enter reconnection loop with exponential backoff (1s, 2s, 4s, 8s, 16s, 32s, 60s cap). Log each retry attempt. On successful reconnect, set health back to Healthy, resume read loop.
- `send()`: Lock client, call send_request("send", params). For group messages (metadata has group_id), use groupId param. For DMs, use recipient param. Return MessageId with response ID or generated UUID.
- `receive()`: Read from inbound_rx.
- `edit_message()` / `send_typing()`: Return Ok(()) (not supported by Signal protocol).
