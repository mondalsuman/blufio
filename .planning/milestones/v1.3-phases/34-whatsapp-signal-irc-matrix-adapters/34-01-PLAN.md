Create the `blufio-whatsapp` crate following the Slack adapter pattern.

**Cargo.toml:** Dependencies: blufio-core, blufio-config (path), async-trait, tokio, serde, serde_json, reqwest (workspace), tracing, semver, hmac (workspace), sha2 (workspace), hex (workspace). Add a `whatsapp-web` feature flag (default off) that conditionally compiles the web module. Do NOT add a `whatsapp-rust` dependency yet -- the Web adapter is a stub.

**types.rs:** Define webhook payload structs per Meta WhatsApp Cloud API:
- `WhatsAppWebhookPayload { object, entry: Vec<WhatsAppEntry> }`
- `WhatsAppEntry { id, changes: Vec<WhatsAppChange> }`
- `WhatsAppChange { field, value: WhatsAppChangeValue }`
- `WhatsAppChangeValue { messaging_product, metadata, contacts, messages, statuses }` (all Option)
- `WhatsAppMetadata { display_phone_number, phone_number_id }`
- `WhatsAppContact { profile: WhatsAppProfile, wa_id }`
- `WhatsAppProfile { name }`
- `WhatsAppMessage { from, id, timestamp, msg_type (renamed from "type"), text: Option<WhatsAppTextBody> }`
- `WhatsAppTextBody { body }`
- `WhatsAppVerifyParams { hub_mode: String, hub_verify_token: String, hub_challenge: String }` (for GET query params, use `hub.mode` etc serde rename)

**api.rs:** Implement `send_whatsapp_message(client: &reqwest::Client, phone_number_id: &str, access_token: &str, to: &str, text: &str) -> Result<String, BlufioError>` that POSTs to `https://graph.facebook.com/v21.0/{phone_number_id}/messages` with JSON body `{ messaging_product: "whatsapp", to, type: "text", text: { body: text } }` using bearer auth. Returns message ID from response.

**cloud.rs:** Implement `WhatsAppCloudChannel` struct with fields: config (WhatsAppConfig), inbound_rx (Mutex<mpsc::Receiver<InboundMessage>>), inbound_tx (mpsc::Sender<InboundMessage>), http_client (Option<reqwest::Client>).
- `new(config: WhatsAppConfig) -> Result<Self>`: Validate variant == "cloud" or None (default to cloud), require phone_number_id and access_token, create mpsc channel(100).
- Implement PluginAdapter: name "whatsapp", version 0.1.0, adapter_type Channel, health_check sends test API call, shutdown no-op.
- Implement ChannelAdapter: capabilities (supports_edit: false, supports_typing: false, supports_images: false, supports_documents: false, supports_voice: false, max_message_length: Some(4096), supports_embeds: false, supports_reactions: false, supports_threads: false). connect() creates reqwest::Client. send() calls api::send_whatsapp_message. receive() reads from inbound_rx. edit_message/send_typing return Ok(()) (not supported).
- Add `pub fn inbound_tx(&self) -> mpsc::Sender<InboundMessage>` so webhook handlers can forward messages.

**web.rs:** Behind `#[cfg(feature = "whatsapp-web")]`. Create `WhatsAppWebChannel` struct that implements ChannelAdapter with all methods returning `BlufioError::Channel { message: "WhatsApp Web adapter is experimental and not yet implemented" }`. Add doc comment: "//! EXPERIMENTAL: WhatsApp Web adapter. This uses unofficial APIs and may result in account bans. Use at your own risk." Include a `pub fn new(config: WhatsAppConfig) -> Result<Self>` that validates variant == "web" and returns the stub.

**lib.rs:** Re-export cloud module always. Conditionally re-export web module with `#[cfg(feature = "whatsapp-web")]`. Provide a factory function: `pub fn create_whatsapp_channel(config: WhatsAppConfig) -> Result<Box<dyn ChannelAdapter + Send + Sync>, BlufioError>` that dispatches based on config.variant ("cloud" | default -> WhatsAppCloudChannel, "web" -> WhatsAppWebChannel if feature enabled, else error).

**webhook.rs:** Create axum route handlers for WhatsApp Cloud API webhooks:

1. `whatsapp_verify(Query(params): Query<WhatsAppVerifyParams>) -> impl IntoResponse`: GET handler for subscription verification. Compare params.hub_verify_token against expected token (from shared state). If match and hub_mode == "subscribe", return (200, hub_challenge). Else (403, empty).

2. `whatsapp_webhook(State(state): State<WhatsAppWebhookState>, headers: HeaderMap, body: Bytes) -> impl IntoResponse`: POST handler for incoming messages.
   - Extract `X-Hub-Signature-256` header, verify HMAC-SHA256 of raw body bytes using app_secret. Reject with 401 if invalid.
   - Parse body as WhatsAppWebhookPayload.
   - Iterate entry[].changes[].value.messages[], extract text messages only (skip status updates, media for now).
   - For each text message: create InboundMessage with id=msg.id, channel="whatsapp", sender_id=msg.from, content=msg.text.body, metadata=json({"chat_id": msg.from, "phone_number_id": value.metadata.phone_number_id}). Send via inbound_tx.
   - Always return 200 (Meta retries on non-200).

3. Define `WhatsAppWebhookState` struct holding: inbound_tx, verify_token, app_secret. Implement Clone.

4. `pub fn whatsapp_webhook_routes(state: WhatsAppWebhookState) -> Router`: Returns Router with GET and POST at "/webhooks/whatsapp".

```rust
use blufio_whatsapp::{create_whatsapp_channel, webhook::WhatsAppWebhookState};

// In run_serve(), after Slack wiring:
#[cfg(feature = "whatsapp")]
{
    if config.whatsapp.phone_number_id.is_some() {
        let whatsapp = create_whatsapp_channel(config.whatsapp.clone()).map_err(|e| {
            error!(error = %e, "failed to initialize WhatsApp channel");
            e
        })?;
        // Get inbound_tx before moving into mux
        // Set up webhook state for gateway
        mux.add_channel("whatsapp".to_string(), whatsapp);
        info!("whatsapp channel added to multiplexer");

        // Redact access token in logs
        if let Some(ref token) = config.whatsapp.access_token {
            blufio_security::RedactingWriter::<std::io::Stderr>::add_vault_value(&vault_values, token.clone());
        }
    } else {
        info!("whatsapp channel skipped (no phone_number_id configured)");
    }
}
```
