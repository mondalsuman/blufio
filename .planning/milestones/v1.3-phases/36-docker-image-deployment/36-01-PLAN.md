**1. Create `crates/blufio/src/healthcheck.rs`:**

Minimal healthcheck module (follow status.rs pattern):

```rust
// SPDX-FileCopyrightText: 2026 Blufio Contributors
// SPDX-License-Identifier: MIT OR Apache-2.0

//! `blufio healthcheck` command for Docker HEALTHCHECK.
//!
//! Connects to the gateway /health endpoint and exits 0 (healthy)
//! or 1 (unhealthy). Designed for use in Dockerfile HEALTHCHECK
//! directives where no shell or curl is available.

use std::time::Duration;
use blufio_config::model::BlufioConfig;
use blufio_core::BlufioError;

/// Run the healthcheck: GET /health and exit 0 or 1.
pub async fn run_healthcheck(config: &BlufioConfig) -> Result<(), BlufioError> {
    let host = &config.gateway.host;
    let port = config.daemon.health_port;
    let url = format!("http://{host}:{port}/health");

    let client = reqwest::Client::builder()
        .timeout(Duration::from_secs(5))
        .build()
        .map_err(|e| BlufioError::Internal(format!("http client error: {e}")))?;

    let resp = client.get(&url).send().await
        .map_err(|e| BlufioError::Internal(format!("health check failed: {e}")))?;

    if resp.status().is_success() {
        Ok(())
    } else {
        Err(BlufioError::Internal(format!("unhealthy: status {}", resp.status())))
    }
}
```

**3. Create `Dockerfile` (project root):**

Multi-stage build with cargo-chef for layer caching:

```dockerfile
# ── Stage 1: Plan dependencies ──────────────────────────────
FROM rust:1.85-bookworm AS chef
RUN cargo install cargo-chef --locked
WORKDIR /src
COPY Cargo.toml Cargo.lock ./
COPY crates/ crates/
RUN cargo chef prepare --recipe-path recipe.json

# ── Stage 2: Build dependencies + binary ────────────────────
FROM rust:1.85-bookworm AS builder

# Install build dependencies for SQLCipher (vendored OpenSSL) and bindgen.
RUN apt-get update && apt-get install -y --no-install-recommends \
    pkg-config libclang-dev cmake make \
    && rm -rf /var/lib/apt/lists/*

RUN cargo install cargo-chef --locked
WORKDIR /src

# Cook dependencies (cached layer — only invalidated when Cargo.toml/Cargo.lock change).
COPY --from=chef /src/recipe.json recipe.json
RUN cargo chef cook --release --recipe-path recipe.json

# Copy full source and build.
COPY Cargo.toml Cargo.lock ./
COPY crates/ crates/
RUN cargo build --release --all-features

# Collect ONNX Runtime shared libraries deposited by ort's copy-dylibs feature.
RUN mkdir -p /ort-libs && \
    find target/release -maxdepth 1 -name 'libonnxruntime*' -exec cp {} /ort-libs/ \;

# ── Stage 3: Minimal runtime image ─────────────────────────
FROM gcr.io/distroless/cc-debian12:nonroot

COPY --from=builder /src/target/release/blufio /blufio
COPY --from=builder /ort-libs/ /usr/lib/

ENV BLUFIO_CONFIG=/config/config.toml
ENV RUST_LOG=blufio=info

EXPOSE 3000

HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
    CMD ["/blufio", "healthcheck"]

ENTRYPOINT ["/blufio"]
CMD ["serve"]
```

Key design decisions:
- Use `gcr.io/distroless/cc-debian12:nonroot` (NOT static-debian12) because ONNX Runtime ships shared glibc-linked .so files
- cargo-chef for dependency layer caching (saves ~15 min on rebuilds)
- ONNX .so files collected from build output and placed in /usr/lib/
- Non-root user enforced by distroless :nonroot tag (uid 65534)
- HEALTHCHECK uses the new `blufio healthcheck` subcommand (no shell/curl needed)

**1. Create `docker-compose.yml` (project root):**

```yaml
# Blufio Docker Compose — single-command deployment.
#
# Usage:
#   1. Copy deploy/.env.example to .env and fill in secrets
#   2. Create config/config.toml (or use defaults)
#   3. docker compose up -d
#
# Volumes:
#   blufio-data: Database, ONNX models, plugins (persistent)
#   ./config:    config.toml (read-only bind mount)

services:
  blufio:
    image: blufio:latest
    # Uncomment to build locally instead of using a pre-built image:
    # build: .
    container_name: blufio
    restart: unless-stopped
    ports:
      - "${BLUFIO_PORT:-3000}:3000"
    volumes:
      - blufio-data:/data
      - ./config:/config:ro
    environment:
      - BLUFIO_CONFIG=/config/config.toml
      - RUST_LOG=${RUST_LOG:-blufio=info}
    env_file:
      - path: .env
        required: false
    healthcheck:
      test: ["/blufio", "healthcheck"]
      interval: 30s
      timeout: 5s
      retries: 3
      start_period: 10s

volumes:
  blufio-data:
    driver: local
```

Design notes:
- Single service (no bundled monitoring/proxy per user decision)
- Named volume `blufio-data` for persistent data
- Config directory bind-mounted read-only
- env_file with `required: false` so compose works without .env
- Health check uses the same blufio healthcheck subcommand
