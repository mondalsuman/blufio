# Plan 14-01: Wire Secure TLS Client into AnthropicClient

## Objective

Replace the plain `reqwest::Client` constructed inside `AnthropicClient::new()` with `build_secure_client()` from blufio-security, which enforces TLS 1.2+ minimum and uses `SsrfSafeResolver` to block SSRF attacks on DNS resolution. On failure, refuse to start.

## must_haves

- [ ] `AnthropicClient` uses a reqwest::Client built by `build_secure_client()` with TLS 1.2+ and SsrfSafeResolver
- [ ] The 300-second streaming timeout is preserved on the secure client
- [ ] Startup fails with a clear error if the secure client cannot be built (no insecure fallback)
- [ ] Existing tests continue to pass (test helper uses plain client internally)
- [ ] `SecurityConfig.allowed_private_ips` is respected for users running local API proxies

## Context

**Current state:** `AnthropicClient::new()` (client.rs:57-64) constructs its own `reqwest::Client::builder()` with only `default_headers` and a 300-second timeout. No TLS minimum, no SSRF protection.

**Target state:** `AnthropicClient::new()` accepts an optional pre-built `reqwest::Client` from `build_secure_client(&config.security)`. When provided, it uses it (adding default headers); when not provided (tests), it builds a plain client.

**Key constraint:** `status` and `doctor` commands use plain reqwest clients for localhost health checks -- those stay untouched (SSRF protection is for outbound API calls only).

   **Better approach:** Instead of passing a pre-built client, accept an `Option<&SecurityConfig>`:
   - When `Some(config)`: call `blufio_security::build_secure_client(config)` and then add headers + timeout
   - Actually, `build_secure_client` returns a `reqwest::Client` -- we need headers. So:
     - Build a `reqwest::ClientBuilder` manually with security settings: `.min_tls_version(reqwest::tls::Version::TLS_1_2)`, `.dns_resolver(Arc::new(SsrfSafeResolver::new(config.allowed_private_ips.clone())))`
     - Then add `.default_headers(headers)` and `.timeout(Duration::from_secs(300))` on the same builder
     - This avoids post-hoc modification of a built client

   **Simplest correct approach:** Refactor to use the security config directly:
   ```rust
   use std::sync::Arc;
   use blufio_config::model::SecurityConfig;
   use blufio_security::SsrfSafeResolver;

   pub fn new(
       api_key: String,
       api_version: String,
       model: String,
       security_config: Option<&SecurityConfig>,
   ) -> Result<Self, BlufioError> {
       let mut headers = HeaderMap::new();
       headers.insert("x-api-key", HeaderValue::from_str(&api_key).map_err(|e| BlufioError::Config(format!("invalid API key header value: {e}")))?);
       headers.insert("anthropic-version", HeaderValue::from_str(&api_version).map_err(|e| BlufioError::Config(format!("invalid API version header value: {e}")))?);
       headers.insert("content-type", HeaderValue::from_static("application/json"));

       let mut builder = reqwest::Client::builder()
           .default_headers(headers)
           .timeout(Duration::from_secs(300));

       // Apply security hardening when config is provided.
       if let Some(sec) = security_config {
           let resolver = SsrfSafeResolver::new(sec.allowed_private_ips.clone());
           builder = builder
               .min_tls_version(reqwest::tls::Version::TLS_1_2)
               .dns_resolver(Arc::new(resolver));
       }

       let client = builder.build().map_err(|e| BlufioError::Provider {
           message: format!("failed to build HTTP client: {e}"),
           source: Some(Box::new(e)),
       })?;

       Ok(Self {
           client,
           default_model: model,
           max_retries: 1,
           base_url: API_BASE_URL.to_string(),
       })
   }
   ```
