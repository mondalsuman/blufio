<objective>
Create the blufio-context crate implementing three-zone context assembly (static/conditional/dynamic) with compaction, and extend the Anthropic API types to support structured system prompt blocks with cache_control for prompt caching.

Purpose: The context engine is the intelligence layer that assembles prompts efficiently -- keeping token overhead low, enabling Anthropic cache hits on the static prefix, and compacting history when it grows too long. This replaces the flat context assembly in blufio-agent/context.rs.

Output: Working blufio-context crate with ContextEngine, StaticZone, ConditionalProvider trait (stubbed), DynamicZone with compaction, and updated Anthropic types supporting cache_control.
</objective>

    Define a return type for compaction-aware message assembly:
    ```rust
    /// Result of dynamic zone assembly, carrying messages and optional compaction cost.
    pub struct DynamicResult {
        /// Assembled messages for the provider request.
        pub messages: Vec<ProviderMessage>,
        /// Token usage from compaction LLM call, if compaction was triggered.
        /// Callers MUST propagate this for cost recording with FeatureType::Compaction.
        pub compaction_usage: Option<TokenUsage>,
    }
    ```

    - `DynamicZone` struct with compaction_threshold (f64), context_budget (u32), compaction_model (String)
    - `DynamicZone::new(config: &ContextConfig) -> Self`
    - `DynamicZone::assemble_messages(provider: &dyn ProviderAdapter, storage: &dyn StorageAdapter, session_id: &str, inbound: &InboundMessage) -> Result<DynamicResult, BlufioError>`
      - Load ALL messages (no limit) from storage for this session
      - Estimate total token count: `messages.iter().map(|m| m.content.len() / 4).sum()` (rough heuristic, 4 chars per token)
      - If estimated_tokens > (context_budget * compaction_threshold), trigger compaction:
        - Split messages into older half and recent half
        - Call `generate_compaction_summary()` for the older half
        - Call `persist_compaction_summary()` to store the summary
        - Capture the `TokenUsage` returned from `generate_compaction_summary()` into `DynamicResult.compaction_usage`
        - Delete or skip the older messages (actually: just don't include them in the returned messages -- the compaction summary message is now in storage and will appear in subsequent loads)
        - Return DynamicResult with: messages = [compaction_summary_as_system_msg] + [recent_messages] + [inbound_message], compaction_usage = Some(usage)
      - Normal path (no compaction): convert all history messages to ProviderMessage format + append inbound. Return DynamicResult with messages and compaction_usage = None.
    - Helper: `message_content_to_blocks()` -- reuse the same logic from blufio-agent/context.rs for converting InboundMessage content to ContentBlock. Import from blufio-agent or duplicate the function (better to duplicate to avoid circular dep since blufio-context should NOT depend on blufio-agent).
