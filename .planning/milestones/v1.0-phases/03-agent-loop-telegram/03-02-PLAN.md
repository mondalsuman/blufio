<objective>
Create the Telegram channel adapter crate (blufio-telegram) implementing the ChannelAdapter trait. This adapter connects to Telegram via long polling, handles message routing with auth filtering, extracts content from multiple message types (text, images, documents, voice), and delivers streaming responses via edit-in-place with proper MarkdownV2 formatting.

Purpose: Telegram is the primary user-facing channel for Blufio v1. This adapter handles all the complexity of the Telegram Bot API -- long polling, rate limiting, media handling, message formatting -- so the agent loop can be channel-agnostic. The streaming response delivery (edit-in-place) gives users immediate feedback as Claude generates.

Output: `blufio-telegram` crate with full ChannelAdapter implementation, media handling, streaming editor, and MarkdownV2 support.
</objective>

    1. Create `crates/blufio-telegram/src/handler.rs`:
       - `is_authorized(msg: &teloxide::types::Message, allowed_users: &[String]) -> bool`:
         - Extract sender user_id (as string) and username
         - Check if either matches any entry in allowed_users
         - If allowed_users is empty, reject all (secure default)
         - Return false for messages without a sender (channel posts)
       - `is_dm(msg: &teloxide::types::Message) -> bool`:
         - Return true only if msg.chat.kind is ChatKind::Private
       - `extract_content(bot: &Bot, msg: &teloxide::types::Message) -> Result<Option<MessageContent>, BlufioError>`:
         - If msg.text() is Some: return MessageContent::Text
         - If msg.photo() is Some: call media::extract_photo_content, attach msg.caption()
         - If msg.document() is Some: call media::extract_document_content
         - If msg.voice() is Some: call media::extract_voice_content
         - Otherwise: return None (unsupported message type)
       - `to_inbound_message(msg: &teloxide::types::Message, content: MessageContent) -> InboundMessage`:
         - Map teloxide Message fields to InboundMessage fields
         - id = msg.id.to_string()
         - channel = "telegram"
         - sender_id = msg.from.user_id.to_string()
         - timestamp = msg.date.to_rfc3339() or ISO 8601

    2. Create `crates/blufio-telegram/src/streaming.rs`:
       - `StreamingEditor` struct: bot (Bot), chat_id (ChatId), message_id (Option&lt;MessageId&gt;), buffer (String), last_edit (Instant), throttle (Duration), messages_sent (Vec&lt;MessageId&gt;)
       - `StreamingEditor::new(bot, chat_id)`: throttle = Duration::from_millis(1500)
       - `StreamingEditor::push_chunk(&mut self, text: &str) -> Result<(), BlufioError>`:
         a. Append text to buffer
         b. Check if buffer exceeds 4000 chars (leave margin below 4096):
            - If yes: find last paragraph boundary (double newline) before 4000 chars
            - Finalize current message up to that boundary
            - Start new message with remainder
         c. Check if last_edit.elapsed() >= throttle:
            - If yes and message_id is None: send_message with escaped text, store message_id
            - If yes and message_id is Some: edit_message_text with escaped text
            - Update last_edit
       - `StreamingEditor::finalize(&mut self) -> Result<(), BlufioError>`:
         - Send final edit with complete buffer content
         - If message_id is None and buffer is not empty: send_message
       - `split_at_paragraph_boundary(text: &str, max_len: usize) -> (&str, &str)`:
         - Find last "\n\n" before max_len, split there
         - If no paragraph boundary: find last "\n", then last space
         - Fallback: hard split at max_len
       - Apply markdown::format_for_telegram() before sending/editing
       - `start_typing_indicator(bot: Bot, chat_id: ChatId, cancel: CancellationToken) -> JoinHandle<()>`:
         - Spawns a tokio task that sends ChatAction::Typing every 5 seconds
         - Stops when cancel token is triggered
         - Uses teloxide `bot.send_chat_action(chat_id, ChatAction::Typing).await`

         - `connect()`: Spawn a tokio task that:
           a. Creates Polling::builder(bot).build()
           b. Iterates over updates via as_stream()
           c. For each Message update: check is_dm, check is_authorized, extract_content, convert to InboundMessage, send via inbound_tx
           d. Non-matching messages are silently dropped

         - `send(msg)`: bot.send_message(ChatId(chat_id), text).parse_mode(MarkdownV2).await, return MessageId
         - `edit_message(chat_id, message_id, text, parse_mode)`: bot.edit_message_text(ChatId, MsgId, text).parse_mode(MarkdownV2).await
         - `send_typing(chat_id)`: bot.send_chat_action(ChatId, ChatAction::Typing).await
         - `shutdown()`: Drop the polling task handle, close the channel
