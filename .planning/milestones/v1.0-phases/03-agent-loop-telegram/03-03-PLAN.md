    6. Create `crates/blufio-agent/src/lib.rs` -- AgentLoop coordinator:
       - `AgentLoop` struct: channel (Box&lt;dyn ChannelAdapter&gt;), provider (Arc&lt;dyn ProviderAdapter&gt;), storage (Arc&lt;dyn StorageAdapter&gt;), config (BlufioConfig), sessions (HashMap&lt;String, SessionActor&gt;), system_prompt (String)
       - `AgentLoop::new(channel, provider, storage, config) -> Result<Self, BlufioError>`:
         - Load system prompt via context::load_system_prompt
         - Initialize empty sessions map
       - `AgentLoop::run(&mut self, cancel: CancellationToken) -> Result<(), BlufioError>`:
         Main loop using tokio::select!:
         ```rust
         loop {
             tokio::select! {
                 msg = self.channel.receive() => {
                     match msg {
                         Ok(inbound) => self.handle_inbound(inbound).await?,
                         Err(e) => tracing::error!("channel receive error: {e}"),
                     }
                 }
                 _ = cancel.cancelled() => {
                     tracing::info!("shutdown signal received");
                     break;
                 }
             }
         }
         // Drain active sessions
         shutdown::drain_sessions(&self.sessions, Duration::from_secs(30)).await;
         // Shutdown storage
         self.storage.close().await?;
         Ok(())
         ```
       - `AgentLoop::handle_inbound(&mut self, inbound: InboundMessage) -> Result<(), BlufioError>`:
         a. Resolve or create session:
            - Look up session by (channel + sender_id) in sessions map
            - If not found: check storage for existing session by user_id + channel
            - If still not found: create new session in storage with uuid, channel="telegram"/"cli", user_id=sender_id, state="active"
         b. Send typing indicator: `self.channel.send_typing(&chat_id)`
         c. Start typing refresh task (every 5 seconds)
         d. Get response stream from session_actor.handle_message(inbound)
         e. Consume stream:
            - If channel supports edit: use StreamingEditor pattern (send initial message, edit as chunks arrive)
            - Else: accumulate text, send as single message
            - Track full response text for persistence
         f. Cancel typing task
         g. Persist assistant response via session_actor.persist_response
         h. Log completion with token usage
