  Changes to SessionActor::handle_message():
  - After `text_content` is extracted, BEFORE budget check:
    1. If routing_enabled:
       a. Parse model override: `let (override_model, clean_text) = blufio_router::ModelRouter::parse_model_override(&text_content);`
       b. If override found, use clean_text for the persisted message content (strip the /opus etc prefix)
       c. Get recent messages for context (last 3 messages from storage): `let recent = self.storage.get_messages(&self.session_id, Some(3)).await?;`
       d. Get budget utilization: `let budget_util = { self.budget_tracker.lock().await.budget_utilization() };`
       e. Call router: `let decision = self.router.route(&clean_text_or_original, &recent_strings, budget_util);`
       f. Store the decision: `self.last_routing_decision = Some(decision.clone());`
       g. Use `decision.actual_model` as the model and `decision.max_tokens` as max_tokens for this request
    2. If not routing_enabled:
       a. Use self.default_model and self.default_max_tokens
       b. Set last_routing_decision = None
  - IMPORTANT: Use the clean text (override stripped) for the persisted user message, NOT the raw text with the /opus prefix
  - Pass the routed model and max_tokens to `self.context_engine.assemble(...)` instead of `&self.model` and `self.max_tokens`
  - When downgraded, build a notification string: `format!("(Using {} -- budget at {:.0}%)", decision.actual_model_short_name(), budget_util * 100.0)` -- this will be prepended by the caller

  Changes to SessionActor::persist_response():
  - When recording cost, use the routing decision's actual_model for the `model` field
  - Set `intended_model` on CostRecord using `decision.intended_model` if routing was used:

  - Use `CostRecord::new(...).with_intended_model(intended)` pattern (or set the field directly if with_intended_model was a builder)

  **Step 4: Spawn heartbeat background task**

  After `agent_loop.run(cancel)` is set up but BEFORE calling it, spawn the heartbeat task:

  ```rust
  // Spawn heartbeat background task if enabled.
  if let Some(ref runner) = heartbeat_runner {
      let hb_runner = runner.clone();
      let hb_cancel = cancel.clone();
      let hb_channel: Box<dyn ChannelAdapter + Send + Sync> = /* need a channel ref for immediate delivery */;
      let interval_secs = config.heartbeat.interval_secs;
      let delivery_mode = config.heartbeat.delivery.clone();

      tokio::spawn(async move {
          let mut interval = tokio::time::interval(
              std::time::Duration::from_secs(interval_secs)
          );
          // Skip the first immediate tick
          interval.tick().await;

          loop {
              tokio::select! {
                  _ = interval.tick() => {
                      match hb_runner.execute().await {
                          Ok(Some(result)) if result.has_content => {
                              if delivery_mode == "immediate" {
                                  // For immediate delivery, we need a channel reference.
                                  // Since we can't easily share the channel, log for now
                                  // and let the agent loop handle it via take_pending_heartbeat.
                                  info!(content = result.content.as_str(), "heartbeat generated (immediate mode)");
                                  // TODO: Send via channel when channel sharing is implemented
                              }
                              // on_next_message mode: content stored internally by HeartbeatRunner
                              info!("heartbeat executed successfully");
                          }
                          Ok(Some(_)) => {
                              debug!("heartbeat executed but no actionable content");
                          }
                          Ok(None) => {
                              debug!("heartbeat skipped (unchanged state)");
                          }
                          Err(e) => {
                              warn!(error = %e, "heartbeat execution failed (non-fatal)");
                          }
                      }
                  }
                  _ = hb_cancel.cancelled() => {
                      info!("heartbeat task shutting down");
                      break;
                  }
              }
          }
      });
  }
  ```

  NOTE: For immediate delivery mode, the heartbeat needs access to a channel. Since the channel is moved into the AgentLoop, the simplest approach for now is:
  - Both delivery modes store the heartbeat content in `pending_heartbeat`
  - For `immediate` mode, the agent loop checks for pending heartbeats periodically and sends them
  - For `on_next_message` mode, the pending heartbeat is retrieved and prepended to the next response
