  **Step 3: Create classifier.rs**

  Define the core classification types and heuristic algorithm:

  ```rust
  /// Query complexity tiers mapped to Claude model families.
  #[derive(Debug, Clone, Copy, PartialEq, Eq)]
  pub enum ComplexityTier {
      Simple,   // Haiku: greetings, time queries, single-fact lookups, yes/no
      Standard, // Sonnet: general conversation, moderate analysis, Q&A
      Complex,  // Opus: multi-step reasoning, code generation, detailed analysis
  }

  /// Result of classifying a query's complexity.
  #[derive(Debug, Clone)]
  pub struct ClassificationResult {
      pub tier: ComplexityTier,
      pub confidence: f32,
      pub reason: &'static str,
  }

  /// Heuristic query classifier. Zero-cost, zero-latency classification.
  pub struct QueryClassifier {
      confidence_threshold: f32, // Below this -> default UP to Standard
  }
  ```

  Classification algorithm (implement in `QueryClassifier::classify(&self, message: &str, recent_context: &[&str]) -> ClassificationResult`):

  1. **Per-message override check**: If message starts with `/opus `, `/haiku `, or `/sonnet `, this is handled by the router, not the classifier. Classifier receives the cleaned message.

  2. **Score accumulation** (i32 score, starts at 0):
     - Message length scoring:
       - 0-3 words: score -= 2 (very short, likely simple)
       - 4-15 words: no change (neutral)
       - 16-50 words: score += 1 (leaning complex)
       - 51+ words: score += 2 (likely complex)
     - Simple pattern matching (case-insensitive exact match against trimmed message):
       - Greetings/farewells: "hi", "hello", "hey", "thanks", "thank you", "bye", "ok", "okay", "yes", "no", "sure", "good", "great", "cool", "nice", "wow", "lol", "haha"
       - If exact match: score -= 3 (strong simple signal)
     - Simple question patterns (case-insensitive contains):
       - "what time", "what day", "what date", "how are you", "what's up", "who are you", "what's your name"
       - If match: score -= 2
     - Complex indicators (case-insensitive contains):
       - "analyze", "compare", "evaluate", "implement", "design", "architecture", "trade-off", "tradeoff", "pros and cons", "step by step", "explain in detail", "debug", "refactor", "code review", "write a function", "write code", "write a program", "optimize", "algorithm", "strategy"
       - If match: score += 2
     - Code block detection:
       - If message contains "```": score += 3 (strong complex signal)
     - Multi-sentence detection:
       - Count sentences (split by `. `, `? `, `! `)
       - 3+ sentences: score += 1
     - Conversation momentum from recent_context:
       - Count recent messages (up to last 3) that contain complex indicators
       - If 2+ complex recent messages: score += 1 (maintain tier)

  3. **Score to tier mapping**:
     - score <= -2: Simple, confidence = min(1.0, (-score as f32) / 5.0)
     - score >= 2: Complex, confidence = min(1.0, (score as f32) / 5.0)
     - else: Standard, confidence = 1.0 - (score.abs() as f32 / 3.0)

  4. **Uncertainty default-up rule**:
     - If confidence < self.confidence_threshold AND tier == Simple: upgrade to Standard
     - Reason: "low confidence, defaulting up"

  Default confidence_threshold: 0.4

  **Step 4: Create router.rs**

  ```rust
  use crate::classifier::{ComplexityTier, QueryClassifier};
  use blufio_config::model::RoutingConfig;

  /// Routing decision with both intended and actual model for cost tracking.
  #[derive(Debug, Clone)]
  pub struct RoutingDecision {
      pub intended_model: String,
      pub actual_model: String,
      pub max_tokens: u32,
      pub downgraded: bool,
      pub tier: ComplexityTier,
      pub reason: String,
  }

  /// Orchestrates model selection: per-message override > global force > classify > budget downgrade.
  pub struct ModelRouter {
      classifier: QueryClassifier,
      config: RoutingConfig,
  }
  ```

  Implement `ModelRouter`:
  - `new(config: RoutingConfig) -> Self`: Create classifier with default threshold
  - `route(&self, message: &str, recent_context: &[&str], budget_utilization: f64) -> RoutingDecision`:
    1. Check per-message override via `parse_model_override(message)`. If found, return immediately with the override model as both intended and actual. Set `downgraded = false`. Return cleaned message text in a separate method or note that the caller should use `parse_model_override` to get the clean text.
    2. Check `config.force_model`. If Some, use that model for both intended and actual.
    3. Classify using `self.classifier.classify(clean_message, recent_context)`
    4. Map tier to model using config: Simple -> config.simple_model, Standard -> config.standard_model, Complex -> config.complex_model
    5. Apply budget downgrade:
       - If budget_utilization >= 0.95: actual = config.simple_model (everything to Haiku)
       - If budget_utilization >= 0.80: downgrade one tier (Complex -> config.standard_model, Standard -> config.simple_model, Simple stays)
       - Else: actual = intended
    6. Set max_tokens based on actual model tier
    7. Set downgraded = intended != actual
    8. Build reason string for logging

  - `parse_model_override(text: &str) -> (Option<String>, &str)`:
    - Static helper function (no &self needed)
    - Check for `/opus `, `/haiku `, `/sonnet ` prefix (with trailing space)
    - Return (model_string, rest_of_message)
    - Use `str::strip_prefix` -- no regex needed

  ```rust
  /// Returns the higher of daily or monthly budget utilization as a fraction (0.0 - 1.0+).
  /// Returns 0.0 if no caps are configured.
  pub fn budget_utilization(&self) -> f64 {
      let daily_util = match self.daily_cap {
          Some(cap) if cap > 0.0 => self.daily_total_usd / cap,
          _ => 0.0,
      };
      let monthly_util = match self.monthly_cap {
          Some(cap) if cap > 0.0 => self.monthly_total_usd / cap,
          _ => 0.0,
      };
      daily_util.max(monthly_util)
  }
  ```
