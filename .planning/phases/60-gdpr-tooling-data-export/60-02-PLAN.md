1. Create `crates/blufio-gdpr/src/erasure.rs`:
   - `pub async fn find_user_sessions(conn: &tokio_rusqlite::Connection, user_id: &str) -> Result<Vec<Session>, GdprError>`:
     Query `SELECT id, channel, user_id, state, metadata, created_at, updated_at, COALESCE(classification, 'Internal') FROM sessions WHERE user_id = ?1 AND deleted_at IS NULL` (respect soft-delete). Return parsed Session structs.

   - `pub fn check_active_sessions(sessions: &[Session]) -> usize`:
     Count sessions where state == "active".

   - `pub async fn execute_erasure(conn: &tokio_rusqlite::Connection, session_ids: &[String], user_id: &str) -> Result<ErasureManifest, GdprError>`:
     All in a single `conn.call(move |conn| { let tx = conn.transaction()?; ... tx.commit()? })`:
     a. DELETE FROM messages WHERE session_id IN (...) AND deleted_at IS NULL -- use dynamic placeholder generation for parameterized IN
     b. DELETE FROM memories WHERE session_id IN (...) -- hard delete (GDPR), FTS5 triggers fire automatically
     c. For each session_id: DELETE FROM compaction_archives WHERE session_ids LIKE '%{sid}%' -- LIKE-based JSON matching (following existing delete_archives_by_session_ids pattern)
     d. UPDATE cost_ledger SET session_id = NULL WHERE session_id IN (...) -- anonymize, preserve aggregates (GDPR-02)
     e. DELETE FROM sessions WHERE id IN (...) AND deleted_at IS NULL -- delete fully owned sessions
     f. Count affected rows for each operation, build ErasureManifest with uuid::Uuid::new_v4() for manifest_id
     Return the manifest.
     Per CONTEXT: all session_ids from find_user_sessions are fully owned (user_id = target). Shared session handling is out of scope since shared sessions have a different user_id and won't appear in find_user_sessions results.

   - `pub async fn erase_audit_trail(audit_conn: &tokio_rusqlite::Connection, user_id: &str) -> Result<u64, String>`:
     Wrap blufio_audit::erase_audit_entries(). Return entries_erased count. On error, return Err(warning_string) -- best effort per CONTEXT.

   - `pub async fn cleanup_memory_index(memory_conn: &tokio_rusqlite::Connection, session_ids: &[String]) -> Result<(), GdprError>`:
     Verify FTS5 consistency: compare `SELECT COUNT(*) FROM memories` vs `SELECT COUNT(*) FROM memories_fts`. If mismatch, run `INSERT INTO memories_fts(memories_fts) VALUES('rebuild')`. This runs synchronously after erasure per CONTEXT decision.

   - `pub fn write_manifest(manifest: &ErasureManifest, export_dir: &Path) -> Result<PathBuf, GdprError>`:

1. Create `crates/blufio-gdpr/src/export.rs`:
   - `pub struct CollectedData { pub messages: Vec<serde_json::Value>, pub sessions: Vec<serde_json::Value>, pub memories: Vec<serde_json::Value>, pub cost_records: Vec<serde_json::Value>, pub restricted_excluded: usize }`

   - `pub async fn collect_user_data(conn: &tokio_rusqlite::Connection, session_ids: &[String], filters: &FilterCriteria) -> Result<CollectedData, GdprError>`:
     a. Query messages: SELECT * FROM messages WHERE session_id IN (...) AND deleted_at IS NULL, apply --since/--until on created_at, apply --session filter, apply --type filter (skip messages if "messages" not in types)
     b. Query sessions: SELECT * FROM sessions WHERE id IN (...) AND deleted_at IS NULL (apply same timestamp/type filters)
     c. Query memories: SELECT id, content, source, confidence, status, session_id, classification, created_at, updated_at FROM memories WHERE session_id IN (...) (skip embedding, apply filters)
     d. Query cost records: SELECT * FROM cost_ledger WHERE session_id IN (...) (apply filters)
     e. For each record, check classification via can_export(). Exclude Restricted records, increment restricted_excluded counter.
     f. Convert each row to serde_json::Value for format-agnostic handling

   - `pub fn apply_redaction(data: &mut CollectedData)`:
     For each message/memory/session with content field, apply `ClassificationGuard::instance().redact_for_export()`. Skip if redact_for_export returns None (Restricted -- should already be filtered).

   - `pub fn write_json_export(data: &CollectedData, metadata: &ExportMetadata, output_path: &Path) -> Result<u64, GdprError>`:
     Build ExportEnvelope, serialize with serde_json::to_string_pretty(), write to file. Create parent dirs if needed. Return file size in bytes. Flush and sync file before returning (data safety per CONTEXT).

   - `pub fn write_csv_export(data: &CollectedData, metadata: &ExportMetadata, output_path: &Path) -> Result<u64, GdprError>`:
     Use csv::Writer. Write one CSV with a "data_type" column to distinguish message/session/memory/cost_record rows.
     Columns: data_type, id, session_id, content, role (messages only, empty for others), created_at, updated_at, classification, metadata_json (raw JSON string for any nested fields).
     Handle CSV escaping via the csv crate (commas, newlines, quotes handled automatically).
     Return file size.

   - `pub fn resolve_export_path(config: &GdprConfig, user_id: &str, format: &str, custom_output: Option<&str>, data_dir: &str) -> PathBuf`:
     If custom_output provided, use that. Otherwise: `{config.export_dir or data_dir/exports}/gdpr-export-{user_id}-{timestamp}.{format}`.

     h. Return ReportData
