3. Create `crates/blufio/src/gdpr_cmd.rs`:
   - `pub async fn handle_gdpr_command(action: GdprCommands, config: &BlufioConfig) -> Result<(), BlufioError>`
     Dispatch to cmd_erase, cmd_report, cmd_export, cmd_list_users.

   - `async fn cmd_erase(config, user, yes, dry_run, skip_export, force, timeout)`:
     a. Fail early if DB encrypted and BLUFIO_DB_KEY not set
     b. Open main DB connection, find user sessions
     c. If no sessions found, print cyan info "No data found for user {user}" and return Ok(())
     d. Check active sessions -- if any and !force, print error and return ActiveSessionsExist
     e. If dry_run: print preview counts (messages, sessions, memories, archives, cost_records) and return
     f. If !yes: show preview counts, prompt "Type YES to confirm erasure for user {user}: ", read line, abort if != "YES"
     g. Emit erasure_started event (if EventBus available -- Optional pattern)
     h. If !skip_export && config.gdpr.export_before_erasure: run export, if fails abort erasure entirely (data safety)
     i. Execute erasure with tokio::time::timeout(Duration::from_secs(timeout))
     j. Write manifest always (even if --skip-export)
     k. Erase audit trail (best-effort, capture warning)
     l. Cleanup memory index (synchronous)
     m. Emit erasure_completed event, record Prometheus metrics
     n. Print green success with summary counts
     o. If audit_warning, print yellow warning

   - `async fn cmd_report(config, user, json)`:
     a. Open DB, find sessions, count data
     b. If json: print serde_json::to_string_pretty of ReportData
     c. Else: print formatted table with cyan headers:
        ```
        Transparency Report for user: {user}
        ─────────────────────────────────
        Messages:      {N}
        Sessions:      {N}
        Memories:      {N}
        Archives:      {N}
        Cost Records:  {N}
        Audit Entries: {N} (not deletable per retention policy)
        ```
     d. Emit report_generated event, record Prometheus counter

   - `async fn cmd_export(config, user, format, session, since, until, data_type, redact, output)`:
     a. Validate format is "json" or "csv"
     b. Open DB, find sessions
     c. If no sessions: print info and return
     d. Build FilterCriteria from args
     e. Collect user data
     f. If redact: apply_redaction
     g. Resolve export path, create parent dirs
     h. Write JSON or CSV based on format
     i. If restricted_excluded > 0: print yellow warning "{N} restricted records excluded from export"
     j. Emit export_completed event, record Prometheus metrics
     k. Print green success with file path and size

   - `async fn cmd_list_users(config, json)`:
     a. Open DB
     b. Query: SELECT user_id, COUNT(*) as session_count FROM sessions WHERE user_id IS NOT NULL AND deleted_at IS NULL GROUP BY user_id ORDER BY user_id
     c. For each user: also count messages, memories, cost_records
     d. If json: print JSON array
     e. Else: print table with columns: User ID | Sessions | Messages | Memories | Cost Records

   - Helper functions:

     - `fn open_audit_db(config: &BlufioConfig) -> Result<Option<tokio_rusqlite::Connection>, BlufioError>`: Open audit.db (stem.audit.db), return None if not found (audit optional)
     - `fn resolve_export_dir(config: &BlufioConfig) -> PathBuf`: From GdprConfig.export_dir or {data_dir}/exports/
