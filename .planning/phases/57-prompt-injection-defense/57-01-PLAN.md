    1. `InjectionMatch` struct: category (InjectionCategory), pattern_index (usize), severity (f64), span (Range<usize>), matched_text (String)

    2. `ClassificationResult` struct: score (f64), matches (Vec<InjectionMatch>), action (String — "clean", "logged", "blocked", "dry_run"), source_type (String), categories (Vec<String> — deduplicated category names)

    3. `InjectionClassifier` struct holding:

    4. Scoring algorithm per CONTEXT.md spec:
       ```
       fn calculate_score(matches: &[InjectionMatch], input_length: usize) -> f64 {
           if matches.is_empty() { return 0.0; }
           let mut score = 0.0;
           for m in matches {
               score += m.severity;  // Base severity 0.1-0.5
               let position_ratio = 1.0 - (m.span.start as f64 / input_length.max(1) as f64);
               score += position_ratio * 0.1;  // Up to 0.1 bonus for early position
           }
           if matches.len() > 1 {
               score += (matches.len() - 1) as f64 * 0.1;  // Multi-match bonus
           }
           score.clamp(0.0, 1.0)
       }
       ```

    5. Two-phase detection (mirrors PII module):
       - Fast path: INJECTION_REGEX_SET.matches(input) — O(1) check if any pattern hits
       - Detail extraction: only if fast path has matches, iterate individual regexes for matched indices to extract spans and categories
       - Also check custom patterns (separate RegexSet)

    6. `classify(&self, input: &str, source_type: &str) -> ClassificationResult`:
       - Run two-phase detection
       - Calculate score
       - Determine action based on mode and thresholds:

         * mode="log": "blocked" if score > threshold for source_type, else "logged" if score > 0, else "clean"
         * mode="block": "blocked" if score > threshold, else "clean" (no log-only in block mode)
