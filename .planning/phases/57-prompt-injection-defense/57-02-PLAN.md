<objective>
Implement L3 HMAC-SHA256 boundary tokens that cryptographically separate system, user, and external content zones in the assembled context.

Purpose: Prevents cross-zone injection by signing each content zone with a per-session HMAC key. Tampered or spoofed content is detected and stripped before reaching the LLM.
Output: BoundaryManager with sign, verify, strip, and wrap operations.
</objective>

let salt = hkdf::Salt::new(hkdf::HKDF_SHA256, session_id.as_bytes());

let okm = prk.expand(&[b"hmac-boundary"], &hmac::HMAC_SHA256).unwrap();

let key = hmac::Key::new(hmac::HMAC_SHA256, &key_bytes);

// ring::hmac — Sign and verify (constant-time)

  <action>
    Create `src/boundary.rs` implementing L3 HMAC boundary token system:

    1. `ZoneType` enum: Static, Conditional, Dynamic (with Display trait for token format)

    2. `BoundaryToken` struct:
       - version: u8 (always 1)
       - zone: ZoneType
       - source: String (provenance: "system", "user", "mcp:{server_name}", "skill:{skill_name}")
       - tag: String (hex-encoded HMAC-SHA256 tag)
       - Parse from/to string format: `<<BLUF-ZONE-v1:{zone}:{source}:{hex_tag}>>`

    3. `BoundedContent` struct:
       - zone: ZoneType
       - source: String
       - content: String (the actual zone content)
       - token: BoundaryToken

    4. `BoundaryManager` struct:
       - session_key: hmac::Key (derived per-session)
       - enabled: bool (from HmacBoundaryConfig)

       Methods:
       - `new(master_key: &[u8; 32], session_id: &str, config: &HmacBoundaryConfig) -> Self`
         * Derives per-session key via HKDF: Salt=session_id.as_bytes(), IKM=master_key, info=b"hmac-boundary"
         * If !config.enabled, stores a flag to bypass all operations
       - `sign_zone(&self, zone: ZoneType, source: &str, content: &str) -> BoundedContent`
         * Computes HMAC-SHA256(session_key, content.as_bytes())
         * Hex-encodes the 32-byte tag
         * Constructs BoundaryToken with v1 prefix
       - `verify_zone(&self, bounded: &BoundedContent) -> bool`
         * Uses ring::hmac::verify (constant-time comparison)
         * Decodes hex tag, verifies against content
       - `wrap_content(&self, zone: ZoneType, source: &str, content: &str) -> String`
         * Returns: `{start_token}\n{content}\n{end_token}`
         * Where tokens are the HMAC boundary strings
       - `validate_and_strip(&self, assembled_context: &str) -> (String, Vec<SecurityEvent>)`
         * Parses all <<BLUF-ZONE-v1:...>> tokens in the text
         * For each zone: extracts content between boundary tokens, verifies HMAC
         * Valid zones: content included in output (tokens stripped)
         * Invalid zones: content stripped entirely, SecurityEvent::BoundaryFailure generated
         * Returns clean text (no boundary tokens visible) + list of failure events
       - `re_sign(&self, zone: ZoneType, source: &str, new_content: &str) -> String`
         * For use after compaction/truncation: generates fresh boundaries for modified content
       - `strip_all_tokens(text: &str) -> String` (static method)
         * Regex-based removal of all <<BLUF-ZONE-v1:...>> patterns

    5. When `enabled=false`: all methods pass through content unchanged (sign returns content as-is, validate returns all content with no failures, strip is no-op).

    6. Update `src/lib.rs` to export boundary module: `pub mod boundary;`

    IMPORTANT: Use `ring::hmac::verify` for timing-safe comparison. NEVER use `==` on HMAC tags.
    IMPORTANT: Token format regex for parsing: `<<BLUF-ZONE-v1:(\w+):([^:]+):([0-9a-f]{64})>>`
