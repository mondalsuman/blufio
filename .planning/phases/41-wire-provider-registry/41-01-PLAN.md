```rust
pub struct ConcreteProviderRegistry {
    providers: HashMap<String, Arc<dyn ProviderAdapter + Send + Sync>>,
    default_provider: String,
}
```

2. **`from_config(config: &BlufioConfig) -> Result<Self, BlufioError>` constructor:**
   - Read `config.providers.default` for the default provider name (fallback to "anthropic")
   - For each provider, use `#[cfg(feature = "...")]` blocks to conditionally init:
     - `anthropic`: `AnthropicProvider::new(config).await` -- only if feature "anthropic" enabled
     - `openai`: `OpenAIProvider::new(config).await` -- only if feature "openai" enabled
     - `ollama`: `OllamaProvider::new(config).await` -- only if feature "ollama" enabled
     - `openrouter`: `OpenRouterProvider::new(config).await` -- only if feature "openrouter" enabled
     - `gemini`: `GeminiProvider::new(config).await` -- only if feature "gemini" enabled
   - Config-required activation: each provider only inits if its config section has meaningful values (e.g., api_key is set for OpenAI/OpenRouter/Gemini, default_model or base_url set for Ollama). Check the provider's config section fields.
   - Per user decision: non-default providers that fail to init log `warn!` and are skipped
   - Per user decision: default provider failure returns hard error
   - Return `ConcreteProviderRegistry { providers, default_provider }`

3. **`from_providers(providers: HashMap<String, Arc<dyn ProviderAdapter + Send + Sync>>, default: String) -> Self` constructor:**
   - Direct construction for testing without API keys
   - No config parsing, no feature gates

4. **`impl ProviderRegistry for ConcreteProviderRegistry`:**
   - `get_provider(&self, name: &str)`: direct HashMap lookup
   - `default_provider(&self) -> &str`: return `&self.default_provider`
   - `list_models(&self, provider_filter: Option<&str>)`:
     - If filter is Some, only query that provider
     - Otherwise iterate all providers
     - For Ollama: call `list_local_models()` (downcast to OllamaProvider or use a method on ProviderAdapter if available)
     - For cloud providers (OpenAI, OpenRouter, Gemini, Anthropic): return static known model lists with ModelInfo { id: "provider/model-name", object: "model", created: 0, owned_by: "provider" }
     - Static lists: OpenAI (gpt-4o, gpt-4o-mini, gpt-4-turbo, o1, o1-mini, o3-mini), Gemini (gemini-2.0-flash, gemini-1.5-pro, gemini-1.5-flash), OpenRouter (pass-through so return empty -- models are cloud-side), Anthropic (claude-sonnet-4-20250514, claude-haiku-35-20241022, claude-opus-4-20250514)

5. **Model routing helper** (used internally or exposed):
   - Parse "provider/model" format: split on first "/" to get (provider_name, model_id)
   - If no "/" found, route to default_provider
