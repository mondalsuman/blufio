     a. Create `pub fn parse_email_body(raw: &[u8]) -> Option<ParsedEmail>` using mail-parser:
        - Create `pub struct ParsedEmail { pub subject: String, pub body: String, pub from: String, pub message_id: Option<String>, pub in_reply_to: Option<String>, pub references: Vec<String>, pub date: Option<String> }`
        - Use `MessageParser::default().parse(raw)?`
        - Extract text body via `message.body_text(0)` -- if None, fall back to `message.body_html(0)` and convert via html2text
        - Extract subject via `message.subject()`
        - Extract from via `message.from()` -- get first address, format as email string
        - Extract message_id via `message.message_id()` as Option<String>
        - Extract in_reply_to via `message.in_reply_to()` as Option<String> (use `.as_text()` or `.as_text_list()` and take first)

        - Iterate lines, collecting until a stop pattern is hit
        - Stop patterns (per CONTEXT.md):
          - Signature delimiter: line == "-- " or line == "--"
          - Gmail/Apple Mail: line starts with "On " and ends with " wrote:" (case-sensitive)
          - Outlook: line starts with "From: " AND next line starts with "Sent: "
        - Skip lines: lines starting with ">" or "> "
        - Trim trailing whitespace from result
        - Return the trimmed result

     c. Create `pub fn html_to_text(html: &str) -> String`:
        - Use `html2text::from_read(html.as_bytes(), 80)` for 80-column wrap
        - Trim result

     d. Create `pub fn markdown_to_html(markdown: &str) -> String`:
        - Use comrak: `comrak::markdown_to_html(markdown, &comrak::Options::default())`

     e. Write tests:

        - `test_strip_quoted_text_outlook`: "Hello\n\nFrom: User\nSent: Monday\nTo: Bot\nSubject: Re: Help" -> "Hello"
        - `test_strip_quoted_text_apple_mail`: "Hello\n\nOn Jan 1, 2026, at 12:00 PM, User <user@example.com> wrote:\n> old" -> "Hello"
        - `test_strip_quoted_text_signature`: "Hello\n\n-- \nJohn Doe\nCEO" -> "Hello"
        - `test_strip_quoted_text_no_quotes`: "Just a plain message" -> "Just a plain message"
        - `test_strip_quoted_text_inline_quotes`: "> quoted\nnot quoted\n> more quoted" -> "not quoted"
        - `test_html_to_text`: "<p>Hello <b>world</b></p>" -> contains "Hello world"
        - `test_markdown_to_html`: "**bold**" -> contains "<strong>bold</strong>"

     a. Create `pub async fn build_smtp_transport(config: &EmailConfig) -> Result<AsyncSmtpTransport<Tokio1Executor>, BlufioError>`:
        - Use SMTP host from `config.smtp_host` (falls back to `config.imap_host`)
        - Port from `config.smtp_port` (default 587)
        - SMTP username from `config.smtp_username` (falls back to `config.username`)
        - SMTP password from `config.smtp_password` (falls back to `config.password`)
        - Use `Credentials::new(username, password)`
        - Use `AsyncSmtpTransport::<Tokio1Executor>::relay(&host)` for TLS or `starttls_relay` for STARTTLS
        - Handle `allow_insecure` config for testing
        - Return the transport

     b. Create `pub async fn send_email_reply(transport: &AsyncSmtpTransport<Tokio1Executor>, config: &EmailConfig, to: &str, subject: &str, text_body: &str, html_body: &str, in_reply_to: Option<&str>, references: Option<&str>) -> Result<String, BlufioError>`:
        - Build lettre Message with `Message::builder()`
        - Set From using `config.from_address` and `config.from_name` (default "Blufio")
        - Set To
        - Set Subject: if starts with "Re: " leave as is, otherwise prepend "Re: "
        - If in_reply_to is Some, add In-Reply-To header
        - If references is Some, add References header
        - Build multipart/alternative body with text/plain and text/html parts
        - If config.email_footer is set, append it to both text and HTML bodies
        - Send via transport, return message ID string
        - On error, return BlufioError::channel_delivery_failed("email", error)

        - Validate config: imap_host, username, password must be set
        - Spawn a tokio task that:
          - Connects to IMAP server via async-imap with TLS (tokio-rustls)
          - Logs in with username/password
          - Enters a loop:
            - Select INBOX (or configured folders)
            - Search for UNSEEN messages
            - For each UNSEEN message:
              - Fetch full RFC822 body
              - Parse with `parsing::parse_email_body()`
              - Check allowed_senders (if list is non-empty, skip if sender not in list)
              - Determine session_id from thread mapping: use in_reply_to or references to look up existing thread. Store message_id -> conceptual thread_id mapping. Use a HashMap<String, String> (message_id -> thread_id) inside the polling loop for thread tracking.
              - Build InboundMessage with channel="email", sender_id=from_address, content=MessageContent::Text(body), metadata=JSON with subject, message_id, in_reply_to, thread_id
              - Send via inbound_tx
              - Mark as \Seen via STORE command
            - Sleep for poll_interval_secs
          - On connection error: exponential backoff retry (5s, 10s, 20s... up to 300s per CONTEXT.md)
          - Log all errors at warn level, connection issues at error level

     b. Create helper `fn build_tls_connector() -> Result<tokio_rustls::TlsConnector, BlufioError>`:
        - Use rustls with default crypto provider and webpki roots
        - Return TlsConnector
