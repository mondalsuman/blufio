1. Create `.github/workflows/bench.yml` with:

   - **Runner**: ubuntu-latest
   - **Steps**:
     a. Checkout code
     b. Install Rust toolchain (stable)
     c. Restore cached baseline from GitHub Actions cache (key: `bench-${{ hashFiles('Cargo.lock') }}-${{ hashFiles('benches/**') }}`)
     d. Run `cargo bench -- --save-baseline main` to generate current results
     e. If cached baseline exists, run `cargo bench -- --baseline main --load-baseline previous` comparison (or use critcmp if needed)
     f. Save current baseline to cache for next run
     g. Upload HTML report (target/criterion/) as GitHub Actions artifact for manual investigation
   - **Regression detection**: Use criterion's built-in comparison. If any benchmark shows >20% regression, the step should fail. Criterion reports "regressed" in output -- grep for it and exit 1.

2. The workflow should:
   - NOT run on PRs (avoid noisy CI on feature branches)
   - Cache baseline keyed on Cargo.lock + bench file hashes
   - Upload target/criterion/ as artifact (HTML reports)
   - Fail if >20% regression detected

3. Keep existing blufio `bench` CLI subcommand unchanged -- this is a parallel system for CI only.
4. Verify workflow YAML is valid: basic syntax check.
