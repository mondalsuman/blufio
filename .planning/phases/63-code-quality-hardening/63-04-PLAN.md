2. **Compaction quality scoring** (crates/blufio-context/tests/proptest_quality.rs):
   - Property: score always in [0.0, 1.0] for any input
   - Property: higher entity/decision retention counts -> higher or equal score (monotonicity)
   - Property: empty input (zero entities, zero decisions) -> score is 0.0
   - Property: perfect retention (all entities and decisions preserved) -> score is 1.0
   - Use `proptest!` macro with `ProptestConfig { cases: 64, ..Default::default() }` for CI speed
   - Generate inputs: random u32 for entity counts (0-100), random u32 for retained counts (0..=total)

3. **PII detection** (crates/blufio-core/tests/proptest_pii.rs):
   - Property: generated valid emails (user@domain.tld pattern) are always detected
   - Property: generated valid phone numbers (international formats) are always detected
   - Property: generated SSN patterns (NNN-NN-NNNN) are always detected
   - Property: generated credit card numbers (Luhn-valid, 16 digits) are always detected
   - Property: random alphanumeric strings (no @ . - digits-only patterns) produce no false positives
   - Use proptest string strategies and custom Luhn-valid CC number generator

4. **Hash chain verification** (crates/blufio-audit/tests/proptest_chain.rs):
   - Property: valid chain of N entries (1-50) always verifies successfully
   - Property: modifying any single entry's content breaks chain verification
   - Property: reordering any two entries breaks verification
   - Property: appending new entries to a valid chain preserves prior chain validity
   - Generate random entry content as arbitrary strings, build chain by computing SHA-256(prev_hash || entry)

5. All proptest files use `ProptestConfig { cases: 64, ..Default::default() }` directly (not env var) per user discretion allowance.
