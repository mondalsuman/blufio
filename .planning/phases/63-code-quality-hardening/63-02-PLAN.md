  <name>Task 1: Unwrap sweep -- blufio-skill, blufio-storage, blufio-memory</name>
  <files>crates/blufio-skill/src/**, crates/blufio-storage/src/**, crates/blufio-memory/src/**</files>
  <action>
For each crate (blufio-skill, blufio-storage, blufio-memory), apply this procedure:

1. Add `#![deny(clippy::unwrap_used)]` to the crate's lib.rs (first line after any existing attributes).
2. Run `cargo clippy -p {crate_name} 2>&1` to find all flagged unwrap() calls.
3. Replace each unwrap() following the decision patterns:
   - **Mutex/RwLock .lock().unwrap()** -> `.lock().expect("lock poisoned")`
   - **Regex::new("literal").unwrap()** -> `.expect("valid regex: pattern_name")`
   - **SystemTime UNIX_EPOCH** -> `.expect("system clock before epoch")`
   - **Channel send** -> `if let Err(e) = send() { tracing::warn!("channel closed: {e}") }`
   - **serde_json parse** -> propagate with `?`
   - **DB/config string parse** -> propagate with `?`
   - **File ops** -> `.map_err(|e| BlufioError::Relevant(format!("context: {e}")))?`
   - **Option unwrap (HashMap get, Vec first)** -> `.ok_or_else(|| BlufioError::Internal("expected X".into()))?`
   - **No #[allow(clippy::unwrap_used)]** -- use .expect() with documented reason instead
   - **Don't touch unwrap_or / unwrap_or_default** -- safe patterns, out of scope
4. DO NOT modify code inside `#[cfg(test)]` modules -- test code keeps unwrap().
5. After fixing all, run `cargo clippy -p {crate_name} -- -D warnings` to confirm clean.
6. Run `cargo test -p {crate_name}` to confirm no regressions.
7. Error context messages follow "operation: details" format (e.g., "parsing skill manifest: invalid TOML").

Process one crate at a time. Commit-worthy state after each crate passes clippy + tests.
  </action>
  <verify>
    <automated>cargo clippy -p blufio-skill -p blufio-storage -p blufio-memory -- -D warnings 2>&1 | tail -30</automated>
  </verify>
  <done>blufio-skill, blufio-storage, blufio-memory all have deny(clippy::unwrap_used) and pass clippy + tests with zero unwrap() in non-test code.</done>
</task>
