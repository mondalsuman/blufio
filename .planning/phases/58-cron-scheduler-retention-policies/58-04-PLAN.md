1. **crates/blufio/src/serve.rs:** Add CronScheduler spawn following the existing background task pattern (same section as heartbeat, memory eviction, webhook delivery, degradation manager):
   - After EventBus init and before the main loop:
   - If `config.cron.enabled`:
     - Create task registry: `let task_registry = Arc::new(blufio_cron::tasks::register_builtin_tasks(db.clone(), config.clone()));`
     - Create scheduler: `let scheduler = CronScheduler::new(db.clone(), task_registry, Some(event_bus.clone()), config.cron.clone()).await;`
     - Spawn with cancellation: `let cron_cancel = cancel_token.clone(); tokio::spawn(async move { scheduler.run(cron_cancel).await; });`
   - The scheduler needs the main DB connection (NOT audit.db). Use the same Database connection used by other subsystems.
   - Add `use blufio_cron::scheduler::CronScheduler;` import.
   - On shutdown: the CancellationToken cancellation already handles graceful stop via the select! in scheduler.run().

2. **crates/blufio/src/doctor.rs:** Add cron health check following existing doctor check pattern:
   - New check function (e.g., `check_cron`):
     - Query `SELECT COUNT(*) FROM cron_jobs WHERE enabled = 1` -- report number of active jobs
     - Query `SELECT COUNT(*) FROM cron_jobs WHERE running = 1` -- warn if any jobs have stale locks (running = 1 but last_run_at is old)
     - Query `SELECT name, last_run_at FROM cron_jobs WHERE enabled = 1 AND last_run_at IS NOT NULL ORDER BY last_run_at DESC LIMIT 1` -- report most recent execution
     - Query `SELECT COUNT(*) FROM cron_history WHERE status = 'failed' AND started_at > datetime('now', '-24 hours')` -- warn on recent failures
   - Output format following existing doctor check pattern:
     - OK: "Cron: 3 active jobs, 0 stale locks, last run 2h ago"
     - WARN: "Cron: 1 stale lock (job 'backup' running since 3h ago)"
     - WARN: "Cron: 2 failed jobs in last 24h"
   - Also add retention health check:
     - If retention enabled: report configured retention periods
     - Query soft-deleted records count: `SELECT COUNT(*) FROM messages WHERE deleted_at IS NOT NULL` (and same for other tables)
     - Report: "Retention: 42 records pending permanent deletion"
  </action>
