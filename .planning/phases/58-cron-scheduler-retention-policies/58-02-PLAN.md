1. **crates/blufio-cron/src/scheduler.rs:** Create `CronScheduler` struct:
   - Fields: `db: Arc<tokio_rusqlite::Connection>`, `task_registry: Arc<HashMap<String, Box<dyn CronTask>>>`, `event_bus: Option<Arc<EventBus>>`, `config: CronConfig`
   - `pub async fn new(db, task_registry, event_bus, config) -> Self` -- loads job definitions from DB (cron_jobs table), syncs with config (upsert jobs from config, mark missing ones disabled)
   - `pub async fn run(self, cancel: CancellationToken)` -- main loop:
     - `tokio::time::interval(Duration::from_secs(60))` with `tick().await` skip
     - On each tick: for each enabled job, parse cron expression with `croner::Cron::from_str(&job.schedule)`, check `find_next_occurrence` against last_run_at, if due:
       - Acquire single-instance lock: `UPDATE cron_jobs SET running = 1 WHERE name = ? AND running = 0` -- check changes() == 1
       - If lock acquired, `tokio::spawn` the task execution with timeout (`tokio::time::timeout`)
       - On completion: release lock, update last_run_at, record history, emit CronEvent on EventBus
       - On failure/timeout: release lock, record history with failed/timeout status, emit CronEvent::Failed
     - `tokio::select!` with `cancel.cancelled()` for graceful shutdown
   - `pub async fn run_now(&self, job_name: &str) -> Result<String, CronError>` -- execute a specific job immediately (for CLI run-now), same lock/history/event pattern
   - Define `CronError` with thiserror: InvalidCronExpression, JobNotFound, JobAlreadyRunning, TaskError, DatabaseError

2. **crates/blufio-cron/src/history.rs:** Job execution history persistence:
   - `pub async fn record_start(conn, job_name) -> Result<i64>` -- INSERT into cron_history with status='running', return id
   - `pub async fn record_finish(conn, history_id, status, duration_ms, output) -> Result<()>` -- UPDATE finished_at, status, duration_ms, output (truncate output to 4096 chars)
   - `pub async fn query_history(conn, job_name: Option<&str>, limit: usize) -> Result<Vec<CronHistoryEntry>>` -- SELECT with optional job_name filter, ORDER BY started_at DESC LIMIT
   - `pub async fn cleanup_old_history(conn, max_per_job: usize) -> Result<u64>` -- DELETE excess entries per job beyond max_history
   - Define `CronHistoryEntry` struct: id, job_name, started_at, finished_at, status, duration_ms, output

3. **crates/blufio-cron/src/lib.rs:** Update to export scheduler and history modules. Add `pub mod scheduler; pub mod history;`

6. **Retention engine (crates/blufio-cron/src/retention/):**

   **mod.rs:** `RetentionEnforcer` struct with `db: Arc<Connection>`, `config: RetentionConfig`.
   - `pub async fn enforce(&self) -> Result<RetentionReport>` -- orchestrates phase 1 then phase 2, returns report.
   - `RetentionReport` struct: soft_deleted_count, permanently_deleted_count, per-table breakdowns.

   **soft_delete.rs:** Phase 1 -- mark records for deletion.
   - For each table (messages, sessions, cost_ledger, memories):
     - `UPDATE {table} SET deleted_at = datetime('now') WHERE deleted_at IS NULL AND created_at < datetime('now', '-{days} days') AND (classification IS NULL OR classification != 'restricted')`
     - Separate query for restricted: `... AND classification = 'restricted' AND created_at < datetime('now', '-{restricted_days} days')`
     - Skip table if retention period is None (not configured)
   - CRITICAL: Never touch audit.db. Retention operates ONLY on main database connection. This is architectural isolation per RETN-04.

   **permanent.rs:** Phase 2 -- permanently delete past-grace records.
   - For each table: `DELETE FROM {table} WHERE deleted_at IS NOT NULL AND deleted_at < datetime('now', '-{grace_period_days} days')`
   - CRITICAL for memories table: Delete from `memories` table directly (not memories_fts). FTS5 AFTER DELETE trigger handles FTS cleanup automatically.
   - Return count of permanently deleted records per table.
