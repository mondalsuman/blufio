   ```rust
   /// Manage scheduled cron jobs.
   #[command(after_help = "Examples:\n  blufio cron list\n  blufio cron add my-job \"0 */6 * * *\" backup\n  blufio cron run-now retention_enforcement\n  blufio cron history --job backup --limit 20\n  blufio cron generate-timers")]

   },
   ```
   Define `CronCommands` enum with 6 variants following existing Subcommand pattern:
   - `List { json: bool }`
   - `Add { name: String, schedule: String, task: String }`
   - `Remove { name: String }`
   - `RunNow { name: String }`
   - `History { job: Option<String>, limit: usize (default 20), json: bool }`
   - `GenerateTimers { output_dir: String (default ".") }`

   Add `Commands::Cron { action } => cron_cmd::handle_cron_command(action, &db_path).await?` to the match.

3. **crates/blufio/src/cron_cmd.rs:** Implement `pub async fn handle_cron_command(action: CronCommands, db_path: &str) -> Result<()>`:
   - **List:** Query cron_jobs table, parse each schedule with `croner::Cron::from_str` to compute next_run, format as table (name | schedule | task | enabled | next_run | last_status). Support --json output.
   - **Add:** Validate cron expression with `croner::Cron::from_str`. INSERT INTO cron_jobs (name, schedule, task, enabled). Print confirmation.
   - **Remove:** DELETE FROM cron_jobs WHERE name = ?. Print confirmation or "not found".
   - **RunNow:** Open DB, look up task name from cron_jobs, create appropriate CronTask instance via `register_builtin_tasks`, call execute() with timeout. Record in history. Print output.
   - **History:** Call `query_history()` from blufio-cron. Format as table (job | started | status | duration | output_preview). Support --json and --job filter.
   - **GenerateTimers:** Call `blufio_cron::systemd::generate_timers()`. Print paths of generated files.

   Follow existing CLI handler pattern (e.g., audit_cmd.rs or similar). Use `open_connection_sync` for direct DB access in CLI context per project convention (Phase 54 decision).

4. **crates/blufio-cron/src/systemd.rs:** Implement timer file generation:
   - `pub fn generate_timers(jobs: &[CronJobRow], blufio_path: &str, output_dir: &Path) -> Result<Vec<PathBuf>>`
   - For each enabled job:
     - Convert cron expression to approximate systemd OnCalendar format. For simple expressions (specific minute/hour/dow), do direct mapping. For complex patterns (L/W/#), use a conservative fallback and emit warning.
     - Write `blufio-cron-{name}.timer` using TIMER_TEMPLATE from research
     - Write `blufio-cron-{name}.service` using SERVICE_TEMPLATE from research
   - Return list of created file paths
   - `fn cron_to_on_calendar(schedule: &str) -> Result<String>` -- best-effort conversion, returns OnCalendar string
