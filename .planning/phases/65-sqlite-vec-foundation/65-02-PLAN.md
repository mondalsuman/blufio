  <name>Task 1: Wire vec0 dual-write into MemoryStore save/evict/soft_delete with population and metrics</name>
  <files>crates/blufio-memory/src/store.rs</files>
  <behavior>
    - Test: save() with vec0_enabled inserts into both memories and memories_vec0 atomically
    - Test: save() with vec0_enabled=false does NOT insert into memories_vec0
    - Test: save() with vec0 write failure rolls back entire transaction (memory not saved either)
    - Test: batch_evict() with vec0_enabled deletes from memories_vec0 in same transaction
    - Test: soft_delete() with vec0_enabled updates status in memories_vec0 to 'forgotten'
    - Test: populate_vec0() copies active embeddings to vec0 in batches, skipping existing rows
    - Test: populate_vec0() is idempotent -- second call inserts 0 rows
    - Test: rebuild_vec0() drops and recreates table, then repopulates
  </behavior>
  <action>
    1. Add `vec0_enabled: bool` field to MemoryStore struct. Set it in constructors:
       - `pub fn new(conn: Connection) -> Self` -- vec0_enabled = false (backward compatible)
       - `pub fn with_event_bus(conn: Connection, event_bus: Arc<EventBus>) -> Self` -- vec0_enabled = false
       - Add `pub fn with_vec0(conn: Connection, event_bus: Option<Arc<EventBus>>, vec0_enabled: bool) -> Self` -- new constructor that accepts the toggle.

    2. Modify `save()` to use a transaction with dual-write when vec0_enabled:
       - Wrap existing INSERT in a transaction
       - After memories INSERT, query `SELECT rowid FROM memories WHERE id = ?1` to get the rowid
       - Call `vec0::vec0_insert()` with the rowid and all columns from the Memory struct
       - Commit transaction (both succeed or both fail per locked decision)
       - If vec0_enabled is false, keep the existing non-transactional INSERT (no behavior change)

    3. Modify `batch_evict()` to sync vec0:
       - Inside the existing transaction, after collecting IDs to evict, also DELETE from memories_vec0 for those rowids
       - Query rowids from memories for the IDs being evicted: `SELECT rowid FROM memories WHERE id IN (...)`
       - Call `vec0::vec0_delete()` for each rowid in the same transaction
       - Only when vec0_enabled is true

    4. Modify `soft_delete()` to sync vec0:
       - Wrap in transaction
       - After status UPDATE on memories, call `vec0::vec0_update_status()` with new status 'forgotten'
       - Query rowid: `SELECT rowid FROM memories WHERE id = ?1`
       - Only when vec0_enabled is true

    5. Add `pub async fn populate_vec0(&self) -> Result<(usize, usize), BlufioError>` method:
       - Delegates to conn.call() with vec0::vec0_populate_batch(conn, 500)
       - Logs info!("Populating vec0: {populated}/{total} memories...")
       - Emits Vec0PopulationComplete event via event_bus
       - Returns (populated, total)

    6. Add `pub async fn rebuild_vec0(&self) -> Result<usize, BlufioError>` method:
       - Delegates to conn.call() with vec0::vec0_drop_and_recreate() then vec0::vec0_populate_batch()
       - Logs info level progress
       - Returns count of repopulated rows

    7. Add Prometheus metrics (using existing metrics crate macros):
       - `histogram!("blufio_memory_vec0_search_duration_seconds")` -- recorded in retriever (Plan 02 Task 2)
       - `counter!("blufio_memory_vec0_fallback_total")` -- recorded in retriever (Plan 02 Task 2)
       - `gauge!("blufio_memory_vec0_row_count")` -- updated after population and rebuild

    8. Write unit tests in store.rs #[cfg(test)] module:
       - All tests create in-memory SQLite, call ensure_sqlite_vec_registered(), run migrations manually (memories table + V15 vec0 table).
       - Test dual-write save, vec0 sync on evict/soft_delete, population idempotency, rebuild.

    3. Replace `vector_search()` method internals:
       ```rust
       async fn vector_search(&self, query_embedding: &[f32]) -> Result<Vec<(String, f32)>, BlufioError> {
           if self.vec0_enabled {
               let start = std::time::Instant::now();
               let result = self.vec0_vector_search(query_embedding).await;
               match result {
                   Ok(results) => {
                       histogram!("blufio_memory_vec0_search_duration_seconds").record(start.elapsed().as_secs_f64());
                       return Ok(results);
                   }
                   Err(e) => {
                       // Per-query fallback: log and fall through to in-memory
                       self.log_vec0_fallback(&e);
                       counter!("blufio_memory_vec0_fallback_total").increment(1);
                       // Fall through to in-memory search below
                   }
               }
           }
           // Existing in-memory cosine similarity search (unchanged)
           self.in_memory_vector_search(query_embedding).await
       }
       ```

    4. Extract existing in-memory search into `async fn in_memory_vector_search()` (private method):
       - Move the current get_active_embeddings() + cosine_similarity loop into this method
