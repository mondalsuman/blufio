  <files>crates/blufio-core/src/format.rs, crates/blufio-core/src/types.rs</files>
  <action>
1. In types.rs (or format.rs, whichever is more appropriate given existing structure), define:
   - `ColumnAlign` enum: Left, Center, Right (derive Debug, Clone, Copy, PartialEq, Serialize, Deserialize)
   - `ListStyle` enum: Bullet, Ordered (derive Debug, Clone, Copy, PartialEq, Serialize, Deserialize)
   - `Table` struct: headers: Vec<String>, rows: Vec<Vec<String>>, alignment: Vec<ColumnAlign>
   - `List` struct: items: Vec<String>, style: ListStyle

2. Add two new variants to `RichContent` enum:
   - `RichContent::Table(Table)`
   - `RichContent::List(List)`

3. Implement Table degradation in FormatPipeline::format() (or a helper function):
   - Accept ChannelCapabilities to determine degradation tier
   - **Tier 1 (supports_code_blocks=true):** Unicode box-drawing table wrapped in ``` code block
     - Use box chars: corner (top-left, top-right, bottom-left, bottom-right), horizontal, vertical, T-junctions
     - Compute column widths using `str.chars().count()` (not `.len()`)
     - Auto-wrap in triple-backtick code fence
   - **Tier 2 (formatting_support=FullMarkdown, supports_code_blocks=false):** GFM markdown table
     - `| Header1 | Header2 |` with alignment markers (`---:`, `:---:`, `:---`)
   - **Tier 3 (PlainText or BasicMarkdown):** Key:value per row
     - Format: `Header1: Value1 | Header2: Value2`
   - Empty table: render headers + "(no data)" line below
   - Cell content truncated with ellipsis (...) if a single row exceeds max_message_length
   - Long tables: split into multiple messages with headers repeated per chunk
   - All tiers produce FormattedOutput::Text (no new FormattedOutput variants)

4. Implement List rendering:
   - Bullet style: `- item1\n- item2\n- item3`
   - Ordered style: `1. item1\n2. item2\n3. item3`
   - Lists degrade cleanly to all channels (universal format)
   - Produces FormattedOutput::Text

5. Update lib.rs re-exports for Table, List, ColumnAlign, ListStyle.
