<task type="auto">
  <name>Task 1: Migrate Anthropic, OpenAI, and Gemini provider crates to typed errors</name>
  <files>crates/blufio-anthropic/src/client.rs, crates/blufio-openai/src/client.rs, crates/blufio-gemini/src/client.rs</files>
  <action>
For each of the 3 provider crates (anthropic, openai, gemini):

1. Remove the per-crate `is_transient_error(status)` function.

2. Replace every `BlufioError::Provider { message: format!(...), source: ... }` construction with the appropriate typed constructor:
   - HTTP error responses: Use `BlufioError::provider_from_http(status.as_u16(), "anthropic"|"openai"|"gemini", Some(Box::new(err)))`. This automatically handles Anthropic 529 -> RateLimited mapping.
   - Timeout errors: Use `BlufioError::provider_timeout("provider_name")`
   - Auth errors (explicit): Use `BlufioError::provider_auth_failed("provider_name")`
   - Model not found: Use `BlufioError::provider_model_not_found(model, "provider_name")`
   - Network/connection errors: Use `BlufioError::Provider { kind: ProviderErrorKind::ServerError, context: ErrorContext { provider_name: Some("...".into()), ..Default::default() }, source: Some(Box::new(err)) }`

3. Extract retry_after header from HTTP responses (header name: `retry-after` for OpenAI, `retry-after` for Anthropic) and pass to ErrorContext.retry_after as Duration.

4. Replace retry loop condition from `is_transient_error(status)` to constructing the error first, then checking `error.is_retryable()`.

5. Remove any `provider_generic()` deprecated fallback calls -- use typed constructors for all sites.

Compile check after each crate: `cargo check -p blufio-anthropic`, etc.
