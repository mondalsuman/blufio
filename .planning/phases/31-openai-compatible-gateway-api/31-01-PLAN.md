**2. Implement SSE streaming in openai_compat/stream.rs:**

```rust
/// Stream provider response as OpenAI-compatible SSE chunks.
/// Maps ProviderStreamChunk events to GatewaySseChunk format.
pub async fn stream_completion(
    provider: Arc<dyn ProviderAdapter + Send + Sync>,
    request: ProviderRequest,
    response_id: String,
    model: String,
    include_usage: bool,
) -> Sse<impl Stream<Item = Result<Event, Infallible>>>
```

Logic:
1. Call `provider.stream(request)` to get the stream
2. Map each `ProviderStreamChunk` to `GatewaySseChunk`:
   - `ContentBlockDelta` with text -> delta.content
   - `ContentBlockStop` with tool_use -> delta.tool_calls entry
   - `MessageDelta` with stop_reason -> finish_reason (mapped via stop_reason_to_finish_reason)
   - `MessageDelta` with usage -> only include if include_usage=true
   - `MessageStop` -> emit final chunk then `data: [DONE]\n\n`
3. Serialize each chunk as `data: {json}\n\n` SSE event
4. Terminate with `data: [DONE]\n\n`

Use `futures::stream` combinators. Handle errors by emitting a final error chunk.

pub async fn post_chat_completions(

Logic:
1. Validate request (model required, messages non-empty)
2. Parse model string: split on first `/` -> (provider_name, model_name) or (default, model)
