<objective>
Create the canary token system for detecting system prompt leaking in LLM output, with event bus integration and Prometheus metrics.

Purpose: Implements INJ-07 -- a canary token is planted in the system prompt and if the LLM echoes it back (indicating prompt extraction attack), the response is blocked. This is independent of the normalization/pattern work in Plan 01 and can execute in parallel.

Output: canary.rs module, output_screen.rs canary detection integration, SecurityEvent::CanaryDetection variant, canary Prometheus metrics, scan duration histogram.

    Integrate canary token detection into the OutputScreener so that LLM responses containing canary tokens are blocked.

    1. Add `canary: Option<crate::canary::CanaryTokenManager>` field to OutputScreener struct. Make it Optional so tests and non-canary contexts can pass None.

    2. Update OutputScreener::new() to accept an optional `canary: Option<crate::canary::CanaryTokenManager>` parameter. This is a signature change -- callers (pipeline.rs) will be updated in Plan 03.

    3. Add a new public method `screen_llm_response(&mut self, response: &str, correlation_id: &str) -> ScreeningResult`:
       - This method scans COMPLETE (buffered) LLM response text for canary token leaks.
       - If canary is None, return Allow immediately.
       - Call `canary.detect_leak(response)`.
       - If leak detected:
         - Determine token_type via `canary.detected_token_type(response)`.
         - Record metric via `metrics::record_canary_detection(token_type)`.
         - Create SecurityEvent via `canary_detection_event(correlation_id, token_type, "blocked", response)`.
         - If dry_run, return DryRun with description.
         - Otherwise, increment session_failure_count and return Block("canary token leak detected").
       - If no leak, return Allow.

    4. In the existing `screen_content` method, do NOT add canary detection (that's for tool args, not LLM response). Canary detection is ONLY in the new `screen_llm_response` method per the decision "Scans complete LLM response (not streamed chunks)".

    6. Add new tests:

       - `clean_llm_response_allowed`: screen_llm_response with no canary tokens returns Allow.
       - `canary_none_always_allows`: screener with canary=None always returns Allow from screen_llm_response.
       - `canary_dry_run_does_not_block`: dry_run screener with canary returns DryRun, not Block.
       - `canary_detection_generates_event`: verify events vec contains CanaryDetection event.
