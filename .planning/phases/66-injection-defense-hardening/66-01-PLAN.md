    - decode_base64_segments finds 20+ char base64 segments via regex, decodes valid ones to UTF-8 strings
    - decode_base64_segments ignores segments that don't decode to valid UTF-8
    - extract_content extracts text from HTML comments (<!-- ... -->), markdown code fences (```...```), and recursively extracts JSON string values
    - extract_json_strings stops after 50KB budget
    - NormalizationReport tracks zero_width_count, confusables_mapped, base64_segments_decoded
    - normalize() runs full pipeline: strip zero-width -> NFKC -> map confusables -> decode base64
    - extract_content produces ExtractedSegment with SegmentSource enum (HtmlComment, MarkdownFence, JsonValue)

    3. Create crates/blufio-injection/src/normalize.rs with:

    Public types:
    - `NormalizationReport { zero_width_count: usize, confusables_mapped: usize, base64_segments_decoded: usize }` (Debug, Clone, Default)
    - `NormalizedInput { text: String, decoded_segments: Vec<String>, extracted_segments: Vec<ExtractedSegment>, report: NormalizationReport }` (Debug, Clone)
    - `ExtractedSegment { content: String, source: SegmentSource }` (Debug, Clone)
    - `SegmentSource` enum: HtmlComment, MarkdownFence, JsonValue (Debug, Clone)

    Public functions:
    - `normalize(input: &str) -> NormalizedInput`: Pipeline: strip_zero_width -> NFKC (unicode_normalization::UnicodeNormalization::nfkc()) -> map_confusables -> decode_base64_segments. The extracted_segments field is empty (populated by extract_content separately for tool outputs).
    - `extract_content(input: &str) -> Vec<ExtractedSegment>`: Extracts HTML comment content via regex `<!--([\s\S]*?)-->`, markdown fence content via regex `` ```[^\n]*\n([\s\S]*?)``` ``, and JSON string values via recursive serde_json::Value walk. Each segment is independent (per-segment scanning, NOT concatenated).

    Internal functions:
    - `strip_zero_width(input: &str) -> (String, usize)`: Remove ZERO_WIDTH_CHARS constant array (U+200B, U+200C, U+200D, U+FEFF, U+2060, U+180E, U+00AD) plus Unicode tag characters (U+E0001..=U+E007F). Return (cleaned, count).
    - `map_confusables(input: &str) -> (String, usize)`: Static match expression mapping ~50-80 Cyrillic/Greek characters to Latin equivalents. See RESEARCH.md Pattern 2 for the initial mapping table. Return (mapped, count).
    - `confusable_to_latin(c: char) -> Option<char>`: The actual char->char mapping via match expression.
    - `decode_base64_segments(input: &str) -> (Vec<String>, usize)`: LazyLock regex `[A-Za-z0-9+/]{20,}={0,2}` to find segments. Decode each with base64::engine::general_purpose::STANDARD. Keep only valid UTF-8 non-empty decoded results.
    - `extract_json_strings(value: &serde_json::Value, out: &mut Vec<String>, budget: &mut usize)`: Recursive walk of JSON values, extracting String values. Budget starts at 50*1024, decremented per string. Stops when budget hits 0.

    Use LazyLock for regex compilation (HTML comment regex, markdown fence regex, base64 segment regex).

    Write comprehensive unit tests in a `#[cfg(test)] mod tests` block covering all behaviors listed above.
