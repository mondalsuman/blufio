    - classify() normalizes input and scans BOTH original AND normalized text, deduplicating matches by (pattern_index, span)
    - Evasion bonus: 0.1 if zero_width_count > 0, 0.1 if confusables_mapped > 0 (additive, max 0.2)
    - Decoded base64 segments are individually classified -- if any decoded segment matches a pattern, an EncodingEvasion match is added
    - calculate_score uses severity_weights: weight * base_severity per match. weight=0.0 skips match. weight clamped to [0.0, 3.0]
    - Invalid weights (negative, NaN) treated as 1.0 with tracing::warn
    - ClassificationResult gains normalization_report field (NormalizationReport)
    - InjectionClassifier stores severity_weights from config
    - Pipeline scan_input wraps classifier.classify() in tokio::time::timeout(Duration::from_millis(20))
    - On timeout: return clean result (score 0, action "clean", flagged false) and log warning
    - On success: record scan duration via metrics::record_scan_duration
    - Pipeline passes CanaryTokenManager to OutputScreener constructor
    - Config re-exports updated types (no code change needed if using wildcard re-export)

       d. Add `normalization_report: Option<crate::normalize::NormalizationReport>` field to ClassificationResult (Option to not break tests that construct it manually).
       e. Rewrite classify() method:
          - Call `crate::normalize::normalize(input)` to get NormalizedInput.
          - Scan ORIGINAL input with INJECTION_REGEX_SET + custom patterns (existing logic).
          - If normalized.text != input, also scan normalized.text with same patterns.
          - Merge matches from both scans, deduplicating by (pattern_index, matched_text) -- use a HashSet of (usize, String) to track seen.
          - For each decoded_segment in normalized.decoded_segments:
            - Classify the decoded segment text against all patterns.
            - If any match found, add an InjectionMatch with category=EncodingEvasion, severity=highest_matched_severity, and the decoded match text.
          - Compute evasion_bonus: 0.0 + (0.1 if report.zero_width_count > 0) + (0.1 if report.confusables_mapped > 0).
          - Call updated calculate_score() with severity_weights and evasion_bonus.
          - Set normalization_report on ClassificationResult.
       f. Update calculate_score() signature to accept `severity_weights: &HashMap<String, f64>` and `evasion_bonus: f64`:
          - For each match, look up weight: `severity_weights.get(&m.category.to_string()).copied().unwrap_or(1.0)`.
          - Validate weight: if negative or NaN, warn and use 1.0. Clamp to [0.0, 3.0].
          - If weight == 0.0, skip this match entirely.
          - `score += m.severity * weight` (instead of just m.severity).
          - Positional bonus and match count bonus unchanged.
          - Add evasion_bonus at the end (independent of category weights).
          - Clamp to [0.0, 1.0].

          - `classify_confusable_chars_detected`: Cyrillic lookalike text for "ignore" detected after normalization

          - `classify_severity_weight_zero_disables_category`: weight=0.0 for role_hijacking makes "ignore previous instructions" score 0
