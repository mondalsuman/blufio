1. Create `crates/blufio-bus/src/lib.rs`:

   ```rust

   ```

   Define `EventBus` struct:
   ```rust
   use tokio::sync::{broadcast, mpsc, RwLock};
   use tracing::warn;

   pub struct EventBus {
       broadcast_tx: broadcast::Sender<BusEvent>,
       reliable_txs: RwLock<Vec<mpsc::Sender<BusEvent>>>,
   }
   ```

   Implement:
   - `EventBus::new(capacity: usize) -> Self` — creates broadcast channel with given capacity, empty reliable_txs
   - `EventBus::publish(&self, event: BusEvent)` — sends to broadcast_tx (ignore send error = no subscribers), then iterates reliable_txs and calls `try_send()` on each. If try_send fails, log `tracing::error!("reliable subscriber dropped event — channel full or closed")`.
   - `EventBus::subscribe(&self) -> broadcast::Receiver<BusEvent>` — returns broadcast_tx.subscribe()
   - `EventBus::subscribe_reliable(&self, buffer: usize) -> mpsc::Receiver<BusEvent>` — creates mpsc::channel(buffer), pushes sender to reliable_txs, returns receiver. Use `.write().await` on RwLock.
   - `EventBus::subscriber_count(&self) -> usize` — returns broadcast_tx.receiver_count()

   Make EventBus Send + Sync by ensuring all fields satisfy bounds (broadcast::Sender and RwLock are already Send + Sync).
