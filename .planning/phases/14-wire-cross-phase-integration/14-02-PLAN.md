# Plan 14-02: Wire RedactingWriter into Tracing Subscriber

## Objective

Replace the plain `tracing_subscriber::fmt()` writer in `init_tracing()` with `RedactingWriter` from blufio-security, so that all log output automatically redacts Anthropic API keys (`sk-ant-*`), Telegram bot tokens, and vault-stored secrets. Initialize with empty vault values (populated later when vault is unlocked). On failure, refuse to start.

## must_haves

- [ ] `init_tracing()` wraps the fmt writer with `RedactingWriter`
- [ ] API keys matching `sk-ant-*` pattern are redacted in log output as `[REDACTED]`
- [ ] Telegram bot tokens matching `\d{8,10}:[a-zA-Z0-9_\-]{35}` are redacted in log output
- [ ] Vault-stored values (when available) are redacted via exact match
- [ ] Vault values are populated dynamically after vault unlock via shared `Arc<RwLock<Vec<String>>>`
- [ ] If RedactingWriter initialization fails, startup halts (refuse to start)
- [ ] Replacement text is uniform `[REDACTED]` with no type metadata leakage

## Context

**Current state:** `init_tracing()` (serve.rs:616-627) creates a `tracing_subscriber::fmt()` subscriber with `EnvFilter`, default stdout writer, and `with_target(true)`. No redaction.

**Target state:** `init_tracing()` returns an `Arc<RwLock<Vec<String>>>` handle that serve.rs later populates with vault values. The tracing subscriber uses a `RedactingMakeWriter` that creates `RedactingWriter<std::io::Stderr>` instances per log event.

**Key insight:** `init_tracing()` is called BEFORE vault startup check. The vault values `Arc<RwLock<Vec<String>>>` starts empty and gets populated later -- the dynamic sharing via Arc<RwLock> handles this naturally.

<title>Implement MakeWriter wrapper for RedactingWriter</title>
<actions>
1. **Edit `crates/blufio-security/src/redact.rs`**: Add a `MakeWriter` implementation for use with tracing-subscriber. This requires the `tracing-subscriber` crate as a dependency.

   **Alternative (simpler):** Instead of depending on tracing-subscriber in blufio-security, implement the MakeWriter directly in serve.rs where it's used. This avoids adding tracing-subscriber to the security crate.

   **Chosen approach:** Implement in serve.rs to keep blufio-security focused on security primitives.

2. **Edit `crates/blufio/src/serve.rs`**: Add a `RedactingMakeWriter` struct at module level:
   ```rust
   use blufio_security::RedactingWriter;
   use std::sync::{Arc, RwLock};

   /// A MakeWriter implementation that creates RedactingWriter instances.
   ///
   /// Used by tracing-subscriber to ensure all log output passes through
   /// secret redaction before reaching stderr.
   struct RedactingMakeWriter {
       vault_values: Arc<RwLock<Vec<String>>>,
   }

   impl<'a> tracing_subscriber::fmt::MakeWriter<'a> for RedactingMakeWriter {
       type Writer = RedactingWriter<std::io::Stderr>;

       fn make_writer(&'a self) -> Self::Writer {
           RedactingWriter::new(std::io::stderr(), self.vault_values.clone())
       }
   }
   ```

   Note: Using `std::io::Stderr` (not Stdout) because tracing-subscriber defaults to stderr.

3. **Edit `crates/blufio/Cargo.toml`**: Add `blufio-security = { path = "../blufio-security" }` to `[dependencies]` if not already present.

<title>Modify init_tracing to use RedactingWriter and return vault handle</title>
<actions>
1. **Edit `crates/blufio/src/serve.rs`**: Change `init_tracing()` signature and implementation:

   **From:**
   ```rust
   fn init_tracing(log_level: &str) {
       use tracing_subscriber::EnvFilter;
       let filter = EnvFilter::try_from_default_env()
           .unwrap_or_else(|_| EnvFilter::new(format!("blufio={log_level},warn")));
       tracing_subscriber::fmt()
           .with_env_filter(filter)
           .with_target(true)
           .with_thread_names(false)
           .init();
   }
   ```

   **To:**
   ```rust
   /// Initializes the tracing subscriber with secret redaction.
   ///
   /// Returns a shared handle to the vault values list. The caller
   /// populates this after vault unlock so that dynamically-loaded
   /// secrets are redacted in subsequent log output.
   ///
   /// # Panics
   /// Panics if a tracing subscriber is already installed.
   fn init_tracing(log_level: &str) -> Arc<RwLock<Vec<String>>> {
       use tracing_subscriber::EnvFilter;

       let vault_values: Arc<RwLock<Vec<String>>> = Arc::new(RwLock::new(Vec::new()));

       let redacting_writer = RedactingMakeWriter {
           vault_values: vault_values.clone(),
       };

       let filter = EnvFilter::try_from_default_env()
           .unwrap_or_else(|_| EnvFilter::new(format!("blufio={log_level},warn")));

       tracing_subscriber::fmt()
           .with_env_filter(filter)
           .with_target(true)
           .with_thread_names(false)
           .with_writer(redacting_writer)
           .init();

       vault_values
   }
   ```
