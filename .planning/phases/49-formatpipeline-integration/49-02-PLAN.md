impl FormatPipeline {
    /// Single entry point for adapters. Auto-detects markdown structures in LLM
    /// text and formats via degradation pipeline. FullMarkdown channels get passthrough.
    pub fn detect_and_format(text: &str, caps: &ChannelCapabilities) -> String;

/// Split text at paragraph boundaries respecting max_message_length.

pub fn split_at_paragraphs(text: &str, max_length: Option<usize>) -> Vec<String>;

  <name>Task 1: Wire pipeline into Telegram, Discord, Slack, and Matrix adapters</name>
  <files>crates/blufio-telegram/src/lib.rs, crates/blufio-discord/src/lib.rs, crates/blufio-slack/src/lib.rs, crates/blufio-matrix/src/lib.rs</files>
  <action>
For each of the 4 adapters, update send() and edit_message() to follow the standardized 4-step pipeline per CONTEXT.md:

**Pipeline order (hard contract):** `detect_and_format(text, caps)` -> `adapter_escape(formatted)` -> `split_at_paragraphs(escaped, limit*0.9)` -> send each chunk

**Telegram (crates/blufio-telegram/src/lib.rs):**
- In send(): Get text from OutboundMessage. Call `FormatPipeline::detect_and_format(&text, &self.capabilities())`. Then call `format_for_telegram()` (existing MarkdownV2 escaping). Then `split_at_paragraphs(&escaped, caps.max_message_length)`. Loop over chunks, send each via API with parse_mode=MarkdownV2. If API returns parse error, retry that chunk as plain text and increment `metrics::counter!("blufio_format_fallback_total", "channel" => "telegram")`. If chunk N fails for a non-parse reason, abort remaining chunks and return error. Return MessageId of first chunk.
- In edit_message(): Same pipeline (detect_and_format -> escape -> but NO split since edits replace full message).
- Verify capabilities() has accurate extended fields: streaming_type=BlockOnly, formatting_support=BasicMarkdown (Telegram MarkdownV2 is not full GFM), rate_limit from Telegram API limits.

**Discord (crates/blufio-discord/src/lib.rs):**
- In send(): Call `FormatPipeline::detect_and_format(&text, &self.capabilities())`. Discord is FullMarkdown, so detect_and_format returns text as-is. No additional escaping needed (Discord renders markdown natively). Call `split_at_paragraphs(&formatted, caps.max_message_length)`. Loop over chunks, send each. Return MessageId of first chunk. If chunk N fails, abort.
- In edit_message(): Same pipeline, no split.
- Verify capabilities(): streaming_type=FullStreaming, formatting_support=FullMarkdown, supports_code_blocks=true, supports_embeds=true, max_message_length=Some(2000).

**Slack (crates/blufio-slack/src/lib.rs):**
- In send(): Call `FormatPipeline::detect_and_format(&text, &self.capabilities())`. Slack is FullMarkdown, detect_and_format returns text as-is (mrkdwn renders natively). No additional escaping. Call `split_at_paragraphs(&formatted, caps.max_message_length)`. Loop over chunks, send each. Return MessageId of first chunk.
- In edit_message(): Same pipeline, no split.
- Verify capabilities(): streaming_type=BlockOnly, formatting_support=FullMarkdown, max_message_length=Some(40000), supports_threads=true.

**Matrix (crates/blufio-matrix/src/lib.rs):**
- In send(): Call `FormatPipeline::detect_and_format(&text, &self.capabilities())`. Matrix is HTML, so detect_and_format returns semantic HTML for tables/lists/code. No additional escaping (HTML is the native format). Call `split_at_paragraphs(&formatted, caps.max_message_length)`. Loop over chunks, send each with format=org.matrix.custom.html. Return MessageId of first chunk.
- In edit_message(): Same pipeline, no split.
- Verify capabilities(): streaming_type=None, formatting_support=HTML, max_message_length=Some(65536).

Add `use blufio_core::format::{FormatPipeline, split_at_paragraphs};` to each adapter. Register the `blufio_format_fallback_total` counter using `metrics::counter!()` macro (Telegram only, per existing Prometheus patterns).
