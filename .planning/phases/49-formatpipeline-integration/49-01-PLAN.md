Add to FormatPipeline in crates/blufio-core/src/format.rs:

1. Add `pub fn detect_and_format(text: &str, caps: &ChannelCapabilities) -> String` to FormatPipeline impl block. This is the single entry point for adapters. Logic:
   - If caps.formatting_support is FullMarkdown, return text as-is (Discord, Slack render markdown natively)
   - Otherwise, scan text for markdown structures using regex-based detection:
     - Tables: look for lines matching `|...|` with a separator row matching `|---|` (or `|:---|`, `|---:|`, `|:---:|`)
     - Lists: lines starting with `- ` or `N. ` (where N is a digit)
     - Code blocks: triple backtick fences ``` with optional language tag

   - For mixed content (text + table + text), process segments sequentially and concatenate into a single String
   - Conservative: if detection regex doesn't match cleanly, leave text as-is

3. Add HTML rendering for lists: if caps.formatting_support == HTML, render List as `<ul><li>` or `<ol><li>`.

4. Add HTML rendering for code blocks: if caps.formatting_support == HTML, render CodeBlock as `<pre><code class="language-X">`.

  <files>crates/blufio-core/src/format.rs</files>
  <behavior>
    - split_at_paragraphs("short text", 4096) returns vec!["short text"] (under limit, no split)
    - split_at_paragraphs(long_text, 4096) splits at double newline boundaries
    - split_at_paragraphs(long_text_no_double_newline, 4096) splits at single newline boundaries
    - split_at_paragraphs(long_text_no_newlines, 4096) splits at sentence boundary (`. `)
    - Code blocks (```...```) are atomic -- never split inside them; if a code block exceeds limit, send as one chunk with a warning log
    - Lists (consecutive `- ` or `N. ` lines) are atomic units
    - Tables with `|---|` separator are atomic but if split, header rows are repeated in each chunk
    - Uses 90% of max_length as threshold to leave room for adapter formatting expansion
    - Returns Vec<String> with no metadata per chunk
    - Empty input returns vec![""]
    - max_length of None returns input as single chunk (no splitting)
  </behavior>
  <action>
Add to crates/blufio-core/src/format.rs:

1. Add `pub fn split_at_paragraphs(text: &str, max_length: Option<usize>) -> Vec<String>` as a free public function (not on FormatPipeline -- utility function).

2. Implementation:
   - If max_length is None, return vec![text.to_string()]
   - Compute effective_limit = (max_length * 9) / 10 (90% threshold, integer math)
   - If text.len() <= effective_limit, return vec![text.to_string()]
   - Identify atomic blocks: scan for code fences (```...```), list blocks (consecutive lines starting with `- ` or digits+`. `), table blocks (lines with `|` and a `|---|` separator)
   - Split priority: double newline (`\n\n`) > single newline (`\n`) > sentence boundary (`. ` followed by uppercase or end)
   - Never split inside an atomic block. If a single atomic block exceeds effective_limit, emit `tracing::warn!` and include it as its own chunk
   - For tables that must be split (very long tables), repeat header + separator rows in each chunk
   - Return Vec<String> of chunks

3. Write comprehensive test matrix per CONTEXT.md decision: {short, at-limit, over-limit} x {plain, code block, list, table, mixed} x {various channel limits}. Also test: None max_length, empty input, single atomic block exceeding limit.
