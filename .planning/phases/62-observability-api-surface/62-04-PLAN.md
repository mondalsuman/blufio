  <action>
1. Read ALL handler files first to understand every request/response type and handler function signature.

2. For each request/response struct, add #[derive(utoipa::ToSchema)] and #[schema(example = ...)] on key fields:
   - handlers.rs: MessageRequest, MessageResponse, HealthResponse, and any other Serialize/Deserialize structs
   - openai_compat/handlers.rs: ChatCompletionRequest, ChatCompletionResponse, ModelInfo, etc.
   - openai_compat/responses.rs: ResponseRequest, ResponseObject, etc.
   - openai_compat/tools.rs: ToolInfo, ToolInvokeRequest, ToolInvokeResponse, etc.
   - api_keys/handlers.rs: CreateApiKeyRequest, ApiKeyInfo, etc.
   - webhooks/handlers.rs: CreateWebhookRequest, WebhookInfo, etc.
   - batch/handlers.rs: BatchRequest, BatchStatus, etc.

   Pattern:
   ```rust
   #[derive(Debug, Deserialize, utoipa::ToSchema)]
   pub struct MessageRequest {
       /// Message content text.
       #[schema(example = "Hello, how are you?")]
       pub content: String,
       ...
   }
   ```

3. For each handler function, add #[utoipa::path(...)] annotation ABOVE the function:
   - Core handlers (handlers.rs):
     ```rust
     #[utoipa::path(
         post,
         path = "/v1/messages",
         tag = "Messages",
         request_body = MessageRequest,
         responses(
             (status = 200, description = "Message processed", body = MessageResponse),
             (status = 400, description = "Invalid request"),
             (status = 401, description = "Unauthorized"),
         ),
         security(("bearer_auth" = []))
     )]
     pub async fn post_messages(...) -> impl IntoResponse { ... }
     ```

   - GET /v1/sessions tag = "Sessions"
   - GET /v1/health tag = "Health", security(("bearer_auth" = []))
   - GET /health tag = "Health" (no security -- public)
   - GET /metrics tag = "Health" (no security -- public)

   - OpenAI compat handlers: tag = "OpenAI Compatible"
     - POST /v1/chat/completions
     - GET /v1/models
     - POST /v1/responses
     - GET /v1/tools
     - POST /v1/tools/invoke

   - API key handlers: tag = "API Keys"
     - POST /v1/api-keys
     - GET /v1/api-keys
     - DELETE /v1/api-keys/{id}

   - Webhook handlers: tag = "Webhooks"
     - POST /v1/webhooks
     - GET /v1/webhooks
     - DELETE /v1/webhooks/{id}

   - Batch handlers: tag = "Batch"
     - POST /v1/batch
     - GET /v1/batch/{id}

4. For path parameters, use utoipa's path(params(...)):
   ```rust
   #[utoipa::path(
       delete,
       path = "/v1/api-keys/{id}",
       tag = "API Keys",
       params(("id" = String, Path, description = "API key ID to delete")),
       responses((status = 200, description = "Key deleted"), (status = 404, description = "Not found")),
       security(("bearer_auth" = []))
   )]
   ```

5. Add a generic ErrorResponse struct if not already present:
   ```rust
   #[derive(Debug, Serialize, utoipa::ToSchema)]
   pub struct ErrorResponse {
       /// Error message.
       #[schema(example = "Invalid request body")]
       pub error: String,
   }
   ```

6. For SSE streaming endpoints (POST /v1/chat/completions with stream=true), document the event format in the description string within the #[utoipa::path] annotation since OpenAPI has limited SSE support.

7. IMPORTANT: Do NOT add ToSchema to GatewayState, HealthState, or other internal state types. State extractors are invisible to OpenAPI -- only request body (Json<T>), query params (Query<T>), path params, and response body types need schemas.

8. If a type uses generics or complex enums that utoipa can't auto-derive, use #[schema(value_type = ...)] to provide a simpler schema representation.

   ```rust
           // Batch
           crate::batch::handlers::post_create_batch,
           crate::batch::handlers::get_batch_status,
       ),
       components(schemas(
           // List all request/response types with ToSchema derives
           // Read the actual types from Task 1 to populate this
       )),
       tags(
           (name = "Messages", description = "Message exchange API"),
           (name = "Sessions", description = "Session management"),
           (name = "OpenAI Compatible", description = "OpenAI-compatible endpoints"),
           (name = "API Keys", description = "API key management"),
           (name = "Webhooks", description = "Webhook management"),
           (name = "Batch", description = "Batch processing"),
           (name = "Health", description = "Health and monitoring"),
       ),
       modifiers(&SecurityAddon),
       info(
           title = "Blufio API",
           version = env!("CARGO_PKG_VERSION"),
           description = "Always-on personal AI agent REST API",
           license(name = "MIT OR Apache-2.0"),
       )
   )]
   pub struct ApiDoc;

   struct SecurityAddon;
   impl utoipa::Modify for SecurityAddon {
       fn modify(&self, openapi: &mut utoipa::openapi::OpenApi) {
           if let Some(components) = openapi.components.as_mut() {
               components.add_security_scheme(
                   "bearer_auth",
                   utoipa::openapi::security::SecurityScheme::Http(
                       utoipa::openapi::security::Http::new(
                           utoipa::openapi::security::HttpAuthScheme::Bearer,
                       ),
                   ),
               );
           }
       }
   }
   ```

2. Add `pub mod openapi;` to lib.rs.

3. In server.rs, add /openapi.json route to public_routes (unauthenticated):
   ```rust
   use utoipa::OpenApi;

   async fn get_openapi_json() -> impl IntoResponse {
       let spec = crate::openapi::ApiDoc::openapi()
           .to_pretty_json()
           .unwrap_or_else(|_| "{}".to_string());
       (
           [(axum::http::header::CONTENT_TYPE, "application/json")],
           spec,
       )
   }

   // Add to public_routes builder:
   .route("/openapi.json", get(get_openapi_json))
   ```

4. In server.rs, add Swagger UI (feature-gated), merge it AFTER all routes are built but before the listener bind. The SwaggerUi needs access to the OpenAPI spec. Read the config from the state or pass it as a parameter to start_server:
   ```rust
   #[cfg(feature = "swagger-ui")]
   {
       // swagger_ui_enabled should be passed to start_server or accessible via state
       if swagger_ui_enabled {
           app = app.merge(
               utoipa_swagger_ui::SwaggerUi::new("/docs")
                   .url("/openapi.json", crate::openapi::ApiDoc::openapi())
           );
       }
   }
   ```

   To pass swagger_ui_enabled to start_server, either:
   - Add it to ServerConfig struct, or
   - Add it as a parameter to start_server

   Prefer adding to ServerConfig since it's a config-driven toggle:
   ```rust
   pub struct ServerConfig {
       // ... existing fields ...
       pub swagger_ui_enabled: bool,
   }
   ```

5. Update the serve.rs call site in the blufio binary that constructs ServerConfig to pass openapi.swagger_ui_enabled from the config.

6. Add snapshot test in openapi.rs:

   ```rust
   #[cfg(test)]
   mod tests {
       use super::*;
       use utoipa::OpenApi;

       #[test]
       fn openapi_spec_snapshot() {
           let spec = ApiDoc::openapi();
           let json: serde_json::Value = serde_json::from_str(
               &spec.to_pretty_json().unwrap()
           ).unwrap();
           insta::assert_json_snapshot!("openapi_spec", json);

       fn openapi_spec_is_valid_json() {

           let _: serde_json::Value = serde_json::from_str(&json).unwrap();
       }
   }
   ```
