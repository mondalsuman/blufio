  <action>
1. Create crates/blufio/src/otel.rs gated behind #[cfg(feature = "otel")]:

2. Implement `try_init_otel_layer()` function that:
   - Takes &OpenTelemetryConfig (from blufio_config::model)
   - Returns Option<(OpenTelemetryLayer, SdkTracerProvider)> -- layer for subscriber, provider for shutdown
   - If config.enabled is false, returns None immediately
   - Builds SpanExporter via SpanExporter::builder().with_http().with_endpoint(&config.endpoint).build()
   - On exporter build failure: tracing::warn!() and return None (never panic)
   - Builds Resource with service.name, service.version (env!("CARGO_PKG_VERSION")), deployment.environment, plus any config.resource_attributes
   - Builds BatchSpanProcessor with configured max_queue_size, max_export_batch_size, batch_timeout_ms (as scheduled_delay Duration)
   - Builds SdkTracerProvider with processor and resource
   - Sets global W3C TraceContextPropagator via opentelemetry::global::set_text_map_propagator()
   - Creates OpenTelemetryLayer from a tracer obtained from the provider
   - Returns Some((layer, provider))

3. Implement `shutdown_otel(provider: SdkTracerProvider)` function that:
   - Calls provider.shutdown() to flush pending spans
   - Logs success/failure at info/warn level
   - Wraps in tokio::task::spawn_blocking if needed for safety (the project uses multi-thread runtime so this should be fine inline)

4. Define helper macro `otel_span!` with two variants:
   ```rust
   #[cfg(feature = "otel")]
   macro_rules! otel_span {
       ($name:expr $(, $key:expr => $val:expr)* $(,)?) => {{
           tracing::info_span!($name $(, $key = %$val)*)
       }};
   }

   #[cfg(not(feature = "otel"))]
   macro_rules! otel_span {
       ($name:expr $(, $key:expr => $val:expr)* $(,)?) => {{
           tracing::debug_span!($name)
       }};
   }
   ```
   Export the macro with #[macro_export] so downstream crates can use it.

5. Add Prometheus meta-metrics (using metrics crate facade):
   - counter!("otel_spans_exported_total")
   - counter!("otel_spans_dropped_total")
   - counter!("otel_export_errors_total")
   Note: These are counters that the BatchSpanProcessor doesn't natively expose. For initial implementation, increment otel_spans_exported_total in the OTel layer creation path and otel_export_errors_total on exporter build failure. Full per-export tracking requires a custom SpanProcessor wrapper -- for now, emit descriptive counters and document the limitation.

6. In main.rs, add `#[cfg(feature = "otel")] mod otel;` after existing module declarations.

7. Add unit test (feature-gated) that verifies:
   - try_init_otel_layer with enabled=false returns None
   - try_init_otel_layer with enabled=true + invalid endpoint returns None (graceful failure)

<task type="auto">
  <name>Task 2: Refactor init_tracing to registry-based layered subscriber</name>
  <files>crates/blufio/src/serve.rs</files>
  <action>
1. Refactor init_tracing to use tracing_subscriber::registry() layered approach instead of fmt().init() shorthand. This is CRITICAL because the fmt shorthand cannot compose with the OTel layer.

   Change signature to:
   ```rust
   fn init_tracing(
       log_level: &str,
       #[cfg(feature = "otel")] otel_config: &blufio_config::model::OpenTelemetryConfig,
   ) -> (
       std::sync::Arc<std::sync::RwLock<Vec<String>>>,
       #[cfg(feature = "otel")] Option<opentelemetry_sdk::trace::SdkTracerProvider>,
   )
   ```

   Note: Since cfg on function params is not stable Rust, instead use a simpler approach -- always pass the config and return an Option<SdkTracerProvider> only when otel feature is enabled. Or just make two separate signatures with cfg:

   Better approach: Return a struct or tuple that's cfg-dependent:
   ```rust
   struct TracingState {
       vault_values: Arc<RwLock<Vec<String>>>,
       #[cfg(feature = "otel")]
       otel_provider: Option<opentelemetry_sdk::trace::SdkTracerProvider>,
   }
   ```

2. Implementation:
   ```rust
   fn init_tracing(log_level: &str, config: &BlufioConfig) -> TracingState {
       use tracing_subscriber::prelude::*;
       use tracing_subscriber::EnvFilter;

       let vault_values = Arc::new(RwLock::new(Vec::new()));
       let redacting_writer = RedactingMakeWriter { vault_values: vault_values.clone() };

       let filter = EnvFilter::try_from_default_env()
           .unwrap_or_else(|_| EnvFilter::new(format!("blufio={log_level},warn")));

       let fmt_layer = tracing_subscriber::fmt::layer()
           .with_target(true)
           .with_thread_names(false)
           .with_writer(redacting_writer);

       #[cfg(feature = "otel")]
       {
           let otel_result = crate::otel::try_init_otel_layer(&config.observability.opentelemetry);
           if let Some((otel_layer, provider)) = otel_result {
               tracing_subscriber::registry()
                   .with(filter)
                   .with(fmt_layer)
                   .with(otel_layer)
                   .init();
               return TracingState {
                   vault_values,
                   otel_provider: Some(provider),
               };
           }
           // OTel disabled or failed -- fall through to no-OTel path
           tracing_subscriber::registry()
               .with(filter)
               .with(fmt_layer)
               .init();
           return TracingState {
               vault_values,
               otel_provider: None,
           };
       }

       #[cfg(not(feature = "otel"))]
       {
           tracing_subscriber::registry()
               .with(filter)
               .with(fmt_layer)
               .init();
           TracingState { vault_values }
       }
   }
   ```

3. Update the call site in `serve()` function:
   - Change `let vault_values = init_tracing(&config.agent.log_level);` to use new signature
   - Store the TracingState (including otel_provider) for later shutdown

4. Add OTel shutdown in graceful shutdown path:
   - After agent_loop.run() returns and before audit trail cleanup
   - #[cfg(feature = "otel")] { if let Some(provider) = tracing_state.otel_provider { crate::otel::shutdown_otel(provider); } }

5. Ensure the `use tracing_subscriber::prelude::*;` import is added for the `.with()` layer combinator methods.

6. IMPORTANT: The refactored init_tracing with registry() must produce IDENTICAL log output to the current fmt().init() version when OTel is disabled. The fmt::layer() with same options (with_target, with_thread_names, with_writer) is the exact equivalent.

7. If feature mismatch detected at runtime (config has otel enabled but feature not compiled), log a warning:

   ```rust
       // Can't use tracing yet since subscriber not installed, use eprintln
       eprintln!("WARNING: OpenTelemetry enabled in config but 'otel' feature not compiled. Rebuild with --features otel");
   }
   ```
