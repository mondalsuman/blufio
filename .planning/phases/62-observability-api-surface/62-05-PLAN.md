Implement Litestream WAL replication support: CLI subcommands (init/status), config template generation, WAL autocheckpoint pragma, SQLCipher incompatibility documentation, and doctor integration.

Purpose: Operators running unencrypted databases can use Litestream for continuous WAL replication to S3. This plan provides the tooling to configure and monitor that integration, while clearly documenting the SQLCipher incompatibility with a recommended alternative.
Output: `blufio litestream init` generates config templates, `blufio litestream status` checks replication, WAL pragma set when enabled, doctor check, SQLCipher warning.

   ```rust
   /// Litestream WAL replication management.
   #[command(
       after_help = "Litestream enables continuous SQLite WAL replication to S3-compatible storage.\n\n\
       NOTE: Litestream is INCOMPATIBLE with SQLCipher encrypted databases.\n\
       If encryption is enabled, use `blufio backup` + cron for scheduled backups.\n\n\
       Examples:\n  blufio litestream init\n  blufio litestream status"

   },
   ```

5. In litestream.rs, implement `run_litestream_init(config: &BlufioConfig)`:
   - Check if litestream binary exists: `which litestream` or `command -v litestream`
     - If not found: print helpful message with install instructions (https://litestream.io/install/)
     - Continue anyway (template is still useful for later)
   - Determine DB path from config.storage.database_path
   - Determine audit DB path: same directory as DB, with .audit.db suffix (following Phase 54 convention)
   - Check if SQLCipher encryption is active (config.storage has encryption settings). If active:
     - Print WARNING that Litestream is incompatible with SQLCipher
     - Recommend `blufio backup` + cron as alternative
     - Ask user if they want to continue generating template anyway
   - Generate litestream.yml template alongside the DB file:
     ```yaml
     # Litestream configuration for Blufio
     # Generated by: blufio litestream init
     #
     # WARNING: Litestream is INCOMPATIBLE with SQLCipher encrypted databases.
     # If encryption is enabled, use `blufio backup` + cron instead.
     # See: https://github.com/benbjohnson/litestream/issues/177
     #
     # Documentation: https://litestream.io/reference/config/

     dbs:
       - path: {db_path}
         replicas:
           - type: s3
             bucket: your-bucket-name
             path: blufio/main
             region: us-east-1
             access-key-id: YOUR_ACCESS_KEY
             secret-access-key: YOUR_SECRET_KEY
             retention: 72h
             snapshot-interval: 24h

       - path: {audit_db_path}
         replicas:
           - type: s3
             bucket: your-bucket-name
             path: blufio/audit
             region: us-east-1
             access-key-id: YOUR_ACCESS_KEY
             secret-access-key: YOUR_SECRET_KEY
             retention: 168h
             snapshot-interval: 24h
     ```
   - Write template to {db_dir}/litestream.yml
   - Print success message with next steps: `litestream replicate -config {path}`

6. Implement `run_litestream_status(config: &BlufioConfig)`:
   - Check if litestream binary exists. If not, error with install instructions.
   - Check if [litestream].enabled in config. If not, warn that it's not enabled.
   - Shell out to `litestream replicate -config {path} -exec "echo ok"` or `litestream databases` to check status
   - Actually, the standard way to check is: look for a running litestream process replicating the DB, or parse `litestream generations` output
   - For simplicity: shell out to `litestream generations -config {config_path}` and parse stdout
   - If command fails, report the error helpfully
   - Display: DB path, last generation, replication lag estimate (if available from Litestream output)

7. Add unit tests:
   - Test that template YAML generation produces valid content (string contains expected keys)
   - Test with mock litestream output parsing (pre-recorded stdout)
   - Test SQLCipher warning detection from config

   ```rust
   async fn check_litestream(config: &BlufioConfig) -> CheckResult {
       let start = Instant::now();

       if !config.litestream.enabled {
           return CheckResult {
               name: "Litestream".to_string(),
               status: CheckStatus::Pass,
               message: "Not enabled (skipped)".to_string(),
               duration: start.elapsed(),
           };
       }

       // Check if litestream binary exists
       let binary_exists = std::process::Command::new("which")
           .arg("litestream")
           .output()
           .map(|o| o.status.success())
           .unwrap_or(false);

       if !binary_exists {
           return CheckResult {
               name: "Litestream".to_string(),
               status: CheckStatus::Warn,
               message: "Litestream enabled in config but binary not found in PATH. Install: https://litestream.io/install/".to_string(),
               duration: start.elapsed(),
           };
       }

       // Check for SQLCipher conflict
       // if encrypted...

       CheckResult {
           name: "Litestream".to_string(),
           status: CheckStatus::Pass,
           message: "Litestream binary found".to_string(),
           duration: start.elapsed(),
       }
   }
   ```
