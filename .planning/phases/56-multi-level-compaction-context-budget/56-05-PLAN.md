**Find CLI structure:** Look at existing CLI subcommand patterns in `crates/blufio/src/cli.rs` (or wherever clap subcommands are defined). Follow the exact same pattern for new subcommands.

**Add `context` subcommand group** with three subcommands:

1. **`blufio context compact --dry-run --session <session_id>`**:
   - Loads messages for the session from storage.
   - Runs L0->L1 compaction via `compact_to_l1()`.
   - Runs quality scoring via `evaluate_quality()`.
   - If `--dry-run`: prints the summary, quality scores (entity/decision/action/numerical), weighted score, and gate result. Does NOT persist.
   - If not dry-run: persists the compaction summary, deletes original messages, prints result.
   - Requires a ProviderAdapter instance (use the same provider setup as shell.rs for CLI commands).
   - Uses sync Database::open (following Phase 55 decision: "CLI uses Database::open for classification query access").

2. **`blufio context archive list [--user <user_id>]`**:
   - Lists all archives, optionally filtered by user_id.
   - Columns: id (truncated), user_id, quality_score, sessions (count), classification, created_at, token_count.
   - Uses `queries::archives::list_archives()`.

3. **`blufio context archive view <archive_id>`**:
   - Shows full archive details: id, user_id, summary (full text), quality_score, session_ids (JSON), classification, created_at, token_count.
   - Uses `queries::archives::get_archive()`.

4. **`blufio context archive prune --user <user_id> [--keep <N>]`**:
   - Enforces rolling window for a user. Default keep = max_archives from config.
   - Deletes oldest archives beyond the keep limit (simple deletion, no merge -- unlike the automated rolling window which merges).
   - Prints how many archives pruned.

5. **`blufio context status --session <session_id>`**:
   - Shows zone usage breakdown for a session:
     - Static zone: X tokens / Y budget (Z%)
     - Conditional zone: X tokens / Y budget (Z%)
     - Dynamic zone: X tokens / Y adaptive budget (Z%)
     - Compaction level: LN (or "none")
     - Message count: N
   - Requires assembling context (without sending to LLM) to get token counts. Use the ContextEngine with a mock/dry-run assembly if possible, or compute token counts directly from stored messages.

**Clap integration:** Add `Context` variant to the top-level `Commands` enum (or whatever the existing pattern is). Add `ContextCommand` enum with `Compact`, `Archive`, `Status` subcommands. Wire into the main match/dispatch.
