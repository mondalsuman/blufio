**Budget module (crates/blufio-context/src/budget.rs):**

1. Define `ZoneBudget` struct:
```rust
pub struct ZoneBudget {
    pub static_budget: u32,       // Default 3000
    pub conditional_budget: u32,  // Default 8000
    pub total_budget: u32,        // From context_budget
}

const SAFETY_MARGIN: f64 = 0.10; // Hardcoded per user decision
```

2. `ZoneBudget::from_config(config: &ContextConfig) -> Self` -- reads static_zone_budget, conditional_zone_budget, context_budget.

3. `ZoneBudget::conditional_effective(&self) -> u32` -- returns `(self.conditional_budget as f64 * (1.0 - SAFETY_MARGIN)) as u32` (budget minus 10% safety margin).

4. `ZoneBudget::dynamic_budget(&self, actual_static_tokens: u32, actual_conditional_tokens: u32) -> u32` -- `self.total_budget.saturating_sub(actual_static_tokens).saturating_sub(actual_conditional_tokens)`.

5. `pub async fn enforce_conditional_budget(messages: Vec<ProviderMessage>, budget: u32, token_cache: &TokenizerCache, model: &str) -> (Vec<ProviderMessage>, Vec<String>)`:
   - Iterates through conditional messages (each from a different provider, in registration order -- memory first, skills second, archive last).
   - Counts tokens for each message group.
   - If total exceeds budget, drops messages from the END (lowest priority = last registered = archive) first, then works backward.
   - Returns `(kept_messages, dropped_provider_names)`.
   - The provider name is extracted from the message metadata or content prefix. If not available, use index-based names ("conditional_provider_0", etc.).
   - Actually, since providers return Vec<ProviderMessage> without names, we need to track which provider produced which messages. Modify the approach: in ContextEngine::assemble(), collect results per-provider with names/indices, THEN pass to enforce_conditional_budget. The function takes `Vec<(String, Vec<ProviderMessage>)>` (provider_name, messages).

6. Add Prometheus metrics: `metrics::gauge!("blufio_context_zone_tokens", "zone" => zone_name).set(token_count as f64)` for static, conditional, dynamic zones.

7. Add unit tests: dynamic_budget computation, conditional_effective with safety margin, budget enforcement drops lowest priority first.

**ContextEngine::assemble() rewrite (crates/blufio-context/src/lib.rs):**

Rewrite the assemble method to enforce zone budgets:

1. **Static zone** (step 1):
   - Get system blocks (existing).
   - Count static zone tokens: `let actual_static = self.static_zone.token_count(&self.token_cache, model).await;`
   - Check budget: `self.static_zone.check_budget(actual_static, self.zone_budget.static_budget);`
   - Record metric: `metrics::gauge!("blufio_context_zone_tokens", "zone" => "static").set(actual_static as f64);`

2. **Conditional zone** (step 2):
   - Collect per-provider results with tracking:
   ```rust
   let mut provider_results: Vec<(String, Vec<ProviderMessage>)> = Vec::new();
   for (i, cp) in self.conditional_providers.iter().enumerate() {
       let ctx = cp.provide_context(session_id).await?;
       let name = format!("conditional_provider_{}", i); // or better name if available
       provider_results.push((name, ctx));
   }
   ```
   - Compute effective conditional budget: `self.zone_budget.conditional_effective()`.
   - Enforce budget: `let (conditional_messages, dropped) = budget::enforce_conditional_budget(provider_results, effective_budget, &self.token_cache, model).await;`
   - Count actual conditional tokens.
   - Record metric: `metrics::gauge!("blufio_context_zone_tokens", "zone" => "conditional").set(actual_conditional as f64);`

3. **Dynamic zone** (step 3):
   - Compute adaptive budget: `let dynamic_budget = self.zone_budget.dynamic_budget(actual_static as u32, actual_conditional as u32);`
   - Pass dynamic_budget to DynamicZone. Modify `assemble_messages()` to accept `dynamic_budget: u32` parameter instead of using `self.context_budget` for threshold computation.
   - Record metric: `metrics::gauge!("blufio_context_zone_tokens", "zone" => "dynamic").set(dynamic_tokens as f64);` (where dynamic_tokens = actual tokens used in dynamic zone).

4. **Combine** (step 4-6):
   - Combine conditional + dynamic messages (existing logic).
   - Build ProviderRequest (existing logic).
   - Return AssembledContext with `dropped_providers: dropped`.
