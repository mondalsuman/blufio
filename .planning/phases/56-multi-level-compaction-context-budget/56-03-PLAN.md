**Quality scoring module (crates/blufio-context/src/compaction/quality.rs):**

1. Define types:
```rust
#[derive(Debug, Clone, serde::Deserialize)]
pub struct QualityScores {
    pub entity: f64,
    pub decision: f64,
    pub action: f64,
    pub numerical: f64,
}

#[derive(Debug, Clone)]
pub struct QualityWeights {
    pub entity: f64,    // default 0.35
    pub decision: f64,  // default 0.25
    pub action: f64,    // default 0.25
    pub numerical: f64, // default 0.15
}

pub enum GateResult {
    Proceed(f64),                    // score >= proceed_threshold
    Retry(f64, String),              // retry_threshold <= score < proceed_threshold, weakest dimension name
    Abort(f64),                      // score < retry_threshold
}
```

2. `QualityWeights::from_config(config: &ContextConfig) -> Self` -- reads weight fields from config.

3. `QualityScores::weighted_score(&self, weights: &QualityWeights) -> f64` -- computes `entity*w.entity + decision*w.decision + action*w.action + numerical*w.numerical`.

4. `QualityScores::weakest_dimension(&self) -> &str` -- returns the name of the dimension with the lowest score (e.g., "numerical" if numerical is lowest).

5. `pub async fn evaluate_quality(provider: &dyn ProviderAdapter, original_messages: &[Message], summary: &str, model: &str) -> Result<QualityScores, BlufioError>`:
   - Build the evaluation prompt (const `QUALITY_SCORING_PROMPT`) per RESEARCH.md Example 3: asks model to score entity/decision/action/numerical retention 0.0-1.0, return ONLY JSON.
   - Call `provider.complete()` with the prompt, max_tokens=256.
   - Parse response with `serde_json::from_str::<QualityScores>()`.
   - On parse failure: log `tracing::warn!("Quality score JSON parse failed, treating as 0.5")`, return `QualityScores { entity: 0.5, decision: 0.5, action: 0.5, numerical: 0.5 }`.

6. `pub fn apply_gate(score: f64, proceed_threshold: f64, retry_threshold: f64, weakest: &str) -> GateResult`:
   - `score >= proceed_threshold` -> `GateResult::Proceed(score)`
   - `score >= retry_threshold` -> `GateResult::Retry(score, weakest.to_string())`
   - else -> `GateResult::Abort(score)`

7. `pub async fn evaluate_with_retry(provider: &dyn ProviderAdapter, original_messages: &[Message], summary: &str, model: &str, weights: &QualityWeights, proceed_threshold: f64, retry_threshold: f64) -> (f64, GateResult)`:
   - Call `evaluate_quality()`, compute weighted score, apply gate.
   - If `GateResult::Retry`: build retry prompt that emphasizes the weakest dimension: "Pay special attention to preserving {dimension}. Previous attempt scored {score}."
   - Call evaluate_quality AGAIN on the same original + summary (not a re-compaction -- just re-evaluation. The retry is for the COMPACTION, not the scoring).
   - Actually, re-reading CONTEXT.md: "1 retry on 0.4-0.6: retry prompt emphasizes the weakest dimension specifically" -- this means RETRY THE COMPACTION with a modified prompt, not re-evaluate. The caller (DynamicZone) handles the actual retry by calling compact_to_l1/l2 again with an enhanced prompt hint.
   - So simplify: `evaluate_with_retry` just evaluates and returns the GateResult. The retry logic (re-compact with emphasis) is in DynamicZone.
   - Return `(weighted_score, gate_result)`.

8. Add Prometheus metric recording: `metrics::histogram!("blufio_compaction_quality_score").record(score)` and `metrics::counter!("blufio_compaction_gate_total", "result" => gate_label).increment(1)` where gate_label is "proceed"/"retry"/"abort".

9. Add unit tests: weighted_score calculation, weakest_dimension identification, gate threshold logic, JSON parse fallback to 0.5.

**Wire quality scoring into DynamicZone (crates/blufio-context/src/dynamic.rs):**
1. After `compact_to_l1()` returns, if `config.quality_scoring` is true:
   - Call `quality::evaluate_quality()` with original messages + summary.
   - Compute weighted score, apply gate.
   - On `GateResult::Proceed`: continue normally, set `result.quality_score = Some(score)`.
   - On `GateResult::Retry`: call `compact_to_l1()` AGAIN with a modified system prompt that adds: "Pay special attention to preserving {weakest_dimension}." Then evaluate again. If second evaluation is still Retry or Abort, treat as Abort.
   - On `GateResult::Abort`: do NOT persist the summary. Log warning. Fall back to truncation (delete oldest messages until under budget). This is the "abort compaction" path.
2. Same for L2 cascade: quality score after compact_to_l2, same gate logic.
3. Store quality score in compaction metadata and in CompactionEvent::Completed.

**Archive module (crates/blufio-context/src/compaction/archive.rs):**

1. `pub async fn generate_l3_archive(provider: &dyn ProviderAdapter, l2_summaries: &[String], session_ids: &[String], model: &str, max_tokens: u32) -> Result<CompactionResult, BlufioError>`:
   - Takes multiple L2 session summaries (from same user, different sessions).
   - Combines them into a cross-session archive summary via LLM call.
   - Prompt: "Create a comprehensive cross-session archive summarizing these conversation sessions. Preserve all named entities, decisions, facts, and action items. Format as a structured summary organized by topic."
   - Returns CompactionResult with level: L3.

2. `pub async fn store_archive(db: &blufio_storage::Database, archive: ArchiveEntry) -> Result<String, BlufioError>`:
   - Calls `queries::archives::insert_archive()` with the archive data.
   - Returns the archive ID.
   - `ArchiveEntry` struct: `{ id: String, user_id: String, summary: String, quality_score: Option<f64>, session_ids: Vec<String>, classification: String, token_count: Option<i64> }`.

3. `pub async fn enforce_rolling_window(db: &blufio_storage::Database, provider: &dyn ProviderAdapter, user_id: &str, max_archives: u32, model: &str, max_tokens: u32) -> Result<(), BlufioError>`:
   - Count archives for user. If count > max_archives:
   - Get the oldest archive. Get the second-oldest archive.
   - Merge them via LLM call into a single "deep archive" summary.
   - Delete the two oldest, insert the merged one with earliest created_at.
   - Repeat until count <= max_archives.
   - Deep merge prompt: "Merge these two archive summaries into one comprehensive summary. Preserve all important information. Prioritize recent events."

4. `pub async fn get_archives_for_context(db: &blufio_storage::Database, user_id: &str, max_token_budget: u32, token_cache: &TokenizerCache, model: &str) -> Result<Vec<String>, BlufioError>`:
   - Fetch recent archives for user (up to max_archives).
   - Accumulate summaries until token budget is reached (using token_cache for counting).
   - Return summaries that fit within the budget.

5. `pub async fn generate_and_store_session_archive(db: &blufio_storage::Database, provider: &dyn ProviderAdapter, session_id: &str, user_id: &str, model: &str, max_tokens: u32, max_archives: u32, config: &ContextConfig) -> Result<Option<String>, BlufioError>`:
   - Called on session close. Fetches L2 summary for the session (from messages table, metadata type = compaction_summary, compaction_level = L2).
   - If no L2 exists: also check for L1 summaries, or raw messages. Generate L2 first if needed.
   - Generate L3 from this session's L2 + any previous archives for the user.
   - Store archive. Enforce rolling window.
   - If archive_enabled is false in config, return None.
   - Classify archive using highest classification from source messages.
   - Filter restricted messages before including in archive (per Phase 53 decision).
