**Level engine (crates/blufio-context/src/compaction/levels.rs):**
1. Define `CompactionLevel` enum: `L0, L1, L2, L3` with `as_str()` method returning "L0"/"L1"/"L2"/"L3".
2. Define `CompactionResult` struct: `summary: String, level: CompactionLevel, quality_score: Option<f64>, tokens_saved: usize, original_count: usize, metadata: serde_json::Value, usage: TokenUsage`.
3. Implement `compact_to_l1`:
   - Takes `provider: &dyn ProviderAdapter, messages: &[Message], model: &str, max_tokens: u32`.
   - Groups messages into user-assistant turn pairs (consecutive user then assistant message).
   - For each turn pair, creates a bullet-point summary (1-2 sentences) via a single LLM call with a prompt that instructs: "Summarize each turn pair as a bullet point. Preserve entities, decisions, actions, and numbers."
   - The prompt receives ALL turn pairs at once and returns bullet points (not one LLM call per pair -- that would be too expensive).
   - Returns `CompactionResult` with `level: L1` and metadata including `compaction_level`, `original_count`, `compacted_at`.
   - max_tokens parameter comes from `config.max_tokens_l1` (default 256 per turn pair, but since all turn pairs go in one call, multiply by pair count capped at 2048).
4. Implement `compact_to_l2`:
   - Takes `provider: &dyn ProviderAdapter, l1_summaries: &str, model: &str, max_tokens: u32`.
   - Takes L1 bullet-point summaries (already in the conversation as a system message) and produces a session-level narrative summary (2-4 paragraphs).
   - Uses the existing `COMPACTION_PROMPT` from mod.rs (which already describes third-person narrative format).
   - Returns `CompactionResult` with `level: L2`.
5. Define L1 compaction prompt as `const L1_COMPACTION_PROMPT: &str` -- instruct the model to produce bullet-point summaries preserving entities, decisions, actions, and numerical data.
6. Add `build_compaction_metadata()` helper that creates the JSON metadata object per CONTEXT.md spec: `{"type": "compaction_summary", "compaction_level": "L1", "original_count": N, "quality_score": ..., "entity": ..., "decision": ..., "action": ..., "numerical": ..., "compacted_at": "..."}`.
7. Add unit tests: verify CompactionLevel::as_str(), verify metadata format, verify L1 prompt contains key preservation instructions.

**DynamicZone rewrite (crates/blufio-context/src/dynamic.rs):**
1. Add new fields to DynamicZone struct:
   - `compaction_enabled: bool`
   - `soft_trigger: f64` (replaces `compaction_threshold`)
   - `hard_trigger: f64`
   - `max_tokens_l1: u32`, `max_tokens_l2: u32`
   - `event_bus: Option<Arc<blufio_bus::EventBus>>`
   - `memory_store: Option<Arc<blufio_memory::store::MemoryStore>>`
   - Remove old `compaction_threshold` field
2. Update `DynamicZone::new()` to accept `ContextConfig` and read new fields. Use `config.effective_soft_trigger()` to handle deprecated compaction_threshold. Add `event_bus` and `memory_store` as optional constructor parameters (None for tests/CLI, Some for serve.rs).
3. Rewrite `assemble_messages()` with dual trigger logic:
   a. Load history, filter restricted messages (existing logic stays).
   b. Count tokens via TokenizerCache (existing logic stays).
   c. Compute `dynamic_budget` -- for now use `self.context_budget` (Plan 04 will compute adaptive budget). Compute `soft_threshold = (dynamic_budget as f64 * self.soft_trigger) as usize` and `hard_threshold = (dynamic_budget as f64 * self.hard_trigger) as usize`.
   d. If `!self.compaction_enabled` or `estimated_tokens <= soft_threshold` or `history.len() <= 2`: normal path (no compaction).
   e. If `estimated_tokens > soft_threshold`: fire L0->L1 compaction:
      - Emit `CompactionEvent::Started` via EventBus (if present).
      - Run entity extraction BEFORE compaction (call `extract::extract_entities()` on the older messages, store via memory_store if present).
      - Call `compact_to_l1()` on older messages (split at midpoint, same as current).
      - Persist L1 summary via `persist_compaction_summary()` with level metadata.
      - Delete original compacted messages via `storage.delete_messages_by_ids()`.
      - Emit `CompactionEvent::Completed`.
   f. After L1, re-estimate tokens. If `new_estimate > hard_threshold`: cascade to L2:
      - Emit CompactionEvent::Started for L2.
      - Call `compact_to_l2()` on the L1 summary text.
      - Replace the L1 summary message with L2 summary (delete L1, persist L2).
      - Emit CompactionEvent::Completed for L2.
   g. Cap cascade at L2 within assembly (L3 is session-close only, handled in Plan 03).
   h. On ANY error during compaction (LLM error, quality gate abort in future): catch with `match` or `if let Err`, log warning, truncate oldest messages to get under budget, continue. NEVER propagate compaction errors to block the agent loop.
4. Update DynamicResult to include `compaction_usages: Vec<TokenUsage>` (may have multiple if cascade). Update lib.rs AssembledContext accordingly -- change `compaction_usage: Option<TokenUsage>` to `compaction_usages: Vec<TokenUsage>` and update all consumers.
5. Fix all existing tests in dynamic.rs for new constructor signature.
