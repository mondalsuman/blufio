  1. In `crates/blufio-memory/src/store.rs`, add three new methods to `impl MemoryStore`:

     a. `pub async fn count_active(&self) -> Result<usize, BlufioError>`:
        - SQL: `SELECT COUNT(*) FROM memories WHERE status = 'active' AND classification != 'restricted'`

     b. `pub async fn get_all_active_with_embeddings(&self) -> Result<Vec<Memory>, BlufioError>`:
        - Same as get_active() but ensures embeddings are loaded (they already are via row_to_memory)
        - Can delegate to get_active() internally

     c. `pub async fn batch_evict(&self, count: usize, decay_factor: f64, decay_floor: f64, importance_boosts: (f64, f64, f64)) -> Result<(usize, f64, f64), BlufioError>`:
        - Compute eviction score in SQL using CASE expression for importance_boost based on source column and julianday arithmetic for decay
        - SQL approach: compute `eviction_score = importance * MAX(power(decay_factor, days), decay_floor)` then DELETE the lowest N rows
        - Use a CTE or subquery: first SELECT the IDs of the lowest-scored active entries, then DELETE those IDs
        - Return (actual_count_deleted, lowest_eviction_score, highest_eviction_score_of_deleted)
        - Wrap in a single transaction to keep FTS5 triggers consistent
        - NOTE: SQLite does not have a native power() function. Use Rust-side computation instead: load (id, source, created_at) for all active memories, compute eviction score in Rust, sort, collect IDs to delete, then DELETE WHERE id IN (...)

  2. Create `crates/blufio-memory/src/eviction.rs`:
     ```rust
     pub async fn run_eviction_sweep(
         store: &MemoryStore,
         config: &MemoryConfig,
         event_bus: &Option<Arc<EventBus>>,
     ) -> Result<(), BlufioError>
     ```
     - Call `store.count_active()`. If count <= config.max_entries, return Ok (nothing to do)
     - Compute evict_count = count - (config.max_entries * 9 / 10) (evict down to 90% of max)
     - Call `store.batch_evict(evict_count, ...)` with config values
     - If evicted > 0, emit `BusEvent::Memory(MemoryEvent::Evicted { count, lowest_score, highest_score })` to event_bus
     - Log eviction results with tracing::info

     a. Define result struct:
        ```rust

            pub duplicates_found: usize,
            pub conflicts_found: usize,
            pub stale_found: usize,
        }
        ```

     b. `pub async fn run_validation(store: &MemoryStore, config: &MemoryConfig, event_bus: &Option<Arc<EventBus>>) -> Result<ValidationResult, BlufioError>`:
        - Load all active memories with embeddings via `store.get_all_active_with_embeddings()`
        - Reuse DEDUP_THRESHOLD (0.9) and CONTRADICTION_THRESHOLD (0.7) as constants (define locally, same values as extractor.rs -- do NOT import from extractor to avoid coupling)
        - O(n^2) pairwise comparison of embeddings using `cosine_similarity()`
        - For duplicates (sim > 0.9): supersede the one with lower confidence
        - For conflicts (0.7 < sim <= 0.9): supersede the older one (newer wins, compare created_at)
        - For stale: find memories older than config.stale_threshold_days where `decay_factor^days <= decay_floor` -- mark as forgotten via soft_delete
        - Emit MemoryEvent::Updated for each supersede, MemoryEvent::Deleted for each stale soft-delete (existing events)
        - Return ValidationResult with counts
        - Log results with tracing::info

     c. `pub fn run_validation_dry_run(memories: &[Memory]) -> ValidationResult`:
        - Same logic but does not modify the store -- returns counts only
        - Used by CLI `blufio memory validate --dry-run`

     ```rust
     pub async fn spawn_background_task(

         event_bus: Option<Arc<EventBus>>,
         cancel: CancellationToken,
     )
     ```
     - Create two interval timers: eviction (config.eviction_sweep_interval_secs), validation (86400 seconds = daily)
     - Skip first immediate tick on both timers

       - On cancel: break with tracing::info log
     - Wrap each call in error handling: `if let Err(e) = ... { tracing::warn!(...) }`
