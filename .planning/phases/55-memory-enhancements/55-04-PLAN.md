  <name>Task 1: Implement file watcher module</name>
  <files>crates/blufio-memory/src/watcher.rs, crates/blufio-memory/src/lib.rs</files>
  <behavior>
    - Test: file_memory_id("./docs/readme.md") produces deterministic "file:" + SHA-256 hash
    - Test: file_memory_id for same path always returns same ID
    - Test: file_memory_id for different paths returns different IDs
    - Test: should_index returns true for "notes.md" when extensions=["md"]
    - Test: should_index returns false for "image.png" when extensions=["md", "txt"]
    - Test: should_index returns true for any extension when extensions is empty (no filter)
    - Test: process_file_change creates a memory with source=FileWatcher and session_id=file_path
    - Test: process_file_change for oversized file is skipped with warning
    - Test: file watcher is not started when config.paths is empty
  </behavior>
  <action>
  1. Create `crates/blufio-memory/src/watcher.rs`:

     a. `pub fn file_memory_id(path: &std::path::Path) -> String`:
        - Canonicalize path (or use as-is if canonicalize fails)
        - Compute SHA-256 of the canonical path string bytes using `sha2::Sha256`
        - Return `format!("file:{}", hex::encode(hash))` -- use `sha2` directly, format the hex manually with `format!("{:x}", hash)` or iterate bytes

     b. `fn should_index(path: &std::path::Path, config: &FileWatcherConfig) -> bool`:
        - If config.extensions is empty, return true (no filter)
        - Check if file extension matches any in config.extensions (case-insensitive)

     c. `async fn process_file_change(path: &std::path::Path, config: &FileWatcherConfig, store: &MemoryStore, embedder: &OnnxEmbedder) -> Result<(), BlufioError>`:
        - If !should_index, skip
        - If file does not exist (deleted), call `store.soft_delete(&file_memory_id(path))` and return
        - Read file metadata. If file size > config.max_file_size, log warning and skip
        - Read file content as UTF-8 string. If not valid UTF-8, skip with warning
        - Generate embedding via `embedder.embed(EmbeddingInput { texts: vec![content.clone()] })`
        - Build Memory struct:
          - id: `file_memory_id(path)`
          - content: file content
          - embedding: from embed result
          - source: MemorySource::FileWatcher
          - confidence: 0.8 (file watcher importance boost level)
          - status: MemoryStatus::Active
          - session_id: Some(path.display().to_string()) -- store file path in session_id field
          - classification: DataClassification::Internal (default)
          - created_at/updated_at: current ISO 8601 timestamp
        - Check if memory with this ID already exists (via store.get_by_id). If exists, this is an update -- delete old first, then save new (ensures FTS5 consistency)
        - Call `store.save(&memory)`

     d. `pub async fn initial_scan(config: &FileWatcherConfig, store: &MemoryStore, embedder: &OnnxEmbedder) -> Result<usize, BlufioError>`:
        - Walk each directory in config.paths recursively
        - For each file matching should_index(), call process_file_change
        - Return count of files indexed

     e. `pub fn start_file_watcher(config: &FileWatcherConfig, store: Arc<MemoryStore>, embedder: Arc<OnnxEmbedder>, cancel: CancellationToken) -> Result<(), BlufioError>`:
        - If config.paths is empty, return Ok (disabled)
        - Create `tokio::sync::mpsc::channel::<Vec<PathBuf>>(100)`
        - Create `notify_debouncer_mini::new_debouncer(Duration::from_millis(500), callback)` where callback uses `tx.blocking_send()` (NOT `.send().await` -- notify runs on its own thread)
        - Watch each path in config.paths with `RecursiveMode::Recursive`
        - Spawn tokio task that loops on `rx.recv()` with `tokio::select!` for cancel
        - In the loop, call `process_file_change()` for each received path
        - Keep the debouncer alive by moving it into the spawned task (`let _debouncer = debouncer;`)

  2. Update `crates/blufio-memory/src/lib.rs`:
     - Add `pub mod watcher;`

  Write unit tests inline for all behaviors listed above.
  </action>
