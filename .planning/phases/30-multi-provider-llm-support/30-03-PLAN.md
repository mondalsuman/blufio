**types.rs** -- OpenRouter wire format:
- Reuse OpenAI-compatible types with additions:
- `RouterRequest`: extends OpenAI chat completion request with `provider: Option<ProviderPreferences>`
- `ProviderPreferences`: order (Vec<String>), allow_fallbacks (bool), data_collection (Option<String>)
- `RouterResponse`: same as OpenAI ChatResponse (id, choices, model, usage)
- SSE types: same as OpenAI (SseChunk, SseDelta, DeltaMessage, DeltaToolCall)
- Note: Do NOT share crate types with blufio-openai -- each crate owns its wire types independently for decoupling

**client.rs** -- HTTP client:
- `OpenRouterClient`: reqwest::Client, base_url (default: "https://openrouter.ai/api/v1"), default_model
- Constructor: takes api_key, SecurityConfig, config values for x_title and http_referer
- Build default headers: Authorization Bearer, Content-Type, X-Title, HTTP-Referer (if set)
- `complete_chat()`: POST to `{base_url}/chat/completions`
- `stream_chat()`: POST with stream: true, return SSE stream (reuse eventsource-stream pattern)
- `validate_key()`: lightweight GET to base or attempt a minimal request to verify API key works. If OpenRouter doesn't have a dedicated auth endpoint, defer validation to first actual request.
- Retry logic: same as Anthropic (429, 500, 503)
- SSRF-safe resolver + TLS 1.2+ from SecurityConfig (OpenRouter is a cloud service)
- `#[cfg(test)] with_base_url()` for wiremock testing

Write wiremock tests verifying correct headers are sent and provider preferences included.

In `crates/blufio-openrouter/src/lib.rs`:

1. Define `OpenRouterProvider` struct with `client: OpenRouterClient`, `system_prompt: String`, `provider_order: Vec<String>`

2. `new(config: &BlufioConfig)`:
   - Resolve API key: `config.providers.openrouter.api_key` first, then `OPENROUTER_API_KEY` env var
   - Load system prompt: file > inline > default pattern
   - Store provider_order from config
   - Construct OpenRouterClient with x_title, http_referer from config

3. Implement `PluginAdapter`:
   - `name()` -> "openrouter"
   - `version()` -> 0.1.0
   - `adapter_type()` -> AdapterType::Provider
   - `health_check()` -> Healthy (defer validation to first request)
   - `shutdown()` -> Ok(())

4. Implement `ProviderAdapter`:
   - `complete()`: Convert ProviderRequest -> RouterRequest (with provider preferences), call client, map response
   - `stream()`: Convert ProviderRequest -> RouterRequest, call client stream, map SSE to ProviderStreamChunk

5. `to_router_request()` conversion:
   - Same as OpenAI mapping (system prompt as system message, ContentBlock conversions)
   - Add `provider` field if `provider_order` is non-empty: `ProviderPreferences { order: self.provider_order.clone(), allow_fallbacks: true }`
   - Model ID passed through directly (full OpenRouter model IDs like "anthropic/claude-sonnet-4")

6. Stream mapping: identical to OpenAI pattern (stateful tool call accumulator, SSE events, [DONE] terminator)

Write tests. Since the wire format is nearly identical to OpenAI, focus tests on the OpenRouter-specific differences: provider preferences, headers, full model IDs.
