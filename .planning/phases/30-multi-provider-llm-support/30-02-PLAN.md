pub trait ProviderAdapter: PluginAdapter {

    async fn stream(&self, request: ProviderRequest) -> Result<Pin<Box<dyn Stream<Item = Result<ProviderStreamChunk, BlufioError>> + Send>>, BlufioError>;

Note: NO blufio-security dependency -- Ollama is local, no TLS enforcement or SSRF protection needed for localhost.

**types.rs** -- Ollama wire format:
- `OllamaRequest`: model, messages (Vec<OllamaMessage>), stream, tools (Option<Vec<OllamaTool>>), format
- `OllamaMessage`: role, content, tool_calls (Option<Vec<OllamaToolCall>>)
- `OllamaTool`: type_="function", function: OllamaFunction (name, description, parameters)
- `OllamaToolCall`: function: OllamaFunctionCall (name, arguments: serde_json::Value)
- `OllamaResponse`: model, message (OllamaMessage), done (bool), done_reason, created_at, prompt_eval_count, eval_count, total_duration, load_duration, prompt_eval_duration, eval_duration
- `TagsResponse`: models (Vec<ModelInfo>)
- `ModelInfo`: name, modified_at, size, digest, details (optional)

**stream.rs** -- NDJSON stream parser:
- `parse_ndjson_stream(response: reqwest::Response) -> Pin<Box<dyn Stream<Item = Result<OllamaResponse, BlufioError>> + Send>>`
- Reads bytes from response, splits on newlines, parses each complete line as JSON
- Uses a buffer to accumulate partial lines across byte chunks
- Empty lines are skipped

**client.rs** -- HTTP client:
- `OllamaClient`: reqwest::Client, base_url, default_model
- Constructor: takes base_url and model. NO api_key. Plain reqwest client (no security hardening -- localhost service).
- `chat()`: POST to `{base_url}/api/chat` with `stream: false`
- `chat_stream()`: POST to `{base_url}/api/chat` with `stream: true`, return NDJSON stream
- `list_tags()`: GET to `{base_url}/api/tags`, return `TagsResponse`
- `health_check()`: GET to `{base_url}/api/tags` (lightweight check that Ollama is running)
- No retry logic for Ollama (local service, transient errors unlikely)
- `#[cfg(test)] with_base_url()` for wiremock testing

Write comprehensive wiremock tests.

2. `new(config: &BlufioConfig)`:
   - Validate `config.providers.ollama.default_model` is Some and non-empty. Error: "Ollama requires an explicit default_model in config. Set providers.ollama.default_model."
   - Construct OllamaClient with base_url from config
   - Attempt health check (GET /api/tags). On failure, error: "Ollama not reachable at {base_url}. Is it running?"
   - Load system prompt: same file > inline > default pattern

3. Implement `PluginAdapter`:
   - `name()` -> "ollama"
   - `version()` -> 0.1.0
   - `adapter_type()` -> AdapterType::Provider
   - `health_check()` -> call client.health_check(), map to Healthy/Unhealthy
   - `shutdown()` -> Ok(())

4. Implement `ProviderAdapter`:
   - `complete()`: Convert ProviderRequest -> OllamaRequest, call client.chat(), map to ProviderResponse
     - Generate UUID for response ID (Ollama doesn't provide one)
     - Map prompt_eval_count -> input_tokens, eval_count -> output_tokens
     - Cache tokens: 0 (Ollama has no prompt caching)
   - `stream()`: Convert ProviderRequest -> OllamaRequest, call client.chat_stream(), map NDJSON chunks to ProviderStreamChunk

5. `to_ollama_request()` conversion:
   - `system_prompt` -> prepend `OllamaMessage { role: "system", content: text }` to messages
   - ContentBlock::Text -> plain text content
   - ContentBlock::ToolUse -> assistant message with tool_calls array
   - ContentBlock::ToolResult -> tool role message
   - ContentBlock::Image -> skip with warning (model-dependent, not universally supported)
   - `tools` -> map ToolDefinition to OllamaTool (same format as OpenAI: type/function)

6. Stream mapping:
   - NDJSON chunks where `done == false` and message.content is non-empty -> ContentBlockDelta with text
   - NDJSON chunks with tool_calls -> ContentBlockStop with tool_use data (Ollama sends complete tool calls, not partials)
   - `done == true` -> MessageStop with stop_reason from done_reason ("stop" -> "end_turn")
   - First chunk -> MessageStart (synthesized)

7. `list_local_models()` public method:
   - Call client.list_tags()
   - Return Vec<String> of model names
