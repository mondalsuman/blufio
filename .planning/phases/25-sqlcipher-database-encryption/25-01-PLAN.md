<task type="auto">
  <name>Task 2: Create centralized connection factory with SQLCipher key support</name>
  <files>crates/blufio-storage/src/database.rs, crates/blufio-storage/src/lib.rs</files>
  <action>
Add the following public functions to `crates/blufio-storage/src/database.rs`:

1. **`apply_encryption_key(conn: &rusqlite::Connection, key: &str) -> Result<(), rusqlite::Error>`** (private helper):
   - If key is exactly 64 characters and all hex digits: use raw hex key with `PRAGMA key = "x'{key}'";`
   - Otherwise: treat as passphrase with `PRAGMA key = '{escaped_key}';` (escape single quotes)

2. **`is_plaintext_sqlite(path: &std::path::Path) -> std::io::Result<bool>`** (public):
   - Read first 16 bytes of file
   - Return true if bytes match `b"SQLite format 3\0"`
   - Return false if file too small or header doesn't match
   - Used by doctor check and encrypt CLI to distinguish plaintext from encrypted

3. **`open_connection(path: &str) -> Result<tokio_rusqlite::Connection, BlufioError>`** (public async):
   - Call `tokio_rusqlite::Connection::open(path)` (create parent dirs first, same as Database::open)
   - Check `std::env::var("BLUFIO_DB_KEY")`
   - If key is set:
     - Call `apply_encryption_key()` as FIRST statement via `conn.call()`
     - Immediately verify with `SELECT count(*) FROM sqlite_master;`
     - On error, map to generic message: "Cannot open database: file is encrypted or not a database. Verify BLUFIO_DB_KEY is correct."
   - If no key set but file exists and is NOT plaintext (check header bytes before opening -- read 16 bytes):
     - Return hard error: "Database is encrypted but BLUFIO_DB_KEY is not set"
   - Return the connection

4. **`open_connection_sync(path: &str, flags: rusqlite::OpenFlags) -> Result<rusqlite::Connection, BlufioError>`** (public):
   - Same logic as open_connection but for sync rusqlite::Connection
   - Used by backup.rs which needs the sync Backup API
   - Call `rusqlite::Connection::open_with_flags(path, flags)`
   - Apply PRAGMA key if BLUFIO_DB_KEY set, verify with SELECT
   - Same error handling as async version

5. **Refactor `Database::open()`** to use `open_connection()` internally:
   - Replace the direct `tokio_rusqlite::Connection::open(path)` call with `open_connection(path).await?`
   - Remove dir creation from Database::open (move to open_connection)
   - Keep the existing PRAGMA setup (WAL, synchronous, etc.) AFTER the connection is opened (PRAGMA key is already applied by factory)
   - Keep migration logic unchanged

6. **Export from lib.rs**: Ensure `open_connection`, `open_connection_sync`, and `is_plaintext_sqlite` are `pub` and accessible from the crate root. Add `pub use database::{open_connection, open_connection_sync, is_plaintext_sqlite};` to lib.rs.

7. **Add tests**:
   - `test_open_connection_no_key`: Opens a connection without BLUFIO_DB_KEY, verifies basic query works
   - `test_open_connection_with_key`: Set BLUFIO_DB_KEY env var, create encrypted DB, verify it works
   - `test_is_plaintext_sqlite_true`: Create a plain DB, check returns true
   - `test_is_plaintext_sqlite_false_encrypted`: Create encrypted DB, check returns false
   - `test_open_connection_wrong_key`: Set wrong key, verify generic error message
   - `test_open_connection_hex_key`: Set 64-char hex key, verify it works
   - `test_encrypted_db_without_key_errors`: Create encrypted DB, unset key, verify hard error about missing key

Use serial_test::serial attribute on tests that modify env vars to avoid race conditions.
