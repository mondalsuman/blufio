## Integration Gaps

### INT-01: TLS Enforcement Not Wired to Anthropic Client (Medium)

- **From:** `blufio-security::tls::build_secure_client()` (Phase 2)
- **To:** `blufio-anthropic::client::AnthropicClient` (Phase 3)
- **Issue:** AnthropicClient creates a plain `reqwest::Client::builder()` without TLS 1.2+ enforcement or `SsrfSafeResolver`
- **Affected:** SEC-03, SEC-04
- **Mitigating:** Anthropic API URL is hardcoded as `https://api.anthropic.com/v1/messages` — reqwest uses HTTPS by default. SSRF is irrelevant for a fixed URL.
- **Fix:** Add `blufio-security` dependency to `blufio-anthropic`, call `build_secure_client()` instead of `Client::builder()`

### INT-02: Secret Redaction Not Installed in Tracing (Medium)

- **From:** `blufio-security::redact::RedactingWriter` (Phase 2)
- **To:** `blufio/src/serve.rs::init_tracing()` (Phase 3)
- **Issue:** `init_tracing()` uses a plain `tracing_subscriber::fmt().init()` — no `RedactingWriter` wrapping
- **Affected:** SEC-08, SEC-09
- **Mitigating:** `RedactingWriter` implementation is complete with regex patterns for API keys, bearer tokens, bot tokens, vault keys. Installation is a single-line change.
- **Fix:** Wrap tracing subscriber writer with `RedactingWriter::new(std::io::stderr())`

### INT-03: Prometheus Business Metrics Not Incremented (Low)

- **From:** `blufio-prometheus::recording::{record_message, record_error, record_tokens, set_active_sessions}` (Phase 9)
- **To:** `blufio-agent::AgentLoop` / `SessionActor` (Phase 3)
- **Issue:** Business-event Prometheus helpers are defined and exported but never called from agent code. Only memory metrics are emitted.
- **Affected:** COST-04, INFRA-05
- **Fix:** Add call sites in `AgentLoop::handle_inbound()` for messages, in error handlers for errors, in `SessionActor::persist_response()` for tokens
