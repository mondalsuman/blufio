# Idea & Concept Review — Devil's Advocate Checklist

## The Acid Tests

### Desirability
- Would YOU use this? Be honest.
- Can you describe the user's pain in ONE sentence without jargon?
- Is this a painkiller or a vitamin? (Painkillers sell. Vitamins don't.)
- If this existed today, how would users discover it?
- What's the user doing RIGHT NOW to solve this problem? (If nothing — maybe it's not a real problem.)

### Viability
- Can you explain the business model in one sentence?
- What's the smallest version that proves the concept?
- Is there a path to revenue within 6 months?
- What needs to be true for this to work? List ALL assumptions.
- Kill criterion: What signal would tell you to stop?

### Feasibility
- Can you build a prototype in a week? If not, it's probably too complex for v1.
- What's the single hardest technical challenge?
- What existing tools/platforms could you build on instead of from scratch?
- Do you have (or can you get) the skills needed?

### Differentiation
- Say "It's like X but Y" — if you can't, the positioning is unclear.
- Google your idea right now. What comes up? Why is yours different?
- If a big company copies this tomorrow, do you survive?

## Anti-Patterns to Flag

- **Solution looking for a problem** — Built cool tech, now searching for users
- **Boiling the ocean** — Trying to serve everyone, serving no one
- **Feature, not product** — Could this just be a feature in an existing tool?
- **Niche of a niche** — Addressable market too small to sustain a business
- **"If we just get 1% of the market"** — This is never a strategy
- **No unfair advantage** — Nothing stopping a clone from eating your lunch

## Output Format

1. **Gut Reaction** — Honest first impression in 2-3 sentences
2. **The Bull Case** — Best-case scenario if everything goes right
3. **The Bear Case** — What kills this idea
4. **Assumptions to Validate** — Ranked list of things to test before building
5. **Suggested Next Step** — One concrete action to take in the next 48 hours
